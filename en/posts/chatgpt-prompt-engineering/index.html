Prompt Engineering Techniques for ChatGPT

Posted on 2026-05-27 Edited on 2026-06-05 In AI도구
Core prompt engineering techniques and patterns for getting the answers you want out of ChatGPT.

What Is Prompt Engineering?

Prompt engineering is the craft of designing input text so that a
large language model (LLM) produces the result you intend. The same model
can yield very different accuracy, format, and depth depending on the
quality of the prompt.

A prompt isn’t just a question — it’s better understood as a kind of
specification that clearly conveys role, context, constraints, and
output format to the model.

Basic Principles

A good prompt includes the following elements.

  1. Clarity: avoid ambiguity, be specific
  2. Context: provide background and purpose
  3. Constraints: specify length, tone, format, etc.
  4. Examples: show samples of the desired output

Zero-shot, One-shot, Few-shot

The way you give examples to an LLM falls into three categories.

Approach Description When to Use
Zero-shot No examples, just instructions For ordinary tasks the model knows well
One-shot A single example When you need to control output format
Few-shot Two or more examples When complex pattern learning is required

Chain-of-Thought (CoT)

For complex reasoning problems, asking the model to think step by step
significantly improves accuracy. The magic phrase “Let’s think step by
step” is a classic.

It’s especially effective on complex code analysis, math problems, and
logic puzzles.

Role Prompting

Assigning a persona to the model changes the tone and depth of its
response.

Role prompting is useful in situations like:

  • When you need advice from a particular job perspective
  • When you want to tune the tone (formal vs. casual)
  • When you want to raise domain expertise

Specifying the Output Format

For business use, structured output is essential. Explicitly ask for
JSON, a table, Markdown, etc.

Forcing the output format makes it much easier to plug into downstream
automation.

Patterns from Practice

Pattern Example
Summary + Constraints “Summarize this article in 5 sentences, in language an elementary school student would understand”
Step Decomposition “Step 1: analyze the problem; Step 2: propose a solution; Step 3: write the code”
Counter-example Request “List 5 edge cases this code might run into”
Self-verification “After you answer, review for errors and fix them yourself”

When It’s Not Working

If your prompt isn’t producing the result you want, check the following.

  • Are you giving too many instructions at once? → break into steps
  • Are there not enough examples? → add few-shot examples
  • Is the output format ambiguous? → specify a template
  • Is the model making up facts? → demand sources or use search-augmented tools

Wrapping Up

Prompt engineering is the new interface design skill of the LLM era.
Just applying the four principles of clear instructions, good examples,
step-by-step reasoning, and structured output dramatically lifts result
quality. Experiment iteratively and build your own personal library of
prompt templates.
