Comparing the TCP and UDP Protocols

Posted on 2026-05-23 Edited on 2026-06-05 In Network , Theory
A summary of the differences between TCP and UDP, the two major transport-layer protocols, and their use scenarios.

The Two Transport-Layer Protocols

TCP (Transmission Control Protocol) and UDP (User Datagram
Protocol) are the most representative protocols belonging to the
transport layer (Layer 4) of the OSI 7-layer model. Both operate
on top of IP and offer different approaches to how application-layer
data is transmitted.

  • TCP: reliability, ordering guarantee, connection-oriented
  • UDP: high speed, simplicity, connectionless

Characteristics of TCP

TCP is a connection-oriented protocol designed for reliable
communication. It guarantees that every packet is delivered in
order and without loss.

The 3-Way Handshake

A TCP connection begins with the following three steps.

  1. SYN: the client sends a connection request and its initial sequence number
  2. SYN-ACK: the server sends a response and its own sequence number
  3. ACK: the client sends the final confirmation

The 4-Way Handshake (Connection Termination)

Reliability Mechanisms

  • Sequence numbers: guarantee packet order
  • Acknowledgments (ACK): confirm receipt
  • Retransmission: retransmit if no ACK arrives within a set time
  • Flow control: adjust the send volume to the receiver’s window size
  • Congestion control: adjust the send rate to the network condition (Slow Start, Congestion Avoidance)

TCP Header Structure

Field Size Description
Source Port 16 bit Source port
Dest. Port 16 bit Destination port
Sequence Number 32 bit Sequence number
Ack Number 32 bit Acknowledgment number
Flags 9 bit SYN, ACK, FIN, etc.
Window 16 bit Window size
Checksum 16 bit Error detection

The total header size is at least 20 bytes.

Characteristics of UDP

UDP is a connectionless protocol focused on simplicity and
speed. It sends data immediately, with no handshake.

UDP Header Structure

Field Size Description
Source Port 16 bit Source port
Dest. Port 16 bit Destination port
Length 16 bit Total length
Checksum 16 bit Error detection

The total header size is 8 bytes — very small compared to TCP.

UDP’s Properties

  • No connection setup: starts sending immediately
  • No ordering guarantee: packets can arrive out of order
  • No retransmission: doesn’t care if data is lost
  • No flow control: sends regardless of the receiver’s state
  • Supports broadcast/multicast

TCP vs UDP Comparison Table

Item TCP UDP
Connection model Connection-oriented Connectionless
Reliability Guaranteed Not guaranteed
Ordering guarantee Guaranteed Not guaranteed
Speed Slow Fast
Header size 20+ bytes 8 bytes
Flow control Yes No
Congestion control Yes No
Broadcast Not supported Supported

Use Scenarios

When TCP Is a Good Fit

  • HTTP/HTTPS: web browsing
  • Email (SMTP, IMAP, POP3)
  • File transfer (FTP, SFTP)
  • Database connections
  • SSH remote access

Choose TCP whenever data accuracy matters more than speed.

When UDP Is a Good Fit

  • Real-time streaming: video/audio (some loss tolerated)
  • Online games: low latency comes first
  • DNS queries: one-off short requests
  • VoIP: voice calls
  • DHCP: IP address assignment
  • QUIC/HTTP/3-based: reliability implemented at the application layer

QUIC and the Future

Recently the QUIC protocol, which guarantees reliability on top
of UDP, has emerged. It’s the foundation of HTTP/3 and combines
TCP’s reliability with UDP’s fast connection setup. This means the
simplistic comparison “TCP is slow, so use UDP” is gradually
becoming meaningless.

Wrapping Up

TCP and UDP aren’t simply fast versus slow — the right tool
differs by purpose. If data accuracy is the key, choose TCP; if
real-time behavior and simplicity are the key, choose UDP. Recently,
hybrid approaches like QUIC that take the strengths of both are also
growing.
