Understanding the Basic Structure of Spring Security

Posted on 2026-05-07 Edited on 2026-07-20 In Backend , Security
A summary of Spring Security's filter chain, the authentication and authorization flow, and how to configure it.

Spring Security Overview

Spring Security is a powerful security framework responsible for
authentication and authorization in Spring-based applications. It
lets you handle varied authentication mechanisms — from simple form
login to OAuth2, JWT, and SAML — in a unified way.

Spring Security’s operation is fundamentally based on the servlet
filter chain. Every HTTP request passes through a
DelegatingFilterProxy to be handled by a SecurityFilterChain,
and the multiple filters inside it inspect the request in sequence.

Core Components

To understand Spring Security, you need to know the following
components.

Component Role
SecurityFilterChain The chain of security filters
AuthenticationManager The entry point for authentication
AuthenticationProvider Verifies the actual credentials
UserDetailsService Looks up user information
SecurityContextHolder Stores the current authentication (ThreadLocal)
GrantedAuthority Represents a user’s permission

Basic Configuration

A standard configuration example as of Spring Boot 3.x.

HttpSecurity uses a lambda DSL for better readability.
WebSecurityConfigurerAdapter has been deprecated since Spring
Security 5.7, so you should use component-based configuration.

The Authentication Flow

The form-login authentication flow is as follows.

  1. The user submits credentials to /login
  2. UsernamePasswordAuthenticationFilter intercepts the request and creates an Authentication object
  3. AuthenticationManager delegates to an AuthenticationProvider
  4. DaoAuthenticationProvider looks up the user via UserDetailsService and verifies the password with PasswordEncoder
  5. On success, the authentication is stored in the SecurityContext
  6. The session keeps the authentication on subsequent requests

Implementing UserDetailsService

For database user authentication, implement UserDetailsService.

Method Security

Besides URL-based security, you can apply security at the method
level too.

JWT-Based Authentication

In REST APIs, JWT (JSON Web Token) is often used instead of
sessions. The key is to create a separate filter that extracts and
verifies the token from the header, then registers it in the
SecurityContext.

In the configuration, set the session to stateless and register the
filter.

Best Practices

  • Always store passwords with a one-way hash like BCryptPasswordEncoder
  • Enable CSRF for session-based web apps, disable it for stateless APIs
  • Declare authorization rules from the most specific first
  • Use @PreAuthorize to express security close to domain rules
  • Enable HTTPS and HSTS in production

Wrapping Up

Spring Security has a steep learning curve, but once you understand
the filter chain and the authentication/authorization flow, you can
solve varied requirements in a consistent way. Before extending to
JWT, OAuth2, or OpenID Connect, it’s important to firmly solidify
the basic flow.
