How a CDN Works and How to Use It

Posted on 2026-05-24 Edited on 2026-06-15 In Network , Theory
The architecture of a CDN that delivers content quickly to users worldwide, plus caching strategies.

What Is a CDN?

A CDN (Content Delivery Network) is a network of servers distributed
across many regions worldwide that delivers content to users from a
location that is geographically close to them. Beyond static assets,
modern CDNs have expanded into dynamic content, APIs, and edge computing.

Using a CDN gives you the following benefits.

  • Lower latency: responses served from an edge close to the user
  • Less origin load: cache hits protect the origin server
  • Lower bandwidth cost: compression and caching at the edge
  • Higher availability: cached content is still served even when the origin is down
  • Better security: DDoS protection, WAF, bot blocking

CDN Architecture

A CDN is broadly made up of two tiers.

1. The Origin Server

The server that holds the original content. Users don’t access it
directly; content is only delivered through the CDN.

2. Edge Servers (Edge / PoP)

Cache servers placed in major cities around the world. Also called
PoPs (Points of Presence). A user’s request is routed to the
closest PoP.

Routing User Requests

There are two main ways a user gets routed to the nearest edge.

DNS-Based Routing

GeoDNS returns different edge IPs based on region.

Anycast Routing

The same IP address is advertised via BGP from many locations worldwide.
The routing protocol automatically forwards traffic to the closest one.
Cloudflare and parts of AWS CloudFront use this approach.

How Caching Works

Cache is the heart of a CDN. Cache behavior is controlled by HTTP headers.

Key Cache Headers

Header Purpose
Cache-Control Cache policy (max-age, public, private, etc.)
Expires Expiration time (older mechanism)
ETag Content identifier (for conditional requests)
Last-Modified Last modification time
Vary Headers to include in the cache key

Cache Hit Rate and Invalidation

If the cache hit rate stays above 90%, origin traffic drops to less than
1/10 of total traffic.

Ways to Invalidate Cache

  1. Wait for TTL to expire: simplest, but immediate updates are hard
  2. Purge API: explicitly invalidate via the CDN
  3. Versioned URLs: app.js?v=2 or app.abc123.js (recommended)
  4. Soft Purge: mark as expired but keep serving stale until refresh

Dynamic Content and Edge Computing

Traditional CDNs focused on static assets, but modern CDNs accelerate
dynamic content too.

Techniques for Accelerating Dynamic Content

  • Connection reuse: keep-alive between edge and origin
  • TCP optimization: congestion control like BBR at the edge
  • TLS termination: TLS handled at the edge; the origin sees plaintext or mTLS
  • Enabling HTTP/3: applying it just on the user-to-edge segment already pays off

Edge Computing

Running code directly at CDN edges has become commonplace.

Service Runtime
Cloudflare Workers V8 isolate
AWS Lambda@Edge Node.js, Python
Vercel Edge Functions V8 isolate
Fastly Compute@Edge WebAssembly

Considerations When Adopting a CDN

Identifying Cacheable Content

Content Cache Suitability Recommended TTL
Images, video Highly suitable 1 year
CSS, JS (hashed) Highly suitable 1 year immutable
HTML Moderate 1–5 minutes
Public GET API Possible 1–60 seconds
API POST/PUT Unsuitable no-store
Personalized pages Unsuitable no-store

Security Considerations

  • Don’t forget cache headers on personal-data responses: Cache-Control: private is mandatory
  • Be careful with caching authenticated responses: use Vary: Authorization or exclude from cache
  • TLS certificate management: the CDN automates issuance and renewal

Major CDN Services

Service Strengths Notes

Akamai Global coverage Enterprise
KT, LG U+ CDN Optimized for Korea Domestic PoPs

Measuring Performance

CDN impact is measured with the following metrics.

  • TTFB (Time To First Byte): drops sharply once you adopt the edge
  • Cache hit rate: aim for over 90%
  • Origin bandwidth: how much is saved by cache
  • Error rate: ratio of 5xx responses

Wrapping Up

A CDN has evolved from a tool that just delivers static files faster
into infrastructure that gives global users consistent performance and
availability. Designing the right cache headers, choosing TTLs per
content type, and adopting edge computing together can cut cost while
significantly improving user experience.
