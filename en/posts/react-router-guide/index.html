How to Use React Router v6

Posted on 2026-05-20 Edited on 2026-06-05 In Frontend , JavaScript
The core React Router v6 APIs - BrowserRouter, Routes, useNavigate - plus nested routing and the data router.

Introducing React Router

React Router is the de facto standard routing library in the
React ecosystem. v6 substantially streamlined the API, making it
more intuitive with a smaller bundle, and since v6.4 a data
router that supports data loading was added.

This post covers the core usage as of v6.

Basic Setup

First, install the library.

Place BrowserRouter at the top of the app and declare Routes
inside Routes.

Unlike v5, you use the element prop instead of component, and
Switch is replaced by Routes.

Links and Programmatic Navigation

Page transitions are done with <Link> or the useNavigate hook.

If you need active-state styling, use <NavLink>.

Dynamic Routes and Queries

Define dynamic segments with the :param notation and read them
with useParams.

Nested Routing and Outlet

Pages that share a layout can be cleanly composed with nested
routes.

In the parent component, place <Outlet> where the child should
render.

The Data Router (createBrowserRouter)

Since v6.4, you can include data fetching and form handling in the
route definition via loader/action.

This pattern fetches data in parallel with routing on page
transitions, reducing waterfall requests.

Protected Routes

Handle routes that require authentication with a wrapper component.

Key Differences from v5

Item v5 v6
Route matching Switch Routes
Specifying the component component element
Nesting Handled manually Automatic + Outlet
useHistory Present useNavigate
Data loading Separate loader/action

Best Practices

  • Separate page components into a pages/ or routes/ directory structure.
  • Make active use of the nested route + Outlet pattern for shared layouts.
  • Use it with React.lazy for code splitting.
  • Adopting the data router makes Suspense, error boundaries, and form handling consistent.

Wrapping Up

React Router v6 provides a well-organized API that lets you do more
with less code. For a small app, start with BrowserRouter, and as
data fetching and forms grow, switching to the data router is a
natural learning path.
