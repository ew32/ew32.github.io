Terraform Basics: Managing Infrastructure with IaC

Posted on 2026-05-26 Edited on 2026-06-15 In Cloud , Infrastructure
A summary of the core concepts and practical usage of managing infrastructure as code with HashiCorp Terraform.

Introduction

As cloud infrastructure grows more complex, managing resources
manually in the console becomes inefficient and risky. IaC
(Infrastructure as Code) is the practice of defining and managing
infrastructure as code, and Terraform is its representative
tool. It supports multi-cloud, is intuitive with declarative syntax,
and has a huge ecosystem.

Why IaC Is Needed

  • Reproducibility: you can create the same environment multiple times
  • Version control: track infrastructure change history with Git
  • Automation: integrate with CI/CD pipelines
  • Documentation: the code itself is the infrastructure spec
  • Collaboration: review changes through code review

Characteristics of Terraform

  • A declarative language (HCL): you describe only the desired state
  • Multi-cloud support: AWS, GCP, Azure, K8s, and more
  • State management: keeps the mapping between actual infrastructure and code
  • Plan and apply separated: you can preview changes in advance

Installation and a First Run

The Basic Workflow

  1. terraform init: install providers
  2. terraform plan: preview changes
  3. terraform apply: apply to the infrastructure
  4. terraform destroy: delete resources

The State File and Backends

Terraform uses a state file (terraform.tfstate) to map actual
infrastructure to code. Storing it locally makes collaboration
hard, so you usually use a remote backend (S3 + DynamoDB).

DynamoDB is used for state locking, preventing multiple people
from applying at the same time.

Best Practices

  • Always store the state file in a remote backend
  • Use environment variables or a Secret Manager for sensitive info (not directly in code)
  • Separate directories per environment (dev/, staging/, prod/)
  • Ensure reusability with modules
  • Attach the terraform plan result to a PR for review
  • Apply security scanners like tfsec, Checkov

Wrapping Up

Terraform is a powerful tool that lets you safely manage cloud
infrastructure as code. Once you learn features like variables,
modules, remote backends, and data sources, you can operate
everything from a small project to a large multi-cloud environment
in a consistent way. I recommend starting with a simple EC2
instance and gradually growing toward adding modules and CI/CD
pipelines.
