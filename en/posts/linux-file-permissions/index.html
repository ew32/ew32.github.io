A Complete Understanding of Linux File Permissions

Posted on 2026-05-25 Edited on 2026-07-20 In System , OS
The Linux file permission system and how to use the chmod and chown commands, with practical examples.

Introduction

Because Linux is a multi-user OS, file permission management is
central. Setting permissions wrong leads to security incidents, or —
the opposite — too-strict settings block normal operation. This post
covers the Linux permission system and related commands like
chmod, chown, and umask.

Checking Permissions

Use ls -l to see permissions for files and directories.

Each part of the output means:

  • - or d: file type (-: regular file, d: directory, l: symbolic link)
  • rw-r--r--: permission info (owner, group, others)
  • 1: hard link count
  • alice: the owner
  • developers: the group
  • 1024: file size

The Three Permission Scopes

Each file has permissions for three kinds of users.

Scope Meaning
User (u) The file owner
Group (g) The group the file belongs to
Others (o) Everyone else

Each scope can hold three permissions.

Symbol Meaning (file) Meaning (directory) Numeric Value
r (read) read list contents 4
w (write) write/modify create/delete files 2
x (execute) execute enter the directory 1

Changing Permissions with chmod

chmod changes permissions. There are two modes.

Numeric (Octal) Mode

Use a number that sums the permissions for each scope.

Octal Permission Meaning
777 rwxrwxrwx All permissions (dangerous)
755 rwxr-xr-x Executables, directories
644 rw-r–r– Regular documents
600 rw——- Secret files
400 r——– Read-only secret

Symbolic Mode

Changing the Owner with chown

chown changes a file’s owner and group. It requires root.

umask: Default Permission Setting

The default permissions of newly created files and directories are
determined by the umask value.

  • Default file permission: 666 - umask
  • Default directory permission: 777 - umask

Special Permissions: SUID, SGID, Sticky Bit

Permission Number Behavior
SUID 4000 Runs with the owner’s privileges (/usr/bin/passwd)
SGID 2000 New files in a directory inherit the group
Sticky Bit 1000 Only the owner can delete files in the directory (/tmp)

ACL: Finer-Grained Permission Control

Traditional permissions make “grant to only a specific user” hard.
An ACL (Access Control List) makes it possible.

Wrapping Up

The Linux permission system looks simple, but in real operations
SUID, SGID, ACLs, and umask interact in combination. Use chmod for
permissions and chown for ownership, and the basis of security is
granting only the minimum necessary permissions. A solid grasp
of permission management helps greatly across server operations,
container deployment, and collaborative environments.
