Spring Boot Actuator Monitoring - Endpoints and Prometheus Integration

Posted on 2026-04-10 Edited on 2026-06-15 In Backend , Java
A practical setup of Spring Boot Actuator's key endpoints, security configuration, Prometheus metric collection via Micrometer, and Grafana visualization.

What Is Actuator?

Spring Boot Actuator is a module that exposes internal information of
a running application — status, metrics, environment, beans,
logging — over HTTP or JMX. Everything from a simple health check
to Prometheus metric collection happens through Actuator.

Getting Started

Add the dependency.

The endpoints exposed by default are health and info. The rest
must be exposed explicitly.

Accessing http://localhost:8080/actuator shows the list of
exposed endpoints.

Key Endpoints

Endpoint Description
/actuator/health Health check. Per-component status — DB, Redis, disk
/actuator/info User-defined info like build/Git info
/actuator/metrics JVM, HTTP, custom metrics
/actuator/prometheus Metrics in Prometheus format
/actuator/env Environment variables, properties
/actuator/loggers Query/change log levels at runtime
/actuator/threaddump A thread dump
/actuator/heapdump Download a heap dump
/actuator/httpexchanges Recent HTTP requests/responses
/actuator/mappings Registered controller mappings

Health Indicators

/actuator/health aggregates the results of registered
HealthIndicators. JDBC DataSource, Redis, RabbitMQ, and others are
auto-registered, and custom ones are easy.

To integrate with Kubernetes liveness/readiness probes, use separate
groups.

This enables the /actuator/health/liveness and
/actuator/health/readiness endpoints.

Micrometer Metrics

Actuator uses Micrometer as its metrics abstraction layer.
Micrometer supports many backends — Prometheus, Datadog, New Relic,
CloudWatch. You write the code once and can swap backends freely.

Auto-Collected Metrics

JVM memory, GC, threads, HTTP requests (http.server.requests),
Tomcat, HikariCP, and more are collected automatically.

Custom Metrics

The three main metric types:

  • Counter — only increments (request count, error count)
  • Gauge — a current value (queue size, active sessions)
  • Timer / Distribution Summary — distributions (request time, response size)

Prometheus Integration

Adding the Micrometer Prometheus registry enables the
/actuator/prometheus endpoint.

tags.application attaches a common tag to all metrics so you can
distinguish metrics from multiple services.

In the Prometheus config, register that endpoint as a scrape target.

Then connect Prometheus as a data source in Grafana and build
dashboards. Importing a public Spring Boot dashboard (e.g., ID 12900,
6756) gets you started immediately.

Security Configuration

Actuator endpoints expose operational info, so they must be
protected.

In production, separating a dedicated Actuator management port is
safer.

This way port 9090 is open only on the internal network, reducing
the risk of external exposure.

Dynamic Log-Level Changes

Very useful when you need to turn on debug logging in production.

Combining with Distributed Tracing

Using Micrometer Tracing (Spring Boot 3+) makes the traceId appear
consistently in metrics and logs, so you can naturally follow
metric anomaly → log → trace.

Wrapping Up

Actuator is basic infrastructure for operating Spring Boot
applications. Even a simple health check has value, but you get
real observability when you combine it with Micrometer + Prometheus

  • Grafana. Be careful with security and limit the exposure scope
    appropriately, and define key business metrics yourself to put on a
    dashboard.
