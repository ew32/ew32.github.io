A Beginner's Guide to MongoDB

Posted on 2026-05-10 Edited on 2026-05-22 In Backend , Database
The structure, basic CRUD, indexes, and data-modeling patterns of MongoDB, the representative document-oriented NoSQL database.

What Is MongoDB?

MongoDB is a document-oriented NoSQL database that stores
BSON documents in a JSON-like format. Its flexible schema enables
fast development, and horizontal scaling (sharding) is built into
its core design.

Document Structure

Documents are stored as BSON (Binary JSON) and can freely contain
nested objects and arrays.

_id is the auto-generated primary key, and an ObjectId is 12 bytes
that includes a timestamp.

Indexes

MongoDB also uses B-Tree indexes. Without an index, a collection
full scan occurs.

Aggregation Pipeline

aggregate is a powerful tool that generalizes SQL’s GROUP BY.

Common stages: $match, $group, $project, $lookup, $unwind,
$sort, $limit.

Data Modeling Patterns

Embedding

Store related data together in one document. Good read performance
and atomicity guarantees.

Referencing

Store another collection’s _id and join with $lookup.

Selection Criteria

Situation Recommended Pattern
1:few, often read together Embedding
1:many, unbounded growth Referencing
Frequently updated shared data Referencing
Risk of exceeding the 16MB document size Referencing

Transactions

Since MongoDB 4.0, multi-document ACID transactions are supported.

Still, where possible it’s better to model with a single document
to obtain atomicity naturally.

Replication and Sharding

  • Replica Set: replicates the same data to multiple nodes, with automatic failover
  • Sharding: distributes data by a shard key for horizontal scaling

Wrapping Up

MongoDB offers a flexible schema and horizontal scalability,
suiting fast-changing domains. But “schemaless” means “the schema
must be managed by the application.” Understanding the trade-offs
between embedding and referencing, your index strategy, and the
aggregation pipeline lets you fully draw out MongoDB’s strengths.
