Web Image Optimization Strategies - WebP, AVIF, and Lazy Loading

Posted on 2026-05-22 Edited on 2026-07-20 In Frontend , HTML
A comprehensive guide to maximizing web performance through modern image formats and loading strategies.

Why Image Optimization Matters

On average, over half of a web page’s transferred bytes are
images. HTTP Archive stats put the average image payload per page
above 1MB — the single biggest contributor to LCP. Image optimization
is the area with the highest performance gain per unit of effort.

Next-Generation Image Formats

WebP

A format Google released in 2010, offering 25–35% smaller files than
JPEG.

  • Pros: supported in every modern browser (96%+), lossy/lossless both, alpha channel
  • Cons: not supported in some older browsers

AVIF

A newer format based on the AV1 codec, roughly 20–30% smaller than
WebP.

  • Pros: best compression, HDR support, wide color gamut
  • Cons: slow to encode, not in all browsers yet
Format Avg size (vs JPEG) Browser support Alpha channel
JPEG 100% 100% No
WebP 70% 96% Yes
AVIF 50% 90% Yes

Format Fallback with picture

To use modern formats while keeping compatibility, use <picture>.

The browser walks top-down and picks the first format it supports —
falling back to the trailing <img> if none are supported.

Responsive Images

Serving the right size for the right screen prevents sending huge
images to mobile.

  • srcset: candidate images with their intrinsic widths
  • sizes: the actual display size at each viewport

Lazy Loading

Defer images that aren’t on-screen to speed up initial load.

Native Lazy Loading

  • loading="lazy": load when the image scrolls near the viewport
  • loading="eager": load immediately (default)
  • Never use lazy on the LCP/above-the-fold image

Using IntersectionObserver

For finer control, use IntersectionObserver.

fetchpriority and preload

Boost critical resources like the LCP image with explicit priority.

Using an Image CDN

An image CDN does format conversion, resizing, and compression
dynamically through URL parameters.

  • Cloudinary: f_auto,q_auto,w_800
  • Imgix: ?auto=format&w=800
  • Cloudflare Images: auto-optimization

SVG and Icon Optimization

Use SVG for icons whenever possible and optimize with SVGO.

Inline SVG cuts an extra request and can be colored via CSS.

Best-Practice Checklist

  • Set width and height on every image (prevents CLS)
  • Provide WebP/AVIF with a JPEG fallback
  • Handle responsiveness with srcset and sizes
  • LCP image: fetchpriority="high" + preload
  • Off-screen images: loading="lazy"
  • Meaningful alt (accessibility)
  • Consider an image CDN

Wrapping Up

Image optimization breaks into three axes: format choice, sizing,
and loading strategy. Combining AVIF/WebP adoption, responsive
images, and lazy loading typically halves page weight and
dramatically improves LCP.
