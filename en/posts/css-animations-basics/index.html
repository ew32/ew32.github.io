CSS Animation Basics - Transitions and Keyframes

Posted on 2026-05-21 Edited on 2026-06-15 In Frontend , CSS
The difference between CSS transitions and @keyframes and how to use each, plus tips for building performant animations.

CSS Animation Overview

Motion on the web tells the user about state changes and guides their
attention. CSS provides two powerful tools for building animations
without JavaScript.

  • Transition: smoothly interpolates between two states (A → B)
  • Animation + @keyframes: defines a complex multi-step motion

This post covers how to use both and the performance considerations.

Transition Basics

transition smooths out a change when a property’s value changes.

The transition shorthand takes four values.

Property Description Example
transition-property Which property to animate opacity, all
transition-duration Duration 300ms, 0.5s
transition-timing-function Acceleration curve ease, linear, cubic-bezier()
transition-delay Delay before starting 100ms

Timing Functions

The acceleration curve hugely affects how the motion feels.

ease-out works well for natural UI; for a springy feel, hand-tune
with cubic-bezier.

Complex Animations with @keyframes

Multi-step motion is defined with @keyframes and applied with the
animation property.

You can also define intermediate keyframes.

The main animation options:

Transition vs Animation

Item Transition Animation
Trigger State change (hover, class added, etc.) Can start automatically
Keyframes Two points: start and end Any number of steps
Repeat Not available Available
Best for Simple state transitions Complex motion, infinite loops

Performant Animations

When the browser handles animation, some properties get GPU
acceleration and others don’t.

Property Cost Recommendation
transform Very low (Composite) Recommended
opacity Very low (Composite) Recommended
width, height, top, left High (Layout) Avoid
background-color, box-shadow Medium (Paint) Avoid if possible

will-change hints to the browser, but overusing it raises memory
usage — apply it only where it’s truly needed.

Accessibility - prefers-reduced-motion

Respect OS settings for users who are sensitive to motion (e.g.
vestibular conditions).

Practical Example - Loading Spinner

Best Practices

  • Stick to transform and opacity to keep 60fps when possible.
  • 200–400ms feels natural for UI transitions.
  • Overly flashy animation actually increases cognitive load.
  • Always handle prefers-reduced-motion for accessibility.

Wrapping Up

CSS alone is plenty for building natural, performant animations. Use
transitions for simple interactions and keyframes for complex or
repeating motion, and reach for GPU-friendly properties. Remember that
small details make a big difference in user experience.
