A Quick Tour of HTTP Status Codes - the Confusing Ones

Posted on 2026-06-23 Edited on 2026-06-05 In Network , HTTP
HTTP status code categories and clear explanations of the codes most often confused (301 vs 308, 401 vs 403, 502 vs 504, and more).

The Big Picture: Five Groups

HTTP status codes are classified by the first digit.

Group Meaning
1xx Informational — in progress
2xx Success — succeeded
3xx Redirection — redirect
4xx Client Error — the client’s fault
5xx Server Error — the server’s fault

For the record — 418 I’m a teapot is also a real code (born of an
April Fools’ joke).

2xx — Success

  • 200 OK: the most common success.
  • 201 Created: a new resource was created. Convention puts the new URL in the Location header.
  • 202 Accepted: request received but still processing. Good for queuing async work.
  • 204 No Content: succeeded with no body. Common for DELETE responses or empty PUT responses.
  • 206 Partial Content: a response to a Range request (video streaming, large-file chunked downloads).

3xx — Redirection

  • 301 Moved Permanently: permanent move. Teaches search engines the new URL.
  • 302 Found: temporary move. The method may switch to GET (historically ambiguous).
  • 303 See Other: redirect to GET after a POST (PRG pattern).
  • 307 Temporary Redirect: method-preserving temporary move.
  • 308 Permanent Redirect: method-preserving permanent move. The modern replacement for 301.
  • 304 Not Modified: the cache is still valid. No body (see ETag/If-None-Match).

If 301 vs 308 trips you up, memorize “use 308 when you need to preserve the method.“

4xx — Client Errors

  • 400 Bad Request: bad syntax/format.
  • 401 Unauthorized: authentication failed (we don’t know who you are). Use with WWW-Authenticate.
  • 403 Forbidden: authorization failed (we know who you are, but you can’t).
  • 404 Not Found: the resource doesn’t exist.
  • 405 Method Not Allowed: the resource exists but doesn’t support that method. Return an Allow header listing what’s allowed.
  • 409 Conflict: conflicts with current state (concurrent update, duplicate registration).
  • 410 Gone: permanently gone. Unlike 404, search engines drop the index faster.
  • 413 Payload Too Large: upload exceeded the limit.
  • 415 Unsupported Media Type: Content-Type not supported.
  • 422 Unprocessable Entity: syntactically valid but semantically invalid (field validation).
  • 429 Too Many Requests: rate limit exceeded. Use Retry-After to indicate when to retry.

The 401 vs 403 confusion is a perennial. Remember “401 is ‘who are you?’ — 403 is ‘I know you, and no.’“

5xx — Server Errors

  • 500 Internal Server Error: the catch-all for any other server failure.
  • 501 Not Implemented: the method itself isn’t implemented.
  • 502 Bad Gateway: a gateway/proxy got a bad response from the backend.
  • 503 Service Unavailable: temporary overload or maintenance. Conventionally paired with Retry-After.
  • 504 Gateway Timeout: a gateway/proxy timed out waiting for the backend.

502 vs 504: 502 if the backend gave a bad response, 504 if it gave no response at all.

A Recommended Mapping for REST Design

Operation Success Failure
Read (GET) 200 / 304 404 / 410
Create (POST) 201 / 202 400 / 409 / 422
Update (PUT/PATCH) 200 / 204 404 / 409 / 422
Delete (DELETE) 204 404 / 409
Authentication required — 401
Not authorized — 403
Too many requests — 429
