Network-Based and Disk-Based File Systems

Posted on 2024-07-08 Edited on 2026-06-15 In System , Storage
A comparison of the structure, pros and cons, and representative examples of network-based file systems and local disk file systems.

Network-Based File Systems

A network-based file system is one where files are stored not on a
local disk but on a server connected over the network. Users and
applications can access files as if they were local, but the actual
files are accessed over the network. Representative network-based
file systems include NFS (Network File System), SMB (Server Message
Block), and DFS (Distributed File System).

Key Characteristics

  • Centralized storage: all files are stored on a central server, making management easy.
  • Data sharing: multiple clients can share the same file, facilitating collaboration.
  • Scalability: storage capacity can be expanded easily.

Disk-Based File Systems

A disk-based file system stores files on the computer’s local disk.
It reads and writes data by directly accessing the physical storage
device, managed by the operating system. Representative disk-based
file systems include NTFS, FAT32, and ext4.

Key Characteristics

  • Local storage: files are stored on the local disk, providing fast access speed.
  • Security: files on the local disk can’t be accessed over the network, so security is strong.
  • Limited sharing: to share files, you must create shared folders through network configuration.

A Closer Look at Disk-Based File Systems

A disk-based file system is a structure designed to store and
retrieve data efficiently. Each file system uses its own structure
and algorithms to manage files and directories.

Major File System Types

  • NTFS (New Technology File System): the file system used by the Windows NT family, with features like security, compression, and large-file support.
  • FAT32 (File Allocation Table 32): compatible with all operating systems, mainly used for small file systems and USB drives.
  • ext4 (Extended File System version 4): the file system mainly used on Linux, with features like journaling, large file and file-system support, and improved performance.

Key Features of a File System

  1. File storage and retrieval: provides a structured data storage scheme so files can be stored and retrieved quickly when needed.
  2. Directory structure: organizes files in a hierarchical structure and manages relationships between files.
  3. Security and permission management: lets you set access permissions on files by user and group.
  4. Journaling: records changes to prevent data loss in case of a system crash or power failure.
  5. Error detection and recovery: includes mechanisms to detect and recover from file system errors.
