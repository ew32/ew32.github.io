How Spring AOP Works and How to Use It

Posted on 2026-05-07 Edited on 2026-05-26 In Backend , Java
An explanation of Spring AOP's proxy-based operation, pointcuts, advice types, and practical use cases.

Why AOP Is Needed

Aspect-Oriented Programming (AOP) is a paradigm for separating
cross-cutting concerns from business logic. Code like logging,
transactions, permission checks, caching, and performance
measurement easily gets scattered and duplicated across many
methods. AOP lets you gather this common code in one place and apply
it declaratively.

Spring provides its own lightweight, proxy-based AOP. It doesn’t
manipulate bytecode like AspectJ, but it can meet most real-world
requirements.

Key Terminology

Term Description
Aspect A modularized unit of a cross-cutting concern
Join Point A point where advice can be applied (a method execution)
Pointcut An expression that selects join points
Advice The actual auxiliary behavior performed
Target The object to which advice is applied
Proxy The object that wraps the target to apply advice

Proxy-Based Operation

Spring AOP wraps a target bean in a proxy object when registering
the bean. When the bean is called from outside, the proxy — not the
real object — is called, and the proxy runs the advice before
delegating to the target method.

  • A bean with an interface gets a JDK dynamic proxy
  • A bean without an interface gets a CGLIB proxy (inheritance-based)
  • Since Spring Boot 2.x, CGLIB is the default

A caveat with this structure is self-invocation. If you call
this.method() from within the same class, it doesn’t go through
the proxy, so advice doesn’t run.

Advice Execution Order

When multiple Aspects apply to the same join point, specify the
order with @Order.

A lower number runs on the outside. It’s common to place things
closer to the core, like transactions, on the inside.

Best Practices

  • @Transactional is the representative use of Spring AOP, so be sure to understand how it works
  • For the self-invocation problem, split the bean into a separate class or work around it with AopContext.currentProxy()
  • Make pointcuts as narrow as possible for performance and predictability
  • Minimize an Aspect’s dependencies and keep it from intruding into domain logic

Wrapping Up

Spring AOP is powerful, but you must understand its proxy-based
essence for it to work as intended. Identifying cross-cutting
concerns and clearly defining pointcuts greatly reduces code
duplication and keeps business logic clean.
