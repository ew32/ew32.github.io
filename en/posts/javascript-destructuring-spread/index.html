JavaScript Destructuring and the Spread Operator

Posted on 2026-05-19 Edited on 2026-05-26 In Frontend , JavaScript
How to use ES6 destructuring assignment and the spread operator, with practical patterns.

What Is Destructuring Assignment?

Destructuring assignment is ES6 syntax that extracts values from
arrays or objects and assigns them to separate variables at once. It
makes code far more concise and clear, and is used widely in
function parameters, return-value handling, and state updates.

Spread (...) expands an array or object into individual

Rest Parameters

Same ... as spread, but in a different position with the opposite
meaning. It gathers, not expands.

Things to Watch For

Caveat Description
Shallow copy Spread copies only one level deep
Big-object cost Creating a new object each time has a cost
Object spread is ES2018 Older environments need transpilation
undefined when destructuring If the object is null/undefined, it throws

Wrapping Up

Destructuring and the spread operator are core syntax that
determine the readability of modern JavaScript. They’re used in
everything from simple variable extraction to React state updates,
function-option merging, and array manipulation. Once you wield
both freely, your code becomes far more declarative and clear.
