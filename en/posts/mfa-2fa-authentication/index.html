A Complete Guide to MFA - TOTP, HOTP, WebAuthn

Posted on 2026-04-25 Edited on 2026-07-20 In Backend , Authentication
From the difference between 2FA and MFA to how the TOTP/HOTP algorithms work and phishing-resistant authentication with WebAuthn (Passkeys).

The Difference Between MFA and 2FA

Multi-Factor Authentication (MFA) combines two or more different
authentication factors. 2FA is a form of MFA that uses exactly
two factors. Factors fall into three categories.

  • Knowledge (something you know): password, PIN
  • Possession (something you have): phone, security key, OTP generator
  • Inherence (something you are): biometrics like fingerprint or face

Two factors from the same category (e.g., password + security
question) aren’t true MFA. You must combine different categories.

HOTP - Counter-Based OTP

HOTP (HMAC-based OTP, RFC 4226) hashes a counter value with
HMAC-SHA1 and converts it into a 6-digit number.

The server and client must share the same secret key K and
counter C. Each time the user uses an OTP, C increments. If
counter synchronization breaks, authentication fails, so a
look-ahead window is typically used.

TOTP - Time-Based OTP

TOTP (Time-based OTP, RFC 6238) replaces HOTP’s counter with time.

The defaults are T0 = 0 and X = 30 seconds. A new code is
generated every 30 seconds, and a ±1 window is usually allowed for
clock drift.

The Limits of TOTP

TOTP is convenient but has these weaknesses.

  • Phishing-vulnerable: if the user enters the 6-digit code on a fake site, the attacker can reuse it within 30 seconds
  • Shared secret key: if the server DB leaks, every user’s OTP can be forged
  • Clock-sync dependent: fails if the client clock drifts significantly

WebAuthn emerged to overcome these limits.

WebAuthn and Passkeys

WebAuthn (a W3C standard) is a public-key-cryptography-based
authentication standard. The client generates a key pair and
registers only the public key with the server. To authenticate, the
client signs a challenge sent by the server with its private key.

WebAuthn’s key advantage is phishing resistance. The browser
binds the key to the domain (rp.id), so authentication won’t work
on a fake site.

Passkey standardizes the user experience of syncing WebAuthn
credentials across a user’s devices, supported by Apple iCloud
Keychain, Google Password Manager, and others.

Which Method to Choose

  • SMS OTP: minimal security. Bypassable via SIM-swapping attacks — avoid if possible.
  • TOTP: free, simple, widespread. A suitable default for general users.
  • Push notification: good UX, but watch out for MFA-fatigue attacks.
  • WebAuthn/Passkey: the strongest and phishing-resistant. Recommended as the default for new services.

Wrapping Up

Passwords alone are no longer safe. Adopt at least TOTP, and where
possible WebAuthn, to secure a defense line against phishing and
credential theft. MFA is the security investment with the highest
return relative to its cost.
