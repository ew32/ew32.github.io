Vim Basics: Modes, Movement, and Editing

Posted on 2026-05-28 Edited on 2026-06-15 In DevEnvironment
Vim's mode concept and basic movement, editing, and visual mode usage, organized from a beginner's perspective.

Why Vim?

Vim is a modal text editor that appeared in 1991, preinstalled
on almost every Unix-like system. It’s designed to let you edit text
efficiently with the keyboard alone, no mouse — and once you get
used to it, it’s so addictive you’ll seek out Vim key bindings in
other editors too.

It’s the standard tool of the terminal environment — for remote
server work, Git commit messages, editing config files, and more —
so it’s good for any developer to learn the fundamentals.

Understanding Modes

Vim’s biggest characteristic is modes. The meaning of a key
changes by mode.

Mode Entry key Purpose
Normal Esc Movement, running commands (the default mode)
Insert i, a, o Entering text
Visual v, V, Ctrl+v Selecting a region
Command : Entering ex commands
Replace R Overwriting

The biggest difference from other editors is that Normal mode is
the default. To type, you must explicitly enter Insert mode.

Basic Movement

In Normal mode, use hjkl instead of the arrow keys.

Command Action
w To the start of the next word
b To the start of the previous word
e To the end of the word
0 To the start of the line
^ To the first non-blank character of the line
$ To the end of the line
gg To the start of the file
G To the end of the file
42G To line 42
Ctrl+d Half a page down
Ctrl+u Half a page up

Entering Insert Mode

Command Action
i Insert at the cursor position
I Insert at the start of the line
a Insert after the cursor
A Insert at the end of the line
o Create a new line below and insert
O Create a new line above and insert

When done inserting, always return to Normal mode with Esc.

Editing Commands

The power of Vim editing comes from the operator + motion
combination.

Operator Meaning
d Delete
c Change (delete then insert)
y Yank (copy)
p Paste

You combine these with a motion.

Text objects enable more precise target selection.

Object Meaning
iw inner word
aw a word (including whitespace)
i" inside quotes
a" including quotes
ip inside a paragraph
i{ inside a brace block

For example, ci" erases everything inside the quotes and enters
insert mode.

Undo and Redo

Command Action
u undo
Ctrl+r redo
. repeat the last command

The . command is where Vim’s true value shows — it lets you
quickly apply the same edit at multiple locations.

Visual Mode

If you’re used to selecting and then editing, use Visual mode.

Command Action
v Character-wise selection
V Line-wise selection
Ctrl+v Block (rectangular) selection

After selecting, you can apply operators like d, y, and c.
Block mode is useful for inserting the same text across multiple
lines.

Learning Tips

  • Run the official 30-minute tutorial with the vimtutor command
  • For the first week, deliberately cut down on using other editors and use only Vim
  • Forbidding the arrow keys makes hjkl quickly become second nature
  • Don’t memorize every command at once; add 3–5 each week

Wrapping Up

Vim’s learning curve is steep, but once you understand the
consistent grammar of mode + operator + motion, you get used to
it surprisingly fast. Installing a Vim key-binding plugin in your
familiar IDE to adopt it gradually is also a good approach. Once it’s
in your hands, it’s a lifelong weapon.
