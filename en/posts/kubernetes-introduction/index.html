An Introduction to Kubernetes - Core Concepts

Posted on 2026-05-25 Edited on 2026-07-20 In Cloud , Infrastructure
The core concepts and operating principles of Kubernetes, the standard for container orchestration.

Introduction

Containerization with Docker is now standard, but managing dozens to
hundreds of containers in production requires a separate tool.
Kubernetes (K8s) is the de facto standard for container
orchestration, automating container deployment, scaling, recovery,
and networking. This post lays out Kubernetes’s core components and
concepts.

What Is Kubernetes?

A container management platform Google open-sourced in 2014. Its key
features:

  • Automatic deployment and rollback: maintain the desired state via declarative configuration
  • Self-healing: automatically restart failed containers
  • Horizontal scaling: add containers automatically when traffic rises
  • Service discovery and load balancing
  • Secret and configuration management

Cluster Architecture

Kubernetes consists of a Control Plane and Worker Nodes.

Control Plane Components

  • kube-apiserver: the entry point for all requests, provides a REST API
  • etcd: a distributed key-value store holding cluster state
  • kube-scheduler: decides which node a new pod goes on
  • kube-controller-manager: runs various controllers (Deployment, ReplicaSet, etc.)

Worker Node Components

  • kubelet: runs pods on the node and reports status
  • kube-proxy: manages the node’s network routing
  • Container runtime: containerd, CRI-O, etc.

Core Objects

Pod

The smallest deployable unit in Kubernetes — one or more
containers bundled and run together. Containers in the same pod
share network and storage.

Deployment

Provides declarative pod management, rolling updates, and rollbacks.
In practice you use a Deployment rather than creating pods directly.

Service

Pods are dynamically created/destroyed and their IPs change. A
Service provides a stable entry point to a group of pods.

Service types:

Type Description
ClusterIP An internal cluster IP (default)
NodePort Exposed externally on a specific port of the nodes
LoadBalancer Connects to a cloud load balancer
ExternalName Maps to an external DNS name

ConfigMap & Secret

A way to separate configuration and secret information from the
container image.

Ingress

Routes HTTP/HTTPS traffic to services inside the cluster.

Frequently Used kubectl Commands

Declarative vs Imperative

Kubernetes’s core philosophy is declarative management. You
describe the “desired state” in YAML, and Kubernetes converges the
actual state toward it.

  • Imperative: kubectl run nginx --image=nginx
  • Declarative: kubectl apply -f nginx.yaml

In practice, the standard is the declarative approach with version
control in Git (GitOps).

Concepts Worth Knowing for Operations

  • Namespace: a virtual partition within the cluster
  • HPA (Horizontal Pod Autoscaler): auto-scale based on CPU/memory
  • Liveness/Readiness Probe: health checks
  • Resource Requests/Limits: resource guarantees and caps
  • PV/PVC: persistent storage management

Wrapping Up

Kubernetes has a steep learning curve, but once learned, it’s a
powerful weapon for container operations. Learn the basic objects —
Pod, Deployment, Service, Ingress — and the declarative philosophy
first, then expand into health checks, autoscaling, storage, and
security. Managed services like AWS EKS, GCP GKE, and Azure AKS also
reduce the burden of managing the control plane.
