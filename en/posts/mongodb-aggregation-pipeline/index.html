A Complete Guide to the MongoDB Aggregation Pipeline

Posted on 2026-04-19 Edited on 2026-06-05 In Backend , Database
A comprehensive look at the key stages of MongoDB's powerful data-analysis tool - the Aggregation Pipeline - with practical examples and performance tips.

What Is the Aggregation Pipeline?

MongoDB’s Aggregation Pipeline is a powerful data-processing
framework that transforms and aggregates documents through multiple
stages. Each stage takes the previous stage’s output as input, and
the result lets you write queries far more expressive than SQL’s
SELECT ... GROUP BY ... HAVING ... ORDER BY.

This one statement sums paid orders per user and returns the top 10.

Key Stages

$match

The most frequently used stage. It works like an ordinary query
condition. Placing it as early as possible in the pipeline helps
performance.

$project

Selects only the fields you need or computes new fields.

$group

The equivalent of SQL’s GROUP BY. Define the group key with _id
and apply various aggregation operators.

$sort

Sorting. Very fast if it can use an index.

$limit / $skip

Pagination. Note that $skip is slow at large values — consider
keyset pagination.

$lookup (join)

Performs a left outer join with another collection.

There’s also an extended syntax for multi-condition joins.

$unwind

Unwinds an array field into multiple rows.

$addFields / $set

Adds fields to the existing document. Unlike $project, it doesn’t
remove other fields.

$facet

Runs multiple pipelines in parallel over the same data. Useful for
getting “total count + top 10 + per-category stats” of a search
result in one call.

$bucket / $bucketAuto

Grouping by range. Good for histograms.

$out / $merge

Saves pipeline results into a new collection or upserts into an
existing one. Useful for ETL and materialized views.

Performance Tips

  1. Place $match and $sort early to reduce the data volume
  2. Create an index that $sort can use
  3. Keep only needed fields with $project to cut memory usage
  4. Index the $lookup target collection
  5. If a sort needs more than 100MB of memory, use allowDiskUse: true
  6. For complex pipelines, check per-stage cost with explain()

Aggregation Framework vs Map-Reduce

Map-Reduce used to be common, but the Aggregation Pipeline is much
faster and more expressive, so new code almost always chooses
Aggregation.

Commonly Used Expressions

  • Arithmetic: $add, $subtract, $multiply, $divide, $mod
  • Comparison: $eq, $ne, $gt, $gte, $lt, $lte
  • Logical: $and, $or, $not
  • Conditional: $cond, $switch, $ifNull
  • Array: $size, $map, $filter, $reduce, $arrayElemAt
  • Date: $dateToString, $dateAdd, $year, $month, $dayOfMonth
  • String: $concat, $toLower, $substr, $regexMatch

Wrapping Up

The MongoDB Aggregation Pipeline enables powerful data analysis and
transformation beyond simple lookups. Once you get used to thinking
of each stage as a small function you compose to refine results
step by step, you can do plenty of ETL and reporting work right
inside MongoDB without a separate analytics system.
