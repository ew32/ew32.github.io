AWS Lambda 서버리스 입문

작성일 2026-05-04 Edited on 2026-05-20 In Cloud , Infrastructure
AWS Lambda의 핵심 개념과 첫 함수를 배포하는 방법, 그리고 실무에서 주의할 점을 정리합니다.

서버리스(Serverless)란?

서버리스는 “서버가 없다”는 의미가 아니라, 개발자가 서버를 직접 관리하지 않아도 되는 컴퓨팅 모델을 말합니다. 클라우드 제공자가 서버의 프로비저닝, 확장, 패치 관리를 모두 처리하며, 개발자는 비즈니스 로직 작성에만 집중할 수 있습니다.

AWS Lambda란?

AWS Lambda는 아마존이 제공하는 대표적인 FaaS(Function as a Service) 입니다. 코드를 함수 단위로 업로드하면, 이벤트 발생 시 자동으로 실행되며, 실행한 시간만큼만 비용을 지불합니다.

주요 특징

  • 자동 확장 (초당 수천 개의 요청 처리 가능)
  • 사용한 만큼만 과금
  • 다양한 런타임 지원 (Node.js, Python, Java, Go, Ruby, .NET 등)
  • 다양한 AWS 서비스와 통합

실무에서 주의할 점

1. 콜드 스타트(Cold Start)

함수가 처음 실행되거나 일정 시간 사용되지 않다가 다시 실행될 때 발생하는 초기화 지연입니다.

완화 방법:

  • Provisioned Concurrency 사용
  • 패키지 크기 최소화
  • 무거운 초기화는 핸들러 외부로 이동
  • 컴파일 언어(Java)보다 인터프리터 언어(Python, Node.js) 선호

2. 실행 시간 제한

Lambda의 최대 실행 시간은 15분입니다. 장시간 작업은 Step Functions나 ECS를 고려하세요.

3. 메모리와 비용

메모리 설정은 CPU 성능과 비례합니다. 적절한 메모리 설정이 오히려 비용을 절감할 수 있습니다.

4. 동시성 제한

계정당 기본 동시 실행 한도가 있으므로(기본 1,000), 트래픽 폭증 시 Throttling이 발생할 수 있습니다.

5. 상태 비저장(Stateless)

함수는 상태를 유지하지 않으므로, 상태가 필요하면 DynamoDB, Redis 등 외부 저장소를 사용해야 합니다.

비용 구조

Lambda 요금은 다음 두 가지로 구성됩니다.

  • 요청 수: 100만 건당 $0.20
  • 실행 시간: GB-초 단위 (메모리 × 시간)

월 100만 요청, 평균 200ms 실행, 512MB 메모리 기준 약 $0.20 + 약 $1.67 = 약 $1.87 정도입니다(프리 티어 미적용 시).

Lambda가 적합한 경우

  • 비동기 이벤트 처리 (이미지 리사이징, 알림 전송 등)
  • 가벼운 API 백엔드
  • 예측 불가능한 트래픽 패턴
  • ETL 및 데이터 파이프라인

Lambda가 부적합한 경우

  • 장시간 실행 작업 (15분 초과)
  • 일정한 고부하 트래픽 (EC2/ECS가 더 경제적)
  • 낮은 지연 시간이 필수인 작업 (콜드 스타트)
  • 대용량 파일을 메모리에서 처리하는 작업

마무리

Lambda는 서버 관리의 부담을 덜어주고 빠른 프로토타이핑을 가능하게 해주는 강력한 도구입니다. 다만 모든 워크로드에 맞는 만능 도구는 아니므로, 특성을 잘 이해하고 적합한 곳에 사용하는 것이 중요합니다.
