tmux 활용 - 터미널 멀티플렉서 입문

작성일 2026-05-28 Edited on 2026-06-15 In 개발환경
tmux의 세션/윈도우/페인 개념과 자주 쓰는 단축키, 설정 커스터마이즈를 소개합니다.

tmux란?

**tmux(Terminal Multiplexer)**는 하나의 터미널 안에서 여러 개의 가상 터미널 세션을 실행하고 관리할 수 있게 해주는 도구입니다. SSH 접속이 끊겨도 세션이 살아있고, 화면을 분할해서 여러 작업을 동시에 진행할 수 있어 서버 운영자와 개발자 모두에게 필수 도구로 꼽힙니다.

주요 사용 이유는 다음과 같습니다.

  • 세션 지속: SSH 끊김에도 작업이 계속 진행됨
  • 화면 분할: 한 화면에서 로그/에디터/REPL 동시 보기
  • 세션 공유: 여러 사용자가 같은 세션 접속 (페어 프로그래밍)
  • 워크플로 자동화: 스크립트로 개발 환경 즉시 구성

설치

설치 후 tmux만 입력하면 새 세션이 시작됩니다.

핵심 개념

tmux는 세 가지 계층 구조로 동작합니다.

  • Session: 가장 큰 단위. 작업 단위로 분리
  • Window: 세션 내의 탭
  • Pane: 윈도우를 분할한 각 영역

프리픽스 키

tmux의 모든 명령은 프리픽스(Prefix) 키를 누른 뒤 명령 키를 누릅니다. 기본 프리픽스는 Ctrl+b입니다.

이 글에서는 <prefix>로 표기합니다.

세션 관리

세션 내부에서 사용하는 명령입니다.

명령 동작
<prefix> d 세션에서 떼어내기 (detach)
<prefix> s 세션 목록
<prefix> $ 세션 이름 변경

detach 후 attach 패턴이 tmux의 핵심 워크플로입니다. SSH 연결이 끊겨도 안에서 돌던 프로세스는 계속 동작합니다.

윈도우 관리

명령 동작
<prefix> c 새 윈도우 생성
<prefix> n 다음 윈도우
<prefix> p 이전 윈도우
<prefix> 0~9 번호로 이동
<prefix> , 윈도우 이름 변경
<prefix> & 윈도우 닫기
<prefix> w 윈도우 목록

페인 분할과 이동

명령 동작
<prefix> % 세로(왼쪽/오른쪽) 분할
<prefix> " 가로(위/아래) 분할
<prefix> 화살표 페인 간 이동
<prefix> o 다음 페인으로
<prefix> z 페인 확대/축소 토글
<prefix> x 페인 닫기
<prefix> { / } 페인 위치 교환
<prefix> Space 레이아웃 순환 변경

<prefix> z로 페인을 일시적으로 전체화면으로 확대하는 기능은 자주 쓰입니다.

복사 모드

마우스 없이 화면을 스크롤하고 텍스트를 복사하려면 복사 모드를 사용합니다.

vi 키바인딩을 활성화하려면 설정 파일에 setw -g mode-keys vi를 추가합니다.

설정 파일 (.tmux.conf)

홈 디렉터리의 ~/.tmux.conf에서 동작을 커스터마이즈할 수 있습니다.

설정 변경 후 <prefix> r 또는 tmux source ~/.tmux.conf로 반영합니다.

워크플로 예시

여러 페인으로 개발 환경을 구성하는 예시입니다.

이런 스크립트를 통해 매번 동일한 개발 환경을 즉시 복원할 수 있습니다. tmuxinator, tmuxp 같은 도구를 쓰면 YAML로 더 깔끔하게 관리됩니다.

추천 플러그인

**TPM(Tmux Plugin Manager)**으로 플러그인을 관리할 수 있습니다.

  • tmux-resurrect: 세션 상태 저장/복원
  • tmux-continuum: 자동 저장
  • tmux-yank: 시스템 클립보드 연동

마무리

tmux는 처음에는 단축키가 낯설지만, 며칠만 사용해보면 터미널 작업의 표준 환경이 됩니다. detach/attach로 세션을 보존하고, 페인 분할로 한눈에 여러 작업을 보며, .tmux.conf로 자기에게 맞게 다듬는 것이 출발점입니다. SSH 서버를 자주 다룬다면 가장 우선 익혀야 할 도구입니다.
