JavaScript 함수형 프로그래밍 입문

작성일 2026-06-03 Edited on 2026-06-15 In Frontend , JavaScript
순수 함수, 불변성, 고차 함수, 합성, 커링 등 JavaScript에서 함수형 패러다임을 적용하는 방법을 정리합니다.

함수형 프로그래밍이란

함수형 프로그래밍(FP)은 부작용(side effect)을 최소화하고 함수의 합성으로 문제를 해결하는 패러다임입니다. JavaScript는 일급 함수, 클로저, 화살표 함수, 스프레드 연산자 등 FP 친화적 문법을 충실히 갖추고 있습니다.

순수 함수

같은 입력에 대해 항상 같은 출력을 반환하고, 외부 상태를 변경하지 않는 함수입니다.

순수 함수는 테스트와 추론이 쉽고, 캐싱(memoization)과 병렬화에 유리합니다.

불변성

데이터를 직접 변경하지 않고 새 데이터를 만들어 반환합니다.

배열에서는 map, filter, slice, concat이 불변, push, pop, splice, sort는 변이입니다. 최신 JS의 toSorted, toReversed, with, toSpliced는 불변 버전을 제공합니다.

고차 함수

함수를 인자로 받거나 반환하는 함수입니다.

map, filter, reduce는 FP의 3대 도구입니다.

함수 합성

작은 함수를 조합해 큰 함수를 만듭니다.

커링과 부분 적용

여러 인자를 받는 함수를 인자 하나씩 받는 함수들의 체인으로 변환합니다.

클로저와 메모이제이션

부작용 격리

순수 함수만으로 프로그램을 만들 수는 없습니다(I/O가 필요). FP에서는 부작용을 가장자리(edge)로 밀어내고 핵심 로직은 순수하게 유지합니다.

라이브러리

  • Ramda: 자동 커링, 데이터 마지막 인자 컨벤션
  • Lodash/fp: lodash의 함수형 변형
  • Immer: 불변 업데이트를 가변처럼 작성

마무리

함수형 프로그래밍은 흑백 논리가 아닙니다. 모든 코드를 순수하게 만들 필요는 없고, “쉽게 추론할 수 있는 단위로 쪼개고 조합한다”는 사고방식을 가져오는 것만으로도 코드 품질이 크게 향상됩니다.
