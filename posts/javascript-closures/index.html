JavaScript 클로저(Closure) 완벽 이해

작성일 2026-05-17 Edited on 2026-05-20 In Frontend , JavaScript
JavaScript 클로저의 개념과 동작 원리, 실전 활용 패턴을 정리합니다.

클로저란?

클로저(Closure) 는 함수가 선언된 렉시컬 환경(lexical environment)을 기억하여, 함수가 외부에서 호출되더라도 해당 환경의 변수에 접근할 수 있는 기능을 말합니다. JavaScript에서 클로저는 함수가 일급 객체(first-class object)라는 특성과 렉시컬 스코프 규칙이 결합되어 자연스럽게 발생하는 현상입니다.

MDN에서는 클로저를 “함수와 그 함수가 선언된 어휘적 환경의 조합”이라고 정의합니다. 즉, 내부 함수가 외부 함수의 지역 변수를 참조할 때, 그 외부 함수가 종료된 이후에도 해당 변수가 메모리에 유지됩니다.

클로저의 기본 동작

다음 예시는 가장 단순한 클로저의 예입니다.

outer 함수는 이미 실행을 끝냈지만, inner 함수는 여전히 message 변수를 참조합니다. 이는 inner가 outer의 렉시컬 환경을 참조하는 [[Environment]] 내부 슬롯을 보유하기 때문입니다.

렉시컬 스코프와의 관계

JavaScript는 렉시컬 스코프(lexical scope) 를 사용합니다. 함수가 어디서 호출되었는지가 아니라 어디서 선언되었는지에 따라 스코프가 결정됩니다.

클로저는 이 렉시컬 스코프 체인을 그대로 유지하는 메커니즘으로 이해할 수 있습니다.

활용 패턴 1: 데이터 은닉

클로저를 사용하면 외부에서 직접 접근할 수 없는 private 변수를 흉내낼 수 있습니다.

count 변수는 외부에서 결코 접근할 수 없으며, 오직 반환된 메서드들을 통해서만 제어 가능합니다.

활용 패턴 2: 함수 팩토리

비슷한 동작을 하는 함수를 동적으로 생성할 때 유용합니다.

활용 패턴 3: 반복문에서의 클로저 문제

var로 선언한 반복 변수는 클로저와 결합 시 의도와 다르게 동작하는 경우가 많습니다.

var는 함수 스코프이므로 모든 콜백이 동일한 i를 참조합니다. 해결책은 let 사용 또는 IIFE입니다.

let은 블록 스코프이며 매 반복마다 새 바인딩을 생성하기 때문입니다.

메모리 관점에서의 주의

클로저가 참조하는 변수는 가비지 컬렉션 대상에서 제외됩니다. 따라서 불필요하게 큰 객체를 클로저가 잡고 있으면 메모리 누수의 원인이 될 수 있습니다.

상황 영향
DOM 요소를 클로저가 참조 페이지에서 제거해도 GC되지 않음
큰 배열/객체를 외부 함수가 보유 내부 함수가 살아있는 한 메모리 점유
이벤트 리스너 미해제 누적되며 누수 발생

해결책은 더 이상 필요하지 않은 참조를 null로 명시적으로 끊거나, 이벤트 리스너를 removeEventListener로 해제하는 것입니다.

마무리

클로저는 JavaScript의 가장 강력하고 동시에 가장 헷갈리기 쉬운 개념 중 하나입니다. 렉시컬 스코프와 함수 일급 객체 특성을 이해하면 자연스럽게 받아들일 수 있으며, 데이터 은닉, 함수 팩토리, 모듈 패턴, 커링 등 다양한 곳에서 활용됩니다. 메모리 누수를 일으키지 않도록 참조 관계를 명확히 설계하는 습관을 들이는 것이 중요합니다.
