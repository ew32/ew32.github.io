Spring Boot 시작하기

작성일 2026-05-07 Edited on 2026-05-18 In Backend , Java
Spring Boot의 핵심 개념과 프로젝트 생성부터 첫 REST API 작성까지의 과정을 소개합니다.

Spring Boot란 무엇인가

Spring Boot는 Spring Framework 기반 애플리케이션을 빠르게 구축할 수 있도록 도와주는 프로젝트입니다. 복잡한 XML 설정과 의존성 관리의 부담을 줄이고, 내장 톰캣(Tomcat) 서버를 통해 별도의 WAS 없이도 실행할 수 있는 독립 실행형 애플리케이션을 만들 수 있습니다.

Spring Boot가 제공하는 주요 가치는 다음과 같습니다.

  • 자동 설정(Auto Configuration): classpath에 있는 라이브러리를 분석해 필요한 빈을 자동 등록
  • Starter 의존성: spring-boot-starter-web처럼 묶음 의존성을 통해 호환되는 버전 일괄 관리
  • 내장 서버: Tomcat, Jetty, Undertow를 내장하여 java -jar로 실행 가능
  • Actuator: 헬스 체크, 메트릭, 환경 변수 등 운영 정보를 즉시 제공

프로젝트 생성

가장 손쉬운 방법은 Spring Initializr를 사용하는 것입니다. Maven 또는 Gradle 빌드 도구를 선택하고, 필요한 의존성을 추가하면 zip 파일로 다운로드할 수 있습니다.

애플리케이션 진입점

Spring Boot 애플리케이션은 @SpringBootApplication 어노테이션이 붙은 메인 클래스에서 시작합니다.

@SpringBootApplication은 다음 세 가지 어노테이션을 합친 것입니다.

어노테이션 역할
@SpringBootConfiguration 설정 클래스로 표시
@EnableAutoConfiguration 자동 설정 활성화
@ComponentScan 패키지 하위 컴포넌트 스캔

첫 REST API 작성

간단한 컨트롤러를 만들어 GET 요청을 처리해봅시다.

@RestController는 @Controller와 @ResponseBody가 결합된 형태로, 응답을 JSON으로 직렬화합니다. 실행 후 http://localhost:8080/api/hello?name=Spring을 호출하면 JSON 응답을 받을 수 있습니다.

설정 파일 다루기

Spring Boot는 application.yml 또는 application.properties 파일로 설정을 관리합니다.

프로필별로 application-dev.yml, application-prod.yml을 두고 SPRING_PROFILES_ACTIVE=prod로 전환할 수 있습니다.

자동 설정의 동작 원리

Spring Boot는 시작 시 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports 파일을 읽어 자동 설정 후보 클래스를 로딩합니다. 각 설정 클래스는 @ConditionalOnClass, @ConditionalOnMissingBean 같은 조건 어노테이션으로 활성화 여부를 결정합니다.

예를 들어 DataSourceAutoConfiguration은 H2나 HikariCP가 classpath에 있을 때만 동작합니다. 어떤 자동 설정이 적용되었는지 확인하려면 --debug 옵션으로 실행해 보세요.

모범 사례

  • 패키지 구조: 메인 클래스가 위치한 패키지를 최상위로 두고 하위에 controller, service, repository를 둡니다.
  • 의존성 관리: BOM(Bill of Materials)을 활용해 라이브러리 버전을 일치시킵니다.
  • 로컬 개발 편의: spring-boot-devtools를 추가하면 코드 변경 시 자동 재시작이 가능합니다.
  • 테스트: @SpringBootTest로 통합 테스트, @WebMvcTest로 슬라이스 테스트를 분리합니다.

마무리

Spring Boot는 단순히 Spring을 쉽게 사용하기 위한 도구가 아니라, 합리적인 기본값(opinionated defaults) 을 제공하여 빠른 프로토타이핑과 생산 환경 배포를 모두 가능하게 합니다. 다음 글에서는 Spring Security, JPA, AOP 등 핵심 모듈을 차례로 살펴보겠습니다.
