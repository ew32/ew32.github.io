Docker 컨테이너 기초

작성일 2026-05-01 Edited on 2026-05-19 In Backend , Infrastructure
Docker의 핵심 개념인 이미지와 컨테이너를 이해하고 자주 사용하는 명령어를 정리합니다.

Docker란?

Docker는 애플리케이션을 컨테이너(Container) 라는 격리된 환경에서 실행할 수 있도록 해주는 오픈소스 가상화 플랫폼입니다. 기존의 가상 머신(VM)이 하드웨어 수준의 가상화를 수행한다면, Docker는 운영체제 수준의 가상화를 통해 훨씬 가볍고 빠른 실행 환경을 제공합니다.

VM과 컨테이너의 차이

항목 가상 머신 (VM) 컨테이너
가상화 수준 하드웨어 수준 OS 수준
부팅 속도 분 단위 초 단위
리소스 사용량 무거움 가벼움
격리 수준 강함 보통
게스트 OS 필요 불필요 (호스트 커널 공유)

핵심 개념

이미지(Image)

컨테이너를 실행하기 위한 읽기 전용 템플릿입니다. 애플리케이션 코드, 런타임, 시스템 라이브러리, 설정 파일 등이 모두 포함됩니다. 이미지는 여러 레이어로 구성되며, 동일한 레이어는 재사용되어 저장 공간을 절약합니다.

컨테이너(Container)

이미지를 실행한 인스턴스입니다. 동일한 이미지로 여러 개의 컨테이너를 실행할 수 있으며, 각 컨테이너는 독립된 파일 시스템과 네트워크를 가집니다.

Dockerfile

이미지를 빌드하기 위한 명령어 모음 텍스트 파일입니다.

자주 사용하는 명령어

Docker Compose

여러 컨테이너로 구성된 애플리케이션을 한 번에 정의하고 실행할 수 있는 도구입니다.

docker-compose up -d 명령으로 전체 서비스를 백그라운드에서 실행할 수 있습니다.

마무리

Docker를 활용하면 “내 컴퓨터에서는 잘 되는데…” 라는 문제를 근본적으로 해결할 수 있습니다. 개발, 테스트, 운영 환경의 일관성을 유지하고, CI/CD 파이프라인 구성에도 필수적인 도구이므로 백엔드 개발자라면 반드시 익혀두는 것이 좋습니다.
