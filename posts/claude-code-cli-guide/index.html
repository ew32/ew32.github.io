Claude Code CLI 활용 가이드

작성일 2026-07-08 Edited on 2026-06-05 In AI도구
터미널 기반 AI 코딩 어시스턴트 Claude Code CLI의 설치, 핵심 명령어, 워크플로 자동화 방법을 정리합니다.

Claude Code CLI란?

Claude Code는 Anthropic이 제공하는 터미널 기반 코딩 어시스턴트입니다. IDE 플러그인이 아니라 셸에서 직접 동작하기 때문에 기존의 git, make, docker 같은 CLI 도구와 자연스럽게 결합할 수 있습니다. 파일 시스템을 직접 읽고 쓰며, 명령을 실행하고, 변경사항을 커밋까지 자동화할 수 있다는 점이 핵심 차별점입니다.

설치와 초기 설정

Node.js 18 이상이 필요합니다.

처음 실행하면 API 키 인증이 진행됩니다. 프로젝트 루트에서 claude 명령으로 세션을 시작하면 해당 디렉터리가 작업 컨텍스트가 됩니다.

CLAUDE.md로 컨텍스트 주입

프로젝트 루트에 CLAUDE.md 파일을 만들면 매 세션 시작 시 자동으로 읽힙니다. 빌드 명령, 코드 스타일, 폴더 구조 등을 적어두면 매번 같은 설명을 반복하지 않아도 됩니다.

자주 쓰는 명령

세션 내에서 사용 가능한 슬래시 명령은 워크플로 단축에 매우 유용합니다.

  • /init — CLAUDE.md 초안 자동 생성
  • /review — 현재 변경사항 코드 리뷰
  • /clear — 컨텍스트 초기화
  • /cost — 토큰 사용량 확인

쉘에서 직접 일회성 명령도 보낼 수 있습니다.

헤드리스 모드와 자동화

-p(print) 옵션과 --output-format json을 조합하면 스크립트나 CI에서 비대화형으로 사용할 수 있습니다.

GitHub Actions에서 PR이 열릴 때마다 자동 리뷰를 돌리거나, pre-push 훅에서 변경 파일에 대한 테스트 작성을 위임하는 식의 통합이 가능합니다.

권한 모델

Claude Code는 파일 수정과 명령 실행에 대해 매번 사용자 확인을 요청합니다. 신뢰할 수 있는 명령은 .claude/settings.json의 permissions.allow에 추가해 프롬프트 횟수를 줄일 수 있습니다.

효과적으로 쓰는 팁

  • 작은 단위로 시키기: “전체 리팩터링”보다 “이 파일에서 useState를 useReducer로”가 결과가 좋습니다.
  • 테스트 우선: 실패하는 테스트를 먼저 만들고 통과시키도록 시키면 검증 가능한 결과가 나옵니다.
  • 계획 단계 분리: 복잡한 작업은 먼저 계획을 출력하게 한 뒤 승인 후 실행시킵니다.
  • 컨텍스트 관리: 긴 세션에서는 /clear로 주기적으로 리셋해 토큰 낭비를 막습니다.

마무리

Claude Code CLI는 단순한 챗봇이 아니라 셸과 결합된 개발 에이전트입니다. CLAUDE.md, 슬래시 명령, 헤드리스 모드를 잘 활용하면 반복 작업을 크게 줄이고 코드 품질도 끌어올릴 수 있습니다.
