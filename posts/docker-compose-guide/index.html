Docker Compose 완벽 가이드

작성일 2026-06-26 Edited on 2026-05-19 In Cloud , Infrastructure
여러 컨테이너로 구성된 애플리케이션을 한 번에 정의하고 실행하는 Docker Compose 사용법과 실전 패턴을 정리합니다.

Docker Compose란?

Docker Compose는 여러 개의 컨테이너로 구성된 애플리케이션을 하나의 YAML 파일로 정의하고, 단일 명령으로 빌드·실행·중단할 수 있게 해주는 도구입니다. 웹 서버, 데이터베이스, 캐시 서버 같은 다중 컴포넌트 환경을 개발·테스트 단계에서 손쉽게 재현할 수 있습니다.

docker run 명령을 수십 줄 작성하지 않아도, docker compose up 한 줄로 모든 서비스가 일관된 네트워크와 볼륨 위에서 동작합니다.

기본 파일 구조

compose.yaml(또는 docker-compose.yml)을 프로젝트 루트에 둡니다. 최신 Compose Spec에서는 version 키가 더 이상 필수가 아닙니다.

핵심 명령어

docker-compose(하이픈) 형태는 v1 레거시이며, 현재는 Docker CLI에 통합된 docker compose(공백) 사용을 권장합니다.

네트워크와 의존성

Compose는 프로젝트별로 기본 브리지 네트워크를 만들고, 각 서비스 이름을 DNS로 등록합니다. 위 예시에서 web 컨테이너는 db:5432로 PostgreSQL에 접근할 수 있습니다.

depends_on은 기동 순서만 보장할 뿐 헬스 체크까지는 수행하지 않습니다. 데이터베이스가 실제로 준비됐는지 확인하려면 healthcheck와 condition: service_healthy를 함께 사용해야 합니다.

환경별 설정 분리

compose.yaml을 베이스로 두고 compose.override.yaml이나 compose.prod.yaml로 오버라이드하면 개발과 운영 환경을 분리할 수 있습니다.

비밀 값은 .env 파일이나 Docker Secrets로 분리하고, 이미지에는 절대 포함하지 않습니다.

실무 팁

  • 볼륨 바인드 마운트(./src:/app)는 개발용으로만 사용하고, 운영에서는 빌드된 이미지를 그대로 사용합니다.
  • restart: unless-stopped로 비정상 종료 시 자동 복구를 활성화합니다.
  • profiles를 사용하면 같은 파일에서 monitoring, dev-tools 등 선택적 서비스를 정의할 수 있습니다.
  • docker compose config로 최종 머지된 설정을 확인하면 디버깅이 쉬워집니다.

마무리

Docker Compose는 단순한 개발 편의 도구를 넘어, 마이크로서비스 환경의 로컬 재현·통합 테스트·CI 파이프라인의 표준 도구로 자리 잡았습니다. 헬스 체크와 환경 분리 패턴을 익혀두면 그대로 Kubernetes 매니페스트로 확장하기도 수월합니다.
