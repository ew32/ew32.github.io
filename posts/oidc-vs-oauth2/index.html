OIDCмҷҖ OAuth2мқҳ м°Ёмқҙ - ID Tokenкіј Access Token

мһ‘м„ұмқј 2026-04-23 Edited on 2026-05-18 In Backend , Authentication
OAuth2лҠ” мқёк°Җ н”„лЎңнҶ мҪңмқҙкі  OIDCлҠ” мқёмҰқ кі„мёөмһ…лӢҲлӢӨ. ID Tokenкіј Access Tokenмқҳ м°Ёмқҙ, мӮ¬мҡ© мӢңм җ, кІҖмҰқ л°©лІ•мқ„ л№„көҗ м •лҰ¬н•©лӢҲлӢӨ.

OAuth2мҷҖ OIDCмқҳ ліём§Ҳм Ғ м°Ёмқҙ

OAuth2лҠ” мқёк°Җ(Authorization) н”„лЎңнҶ мҪңмһ…лӢҲлӢӨ. вҖңмқҙ нҒҙлқјмқҙм–ёнҠёк°Җ мӮ¬мҡ©мһҗмқҳ лҰ¬мҶҢмҠӨм—җ м ‘к·јн•  мҲҳ мһҲлҠ”к°Җ?вҖқлқјлҠ” м§Ҳл¬ём—җ лӢөн•©лӢҲлӢӨ. л°ҳл©ҙ OpenID Connect(OIDC)лҠ” OAuth2 мң„м—җ кө¬м¶•лҗң мқёмҰқ(Authentication) кі„мёөмңјлЎң, вҖңмқҙ мӮ¬мҡ©мһҗк°Җ лҲ„кө¬мқёк°Җ?вҖқлҘј н‘ңмӨҖнҷ”лҗң л°©мӢқмңјлЎң м•Ңл ӨмӨҚлӢҲлӢӨ.

OAuth2л§Ң мӮ¬мҡ©н•ҳл©ҙ м•Ўм„ёмҠӨ нҶ нҒ°мқ„ л°ӣм§Җл§Ң к·ё нҶ нҒ°мқҳ мҶҢмң мһҗк°Җ лҲ„кө¬мқём§ҖлҠ” лӘ…м„ём—җ м •мқҳлҗҳм–ҙ мһҲм§Җ м•ҠмҠөлӢҲлӢӨ. OIDCлҠ” мқҙ кіөл°ұмқ„ л©”мҡ°кё° мң„н•ҙ ID TokenмқҙлқјлҠ” н‘ңмӨҖ нҶ нҒ°мқ„ лҸ„мһ…н–ҲмҠөлӢҲлӢӨ.

ID Token vs Access Token

л‘җ нҶ нҒ°мқҖ нҳ•мӢқмқҙ л№„мҠ·н•ҙ ліҙмқҙм§Җл§Ң лӘ©м Ғкіј кІҖмҰқ мЈјмІҙк°Җ лӢӨлҰ…лӢҲлӢӨ.

н•ӯлӘ© ID Token Access Token
лӘ©м Ғ мӮ¬мҡ©мһҗ мӢ мӣҗ мҰқлӘ… лҰ¬мҶҢмҠӨ м ‘к·ј к¶Ңн•ң
нҳ•мӢқ н•ӯмғҒ JWT JWT лҳҗлҠ” л¶ҲнҲ¬лӘ… л¬ёмһҗм—ҙ
лҢҖмғҒ(audience) нҒҙлқјмқҙм–ёнҠё м• н”ҢлҰ¬мјҖмқҙм…ҳ лҰ¬мҶҢмҠӨ м„ңлІ„
кІҖмҰқ мЈјмІҙ нҒҙлқјмқҙм–ёнҠёк°Җ м§Ғм ‘ кІҖмҰқ лҰ¬мҶҢмҠӨ м„ңлІ„к°Җ кІҖмҰқ
мҲҳлӘ… м§§мқҢ(мҲҳ л¶„) м§§мқҢ~мӨ‘к°„

ID TokenмқҖ нҒҙлқјмқҙм–ёнҠёк°Җ мӮ¬мҡ©мһҗ м •ліҙлҘј м–»кё° мң„н•ң кІғмқҙлҜҖлЎң, нҒҙлқјмқҙм–ёнҠёк°Җ м§Ғм ‘ м„ңлӘ…мқ„ кІҖмҰқн•ҳкі  нҒҙл Ҳмһ„мқ„ мқҪмҠөлӢҲлӢӨ. Access TokenмқҖ API нҳём¶ң мӢң Bearer н—ӨлҚ”лЎң м „мҶЎлҗҳл©°, лҰ¬мҶҢмҠӨ м„ңлІ„к°Җ кІҖмҰқн•©лӢҲлӢӨ.

OIDC мқёмҰқ нқҗлҰ„ мҳҲмӢң

к°ҖмһҘ ліҙнҺём Ғмқё Authorization Code Flowм—җм„ң нҶ нҒ° м—”л“ңнҸ¬мқёнҠё мқ‘лӢөмқҖ лӢӨмқҢкіј к°ҷмҠөлӢҲлӢӨ.

ID Tokenмқҳ нҺҳмқҙлЎңл“ңлҠ” лӢӨмқҢкіј к°ҷмқҖ н‘ңмӨҖ нҒҙл Ҳмһ„мқ„ нҸ¬н•Ён•©лӢҲлӢӨ.

ID Token кІҖмҰқ м Ҳм°Ё

нҒҙлқјмқҙм–ёнҠёлҠ” лӢӨмқҢ лӢЁкі„лҘј лӘЁл‘җ нҶөкіјн•ң ID Tokenл§Ң мӢ лў°н•ҙм•ј н•©лӢҲлӢӨ.

н•өмӢ¬ кІҖмҰқ н•ӯлӘ©мқҖ м„ңлӘ…, iss, aud, exp, nonceмһ…лӢҲлӢӨ. nonceлҠ” мһ¬мғқ кіөкІ©(replay attack)мқ„ л§үкё° мң„н•ҙ мқёмҰқ мҡ”мІӯ мӢң мғқм„ұн•ң к°’кіј мқ‘лӢө ID Tokenмқҳ к°’мқҙ к°ҷм•„м•ј н•©лӢҲлӢӨ.

Access Tokenмқ„ ID TokenмІҳлҹј м“°л©ҙ м•Ҳ лҗҳлҠ” мқҙмң 

нқ”н•ң м•ҲнӢ°нҢЁн„ҙмқҖ нҒҙлқјмқҙм–ёнҠёк°Җ Access Tokenмқ„ л””мҪ”л”©н•ҙ мӮ¬мҡ©мһҗ м •ліҙлҘј м¶”м¶ңн•ҳлҠ” кІғмһ…лӢҲлӢӨ. мқҙлҠ” л‘җ к°Җм§Җ л¬ём ңлҘј л§Ңл“ӯлӢҲлӢӨ.

  • Access Tokenмқҙ JWTк°Җ м•„лӢҗ мҲҳ мһҲкі , нҳ•мӢқмқҙ ліҖкІҪлҗ  мҲҳ мһҲмҠөлӢҲлӢӨ.
  • audienceк°Җ лҰ¬мҶҢмҠӨ м„ңлІ„мқҙлҜҖлЎң нҒҙлқјмқҙм–ёнҠёк°Җ м§Ғм ‘ кІҖмҰқн•ҳл©ҙ м•Ҳ лҗ©лӢҲлӢӨ.

мӮ¬мҡ©мһҗ м •ліҙлҠ” ID Token лҳҗлҠ” /userinfo м—”л“ңнҸ¬мқёнҠём—җм„ң м–»м–ҙм•ј н•©лӢҲлӢӨ.

м–ём ң OIDCк°Җ н•„мҡ”н•ңк°Җ

  • мҶҢм…ң лЎңк·ёмқё(кө¬кёҖ, нҺҳмқҙмҠӨл¶Ғ л“ұ)мқ„ нҶөн•ң мӮ¬мҡ©мһҗ мқёмҰқ
  • кё°м—… лӮҙ SSOм—җм„ң мӮ¬мҡ©мһҗ мӢ мӣҗ м „лӢ¬
  • B2C м„ңл№„мҠӨмқҳ н‘ңмӨҖнҷ”лҗң лЎңк·ёмқё кё°л°ҳ

лӢЁмҲңнһҲ мҷёл¶Җ APIмқҳ лҰ¬мҶҢмҠӨ м ‘к·јл§Ң н•„мҡ”н•ҳлӢӨл©ҙ OAuth2л§ҢмңјлЎң м¶©л¶„н•©лӢҲлӢӨ. к·ёлҹ¬лӮҳ вҖңлҲ„к°Җ лЎңк·ёмқён–ҲлҠ”м§ҖвҖқлҘј м•Ңм•„м•ј н•ңлӢӨл©ҙ OIDCлҘј м„ нғқн•ҙм•ј н•©лӢҲлӢӨ.

л§Ҳл¬ҙлҰ¬

OAuth2лҠ” к¶Ңн•ң мң„мһ„, OIDCлҠ” мӢ мӣҗ нҷ•мқёмқ„ лӢҙлӢ№н•©лӢҲлӢӨ. ID TokenмңјлЎң мӮ¬мҡ©мһҗк°Җ лҲ„кө¬мқём§Җ нҷ•мқён•ҳкі , Access TokenмңјлЎң APIлҘј нҳём¶ңн•ңлӢӨлҠ” м—ӯн•  л¶„лҰ¬лҘј м§ҖнӮӨл©ҙ мқёмҰқ/мқёк°Җ м„Өкі„к°Җ лӢЁмҲңн•ҙм§‘лӢҲлӢӨ. JWT кІҖмҰқ лқјмқҙлёҢлҹ¬лҰ¬м—җ мқҳмЎҙн•ҳкё°ліҙлӢӨ, кІҖмҰқ н•ӯлӘ©мқ„ лӘ…мӢңм ҒмңјлЎң мқҙн•ҙн•ҳкі  м Ғмҡ©н•ҳлҠ” кІғмқҙ м•Ҳм „н•©лӢҲлӢӨ.
