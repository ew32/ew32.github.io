JavaScript 깊은 복사와 얕은 복사

작성일 2026-05-19 Edited on 2026-05-20 In Frontend , JavaScript
JavaScript에서 객체와 배열을 복사하는 방법, 깊은 복사와 얕은 복사의 차이를 정리합니다.

값 타입과 참조 타입

JavaScript의 데이터 타입은 크게 두 가지로 나뉩니다.

분류 타입
원시 타입(primitive) Number, String, Boolean, null, undefined, Symbol, BigInt
참조 타입(reference) Object, Array, Function, Date, Map, Set 등

원시 타입은 값 자체가 복사되지만, 참조 타입은 메모리 주소(참조) 가 복사됩니다.

이 차이가 깊은 복사/얕은 복사 개념의 출발점입니다.

얕은 복사(Shallow Copy)

얕은 복사는 최상위 레벨의 프로퍼티만 새로 만들고, 중첩된 객체는 여전히 같은 참조를 공유합니다.

얕은 복사 방법

얕은 복사의 한계

중첩 객체가 있는 경우 의도하지 않은 부작용이 발생할 수 있습니다.

깊은 복사(Deep Copy)

깊은 복사는 중첩된 모든 레벨까지 재귀적으로 새 객체를 만들어 완전히 분리된 복사본을 생성합니다.

방법 1: JSON.parse(JSON.stringify)

가장 흔하지만 한계가 분명한 방법입니다.

단점:

  • 함수, undefined, Symbol은 사라짐
  • Date는 문자열로 변환됨
  • Map, Set은 빈 객체가 됨
  • 순환 참조 시 에러
  • Infinity, NaN은 null이 됨

방법 2: structuredClone (권장)

ES2022/Node 17+ 표준으로 도입된 structuredClone 은 현재 가장 권장되는 방법입니다.

Date, Map, Set, RegExp, ArrayBuffer 등을 모두 정확히 복제하며, 순환 참조도 지원합니다. 단, 함수와 DOM 노드는 복사할 수 없습니다.

방법 3: 재귀 구현

라이브러리 없이 직접 구현할 수도 있습니다.

방법 4: Lodash cloneDeep

가장 다양한 케이스를 안정적으로 처리합니다.

방법별 비교

방법 함수 Date Map/Set 순환참조 성능
스프레드/Object.assign 얕은 복사만 - - - 빠름
JSON 방식 손실 문자열화 손실 에러 보통
structuredClone 에러 OK OK OK 빠름
Lodash cloneDeep 함수 참조 OK OK OK 느림

불변성과의 관계

React, Redux 같은 라이브러리는 불변성(immutability) 을 전제로 합니다. 상태를 직접 수정하지 않고 매번 새 객체를 만드는 것이죠.

깊이 중첩된 객체일수록 코드가 복잡해지므로 Immer 같은 라이브러리를 활용하면 편리합니다.

성능 고려

  • 얕은 복사는 O(n)이며 매우 빠름
  • 깊은 복사는 객체 크기에 비례하여 비용 증가
  • 매 렌더마다 큰 객체를 깊은 복사하면 성능 저하 발생
  • 가능한 한 얕은 복사 + 변경되는 부분만 새로 만들기 패턴 사용

마무리

JavaScript의 복사는 원시 타입과 참조 타입의 차이를 이해하는 데서 시작합니다. 단순한 데이터는 스프레드로 얕은 복사를 하고, 중첩이 깊거나 특수 타입이 포함된 경우 structuredClone을 사용하는 것이 모던하면서도 안전한 접근입니다. 불변성을 유지하는 코드는 디버깅과 상태 관리를 훨씬 수월하게 만들어 줍니다.
