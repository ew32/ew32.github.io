DNS 동작 원리 한 번에 이해하기

작성일 2026-05-23 Edited on 2026-06-15 In Network , Theory
도메인이 IP 주소로 변환되는 DNS의 계층 구조와 질의 흐름을 정리합니다.

DNS란?

**DNS(Domain Name System)**는 사람이 읽기 쉬운 도메인 이름(example.com)을 컴퓨터가 통신에 사용하는 IP 주소(93.184.216.34)로 변환해주는 분산형 명명 시스템입니다. 인터넷의 “전화번호부”라고 부르는 이유는, 모든 네트워크 통신이 결국 IP를 알아야 시작되기 때문입니다.

DNS는 1983년 RFC 882/883으로 처음 제안되었으며, 오늘날 전 세계에 분산된 수많은 서버가 협력하여 동작하는 가장 큰 분산 시스템 중 하나입니다.

DNS의 계층 구조

DNS는 트리 형태의 계층 구조를 가집니다.

계층 예시 관리 주체
Root . ICANN/IANA
TLD .com, .kr 도메인 레지스트리
SLD example.com 도메인 소유자
서브도메인 www.example.com 도메인 소유자

DNS 서버의 종류

DNS 질의 과정에는 여러 종류의 서버가 관여합니다.

1. 스텁 리졸버 (Stub Resolver)

운영체제에 내장된 클라이언트 측 리졸버. 애플리케이션의 DNS 요청을 받아 재귀 리졸버에 위임합니다.

2. 재귀 리졸버 (Recursive Resolver)

ISP나 공개 DNS 서비스(Google 8.8.8.8, Cloudflare 1.1.1.1)가 운영. 권한 있는 서버를 차례로 탐색하여 최종 답을 반환합니다.

3. 루트 네임서버 (Root Name Server)

전 세계 13개의 논리적 루트 서버. TLD 서버 정보를 제공합니다.

4. TLD 네임서버

.com, .kr 같은 최상위 도메인의 권한 있는 정보를 가집니다.

5. 권한 있는 네임서버 (Authoritative Name Server)

특정 도메인의 실제 레코드를 보유한 서버입니다.

DNS 질의 흐름

www.example.com을 조회하는 전체 과정입니다.

이 과정은 보통 수십~수백 밀리초 안에 완료되며, 대부분의 단계는 캐시로 단축됩니다.

주요 DNS 레코드 타입

레코드 용도 예시
A IPv4 주소 매핑 93.184.216.34
AAAA IPv6 주소 매핑 2606:2800:220:1::
CNAME 별칭 (다른 이름 가리킴) www -> example.com
MX 메일 서버 mail.example.com priority 10
TXT 텍스트 정보 (SPF, DKIM 등) v=spf1 include:_spf...
NS 권한 네임서버 지정 ns1.example.com
SOA 영역의 권한 정보 시리얼, refresh 등
PTR 역방향 조회 IP -> 도메인

실습: dig 명령어

리눅스/맥에서 dig 명령으로 DNS 조회 과정을 확인할 수 있습니다.

86400은 TTL(Time To Live)로, 캐시 유효 시간을 초 단위로 표시합니다.

TTL과 캐싱

DNS의 확장성과 성능은 캐싱에 크게 의존합니다.

  • OS 캐시: 운영체제 수준에서 저장
  • 브라우저 캐시: Chrome은 chrome://net-internals/#dns에서 확인
  • 재귀 리졸버 캐시: TTL 기반으로 저장
  • CDN 캐시: 지역별 빠른 응답을 위해

TTL을 짧게 설정하면 변경이 빠르게 전파되지만, 캐시 효율이 떨어집니다. 일반적으로 다음 값이 사용됩니다.

상황 권장 TTL
안정된 운영 24시간 (86400)
변경 예정 5분 (300)
A/B 테스트 1분 (60)

DNS 보안

DNS는 원래 인증이나 암호화 없이 설계되어 다양한 공격에 취약합니다.

주요 위협

  • DNS 스푸핑/캐시 포이즈닝: 가짜 응답 주입
  • DNS Hijacking: 라우터 또는 DNS 서버 탈취
  • DDoS: DNS 서버에 대한 트래픽 공격
  • DNS Tunneling: 데이터 유출 채널로 악용

보안 강화 기술

  • DNSSEC: 디지털 서명으로 응답 무결성 검증
  • DoT (DNS over TLS): 853 포트, TLS 암호화
  • DoH (DNS over HTTPS): 443 포트, HTTPS 위에서 질의
  • DoQ (DNS over QUIC): QUIC 기반

공개 DNS 서비스 비교

서비스 IPv4 특징
Google 8.8.8.8 빠르고 안정적
Cloudflare 1.1.1.1 프라이버시 강조, 빠름
Quad9 9.9.9.9 보안 필터링
OpenDNS 208.67.222.222 콘텐츠 필터링 옵션

마무리

DNS는 인터넷의 가장 근본적인 인프라이지만 평소엔 보이지 않아 간과되기 쉽습니다. 계층 구조와 질의 흐름, 레코드 타입, TTL과 캐싱, 그리고 DNSSEC/DoH 같은 보안 기술까지 이해하면 장애 진단과 성능 최적화에서 큰 차이를 만들 수 있습니다.
