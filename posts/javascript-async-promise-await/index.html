JavaScript 비동기 처리 - Promise와 async/await

작성일 2026-05-17 Edited on 2026-05-19 In Frontend , JavaScript
JavaScript의 비동기 처리 방식인 콜백, Promise, async/await의 차이와 사용법을 정리합니다.

비동기 처리가 필요한 이유

JavaScript는 단일 스레드(single-threaded) 언어입니다. 한 번에 하나의 작업만 처리할 수 있기 때문에, 네트워크 요청이나 파일 I/O 같은 시간이 오래 걸리는 작업을 동기적으로 처리하면 UI가 멈춰버립니다. 이를 해결하기 위해 등장한 것이 비동기(asynchronous) 처리입니다.

비동기 처리는 시간이 오래 걸리는 작업을 백그라운드에 넘기고, 작업이 끝나면 결과를 받아 처리하는 방식입니다.

콜백 함수와 콜백 헬

비동기 처리의 가장 원시적인 방법은 콜백 함수입니다.

그러나 콜백이 중첩되면 콜백 헬(callback hell) 이라 불리는 가독성 문제가 발생합니다.

Promise의 등장

ES6에서 도입된 Promise는 비동기 작업의 최종 완료(또는 실패)와 그 결과 값을 나타내는 객체입니다. Promise는 다음 세 가지 상태 중 하나를 가집니다.

상태 설명
pending 초기 상태, 아직 완료되지 않음
fulfilled 작업 성공 완료
rejected 작업 실패

Promise 체이닝

then은 항상 새로운 Promise를 반환하므로 체이닝이 가능합니다.

콜백 헬보다 훨씬 평탄한 구조가 됩니다.

async/await

ES2017에서 도입된 async/await는 Promise를 더 직관적이고 동기 코드처럼 작성할 수 있도록 해줍니다.

  • async 키워드가 붙은 함수는 항상 Promise를 반환합니다.
  • await는 Promise가 settled 될 때까지 함수 실행을 일시정지합니다.
  • 에러는 try/catch로 처리합니다.

병렬 처리: Promise.all과 Promise.allSettled

여러 비동기 작업을 병렬로 처리해야 한다면 Promise.all을 사용합니다.

Promise.all은 하나라도 reject되면 즉시 전체가 reject됩니다. 모든 결과를 받고 싶다면 Promise.allSettled를 사용합니다.

비교 정리

방식 가독성 에러 처리 병렬 처리
콜백 나쁨(중첩) 콜백마다 처리 직접 구현
Promise 보통 .catch() Promise.all
async/await 좋음 try/catch Promise.all + await

자주 하는 실수

  1. await를 잊는 경우: await 없이 사용하면 Promise 객체 자체를 받게 됩니다.
  2. 반복문 안에서 순차 처리만 하는 경우: for...of에 await를 쓰면 순차 실행됩니다. 병렬이 필요하면 Promise.all + map 패턴을 사용하세요.
  3. 에러 미처리: .catch() 또는 try/catch를 빠뜨리면 unhandled rejection이 발생합니다.

마무리

JavaScript의 비동기 처리는 콜백 → Promise → async/await로 진화해 왔습니다. 현대 코드에서는 async/await가 가장 일반적이지만, 내부 동작이 Promise라는 점을 잊지 말아야 합니다. 적절한 병렬화와 에러 처리를 통해 안정적이고 효율적인 비동기 코드를 작성할 수 있습니다.
