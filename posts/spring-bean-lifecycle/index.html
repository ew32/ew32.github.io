Spring Bean 라이프사이클 완전 정복

작성일 2026-05-08 Edited on 2026-05-15 In Backend , Java
Spring 컨테이너가 빈을 생성, 의존성 주입, 초기화, 소멸시키는 전체 흐름을 단계별로 살펴봅니다.

Spring 컨테이너와 빈

Spring의 핵심은 IoC 컨테이너가 빈(Bean)의 생명주기를 관리한다는 점입니다. 객체 생성 시점, 의존성 주입 시점, 초기화 콜백, 소멸 콜백이 모두 컨테이너의 통제 아래 있으며, 이를 정확히 이해하면 미묘한 버그를 예방하고 자원을 안전하게 다룰 수 있습니다.

라이프사이클 단계 한눈에 보기

단계 설명
1. 인스턴스화 빈 클래스의 인스턴스 생성
2. 의존성 주입 생성자/세터/필드를 통한 DI
3. Aware 콜백 BeanNameAware, ApplicationContextAware 등
4. BeanPostProcessor 전처리 postProcessBeforeInitialization
5. 초기화 콜백 @PostConstruct, InitializingBean, init-method
6. BeanPostProcessor 후처리 postProcessAfterInitialization (AOP 프록시 생성 시점)
7. 사용 빈 사용
8. 소멸 콜백 @PreDestroy, DisposableBean, destroy-method

초기화 콜백 사용법

빈이 모든 의존성을 갖춘 후 한 번 실행할 코드를 넣는 가장 일반적인 방법은 @PostConstruct입니다.

세 가지 방식의 차이를 정리하면 다음과 같습니다.

방식 특징
@PostConstruct JSR-250 표준, 가장 권장
InitializingBean.afterPropertiesSet() Spring API 의존
@Bean(initMethod = "...") 외부 라이브러리 빈 초기화에 유용

소멸 콜백

애플리케이션 종료 시 자원을 정리해야 한다면 @PreDestroy를 사용합니다.

@Bean(destroyMethod = "...")는 빈 정의 시 외부 클래스의 종료 메서드를 지정합니다. 빈이 AutoCloseable을 구현하면 destroyMethod 추론으로 close()가 자동 호출됩니다.

스코프와 라이프사이클

빈의 스코프(scope)는 생명주기에 큰 영향을 줍니다.

  • singleton (기본): 컨테이너 시작 시 한 번 생성, 컨테이너 종료 시 소멸
  • prototype: 요청 시마다 새로 생성, 컨테이너가 소멸 콜백을 호출하지 않음
  • request / session: 웹 요청 또는 세션 단위로 생성/소멸

프로토타입 빈은 사용자가 자원 정리를 직접 책임져야 합니다.

의존성 주입 시점

생성자 주입은 인스턴스 생성과 동시에 의존성이 채워지지만, 세터 주입과 필드 주입은 인스턴스 생성 후에 채워집니다. 따라서 생성자에서는 다른 빈을 사용할 수 없지만, @PostConstruct 시점에는 안전하게 사용할 수 있습니다.

BeanPostProcessor의 역할

BeanPostProcessor는 초기화 콜백 전후에 모든 빈을 가로채 변경할 수 있는 강력한 확장점입니다. Spring AOP가 만드는 프록시도 사실 AnnotationAwareAspectJAutoProxyCreator라는 BeanPostProcessor가 후처리 단계에서 생성합니다.

이 때문에 @PostConstruct 안에서 this가 프록시가 아닌 원본인 경우가 있어, 자기 호출 시 어드바이스가 적용되지 않습니다.

순환 의존성 처리

생성자 주입은 순환 의존이 있으면 컨테이너 기동 시 즉시 실패합니다. Spring Boot 2.6부터는 기본적으로 허용되지 않으며, 다음 방법으로 해결합니다.

  • 설계 재검토로 의존을 끊기
  • @Lazy를 통한 지연 로딩
  • 이벤트 발행 등 간접 의존으로 전환

디버깅 팁

빈이 언제 만들어지고 어떤 후처리를 거치는지 확인하려면 다음 도구가 유용합니다.

  • --debug 옵션으로 자동 설정 보고서 출력
  • ApplicationContext.getBeanDefinitionNames()로 등록된 빈 확인
  • BeanPostProcessor 추적 로깅으로 초기화 순서 분석

마무리

Spring 빈의 라이프사이클은 단순해 보이지만, AOP 프록시 생성, 스코프, 순환 참조 등의 주제와 맞물려 미묘한 동작 차이를 만듭니다. 각 단계의 역할을 명확히 이해하면 자원 누수, 초기화 누락, 프록시 미적용 같은 문제를 손쉽게 진단할 수 있습니다.
