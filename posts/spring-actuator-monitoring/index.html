Spring Boot Actuator 모니터링 - 엔드포인트와 Prometheus 연동

작성일 2026-04-10 Edited on 2026-06-15 In Backend , Java
Spring Boot Actuator의 주요 엔드포인트, 보안 설정, Micrometer를 통한 Prometheus 메트릭 수집과 Grafana 시각화까지 실전 구성으로 정리합니다.

Actuator란

Spring Boot Actuator는 운영 중인 애플리케이션의 상태, 메트릭, 환경, 빈, 로깅 등 내부 정보를 HTTP 또는 JMX로 노출하는 모듈입니다. 단순 헬스 체크부터 Prometheus 메트릭 수집까지 모두 Actuator를 통해 이뤄집니다.

시작하기

의존성을 추가합니다.

기본 노출 엔드포인트는 health와 info 두 개입니다. 나머지는 명시적으로 노출해야 합니다.

http://localhost:8080/actuator로 접속하면 노출된 엔드포인트 목록을 볼 수 있습니다.

주요 엔드포인트

엔드포인트 설명
/actuator/health 헬스 체크. DB, Redis, 디스크 등 컴포넌트별 상태
/actuator/info 빌드/Git 정보 등 사용자 정의 정보
/actuator/metrics JVM, HTTP, 커스텀 메트릭
/actuator/prometheus Prometheus 형식 메트릭
/actuator/env 환경 변수, 프로퍼티
/actuator/loggers 런타임 로그 레벨 조회·변경
/actuator/threaddump 스레드 덤프
/actuator/heapdump 힙 덤프 다운로드
/actuator/httpexchanges 최근 HTTP 요청·응답
/actuator/mappings 등록된 컨트롤러 매핑

Health Indicator

/actuator/health는 등록된 HealthIndicator들의 결과를 종합합니다. JDBC DataSource, Redis, RabbitMQ 등은 자동 등록되며, 사용자 정의도 쉽습니다.

Kubernetes liveness/readiness 프로브와 연동하려면 별도 그룹을 사용합니다.

이렇게 하면 /actuator/health/liveness, /actuator/health/readiness 엔드포인트가 활성화됩니다.

Micrometer 메트릭

Actuator는 Micrometer를 메트릭 추상화 레이어로 사용합니다. Micrometer는 Prometheus, Datadog, New Relic, CloudWatch 등 다양한 백엔드를 지원합니다. 코드는 한 번만 작성하면 백엔드 교체가 자유롭습니다.

자동 수집되는 메트릭

JVM 메모리, GC, 스레드, HTTP 요청(http.server.requests), Tomcat, HikariCP 등이 자동으로 수집됩니다.

커스텀 메트릭

세 가지 주요 메트릭 타입:

  • Counter — 증가만 가능 (요청 수, 에러 수)
  • Gauge — 현재 값 (큐 크기, 활성 세션)
  • Timer / Distribution Summary — 분포(요청 시간, 응답 크기)

Prometheus 연동

Micrometer Prometheus 레지스트리를 추가하면 /actuator/prometheus 엔드포인트가 활성화됩니다.

tags.application은 모든 메트릭에 공통 태그를 붙여 여러 서비스 메트릭을 구분할 수 있게 해 줍니다.

Prometheus 설정에서 해당 엔드포인트를 스크랩 대상으로 등록합니다.

이후 Grafana에서 Prometheus를 데이터 소스로 연결해 대시보드를 구성합니다. Spring Boot용 공개 대시보드(예: ID 12900, 6756)를 임포트하면 즉시 사용할 수 있습니다.

보안 설정

Actuator 엔드포인트는 운영 정보를 노출하므로 반드시 보호해야 합니다.

운영 환경에서는 Actuator 전용 관리 포트를 분리하는 것이 더 안전합니다.

이렇게 하면 9090 포트가 내부망에서만 열려 외부 노출 위험이 줄어듭니다.

로그 레벨 동적 변경

운영 중에 디버그 로그를 켜야 할 때 매우 유용합니다.

분산 추적과 결합

Micrometer Tracing(Spring Boot 3+)을 함께 사용하면 traceId가 메트릭과 로그에 일관되게 들어가 메트릭 이상 → 로그 → 트레이스로 자연스럽게 따라갈 수 있습니다.

마무리

Actuator는 Spring Boot 애플리케이션 운영의 기본 인프라입니다. 단순한 헬스 체크만으로도 가치가 있지만, Micrometer + Prometheus + Grafana 조합과 결합할 때 본격적인 관측 가능성(observability)을 얻게 됩니다. 보안에 유의해 노출 범위를 적절히 제한하고, 핵심 비즈니스 메트릭을 직접 정의해 대시보드에 올려 두는 것을 추천합니다.
