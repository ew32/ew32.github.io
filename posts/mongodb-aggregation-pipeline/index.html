MongoDB Aggregation Pipeline 완벽 가이드

작성일 2026-04-19 Edited on 2026-06-15 In Backend , Database
MongoDB의 강력한 데이터 분석 도구인 Aggregation Pipeline의 주요 stage와 실전 예제, 성능 최적화 팁을 종합적으로 설명합니다.

Aggregation Pipeline이란?

MongoDB의 Aggregation Pipeline은 문서를 여러 단계(stage)를 거치며 변환·집계하는 강력한 데이터 처리 프레임워크입니다. 각 stage는 이전 stage의 출력을 입력으로 받아 처리하며, 결과적으로 SQL의 SELECT ... GROUP BY ... HAVING ... ORDER BY보다 훨씬 표현력이 풍부한 쿼리를 작성할 수 있습니다.

이 한 줄로 결제 상태인 주문을 사용자별로 합산하여 상위 10명을 가져옵니다.

주요 Stage 정리

$match

가장 자주 쓰이는 stage. 일반 쿼리 조건과 동일하게 작동합니다. 가능한 한 파이프라인의 가장 앞에 두는 것 이 성능에 유리합니다.

$project

필요한 필드만 선택하거나 새로운 필드를 계산합니다.

$group

SQL의 GROUP BY에 해당합니다. _id로 그룹 키를 정의하고, 다양한 집계 연산자를 적용합니다.

$sort

정렬. 인덱스를 활용할 수 있다면 매우 빠릅니다.

$limit / $skip

페이징. 단, $skip은 큰 값에서 느려지므로 keyset 페이징을 고려합니다.

$lookup (조인)

다른 컬렉션과 left outer join을 수행합니다.

여러 조건 조인을 위한 확장 문법도 지원합니다.

$unwind

배열 필드를 풀어 여러 행으로 만듭니다.

$addFields / $set

기존 문서에 필드를 추가합니다. $project와 달리 다른 필드를 제거하지 않습니다.

$facet

같은 데이터로 여러 파이프라인을 병렬 실행합니다. 검색 결과 페이지의 “전체 카운트 + 상위 10건 + 카테고리별 통계”를 한 번에 얻을 때 유용합니다.

$bucket / $bucketAuto

범위별 그룹화. 히스토그램에 적합합니다.

$out / $merge

파이프라인 결과를 새 컬렉션으로 저장하거나 기존 컬렉션에 upsert. ETL과 머터리얼라이즈드 뷰에 유용합니다.

성능 최적화 팁

  1. $match와 $sort를 앞쪽에 배치해 데이터 양을 줄인다
  2. $sort에 활용할 수 있는 인덱스를 만든다
  3. $project로 필요한 필드만 남겨 메모리 사용량 감소
  4. $lookup 대상 컬렉션에 인덱스 필수
  5. 정렬에 100MB 이상 메모리가 필요하면 allowDiskUse: true
  6. 복잡한 파이프라인은 explain()으로 단계별 비용 확인

Aggregation Framework vs Map-Reduce

과거에는 Map-Reduce도 자주 쓰였지만, Aggregation Pipeline이 훨씬 빠르고 표현력이 풍부하므로 새 코드에서는 거의 항상 Aggregation을 선택합니다.

자주 쓰는 표현식

  • 산술: $add, $subtract, $multiply, $divide, $mod
  • 비교: $eq, $ne, $gt, $gte, $lt, $lte
  • 논리: $and, $or, $not
  • 조건: $cond, $switch, $ifNull
  • 배열: $size, $map, $filter, $reduce, $arrayElemAt
  • 날짜: $dateToString, $dateAdd, $year, $month, $dayOfMonth
  • 문자열: $concat, $toLower, $substr, $regexMatch

마무리

MongoDB Aggregation Pipeline은 단순한 검색을 넘어 강력한 데이터 분석과 변환을 가능하게 합니다. 각 stage를 작은 함수처럼 조합해 점진적으로 결과를 다듬는 사고방식이 익숙해지면, 별도의 분석 시스템 없이도 MongoDB 안에서 ETL과 리포팅 작업을 충분히 수행할 수 있습니다.
