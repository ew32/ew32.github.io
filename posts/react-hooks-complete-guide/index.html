React Hooks 완벽 가이드

작성일 2026-05-02 Edited on 2026-05-26 In Frontend , JavaScript
useState, useEffect 등 React Hooks의 주요 API를 예제와 함께 정리합니다.

React Hooks란?

React Hooks는 React 16.8부터 도입된 기능으로, 함수형 컴포넌트에서 상태(state)와 라이프사이클을 관리할 수 있게 해줍니다. 이전에는 클래스 컴포넌트로만 가능했던 기능들이 함수형 컴포넌트로 이전되면서 코드가 훨씬 간결해졌습니다.

useState

컴포넌트에 상태를 추가하는 가장 기본적인 Hook입니다.

이전 상태를 기반으로 업데이트할 때는 함수형 업데이트(prev => prev - 1)를 사용해야 합니다.

useEffect

부수 효과(side effect)를 처리하는 Hook입니다. 데이터 페칭, 구독 설정, DOM 조작 등에 사용됩니다.

의존성 배열의 종류:

  • []: 마운트 시 한 번만 실행
  • [dep1, dep2]: 의존성이 변경될 때마다 실행
  • 생략: 매 렌더링마다 실행

useContext

Prop drilling 없이 컴포넌트 트리 전체에 값을 전달할 수 있습니다.

useMemo와 useCallback

성능 최적화를 위한 메모이제이션 Hook입니다.

주의: 모든 곳에 사용하면 오히려 성능이 저하될 수 있으므로, 실제로 비용이 큰 계산이나 자식 컴포넌트의 불필요한 리렌더링을 막을 때만 사용하세요.

useRef

DOM 요소에 직접 접근하거나, 렌더링을 발생시키지 않는 값을 저장할 때 사용합니다.

커스텀 Hook

로직을 재사용 가능한 함수로 분리할 수 있습니다.

Hooks 사용 규칙

  1. 최상위에서만 호출: 조건문, 반복문, 중첩 함수 내부에서 호출 금지
  2. React 함수 내에서만 호출: 일반 JavaScript 함수에서는 사용 불가

마무리

Hooks를 잘 활용하면 컴포넌트 로직을 더 직관적이고 재사용 가능하게 작성할 수 있습니다. 특히 커스텀 Hook을 통해 로직을 추상화하는 것은 React 개발의 핵심 패턴이므로 적극 활용해보세요.
