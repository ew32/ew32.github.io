대칭키와 비대칭키 암호화 비교

작성일 2026-05-13 Edited on 2026-07-20 In Security , Cryptography
AES와 RSA로 대표되는 대칭키/비대칭키 암호화의 원리와 차이, 그리고 실무에서의 조합 활용을 살펴봅니다.

암호화의 두 갈래

암호화 알고리즘은 키의 사용 방식에 따라 **대칭키(Symmetric)**와 **비대칭키(Asymmetric)**로 나뉩니다. 둘은 경쟁 관계가 아니라 서로의 단점을 보완하는 관계이며, 현대 통신 보안(TLS, SSH, 이메일 암호화)은 거의 모두 두 방식을 함께 씁니다.

대칭키 암호화

암호화와 복호화에 동일한 키를 사용합니다. 대표 알고리즘은 AES입니다.

특징:

  • 빠르다 — 하드웨어 가속(AES-NI)으로 GB/s 처리 가능
  • 키 관리가 어렵다 — 송수신자가 키를 안전하게 공유해야 함
  • 인증/부인방지 기능 없음

AES 운영 모드와 IV

AES는 블록 암호이므로, 평문이 16바이트보다 길면 모드(Mode)를 선택해야 합니다. ECB는 절대 쓰지 마세요 — 같은 평문 블록이 같은 암호문이 되어 패턴이 노출됩니다.

모드 특징 권장도
ECB 패턴 노출 사용 금지
CBC IV 필요, 무결성 별도 레거시
CTR 스트림처럼 동작 양호
GCM 암호화 + 인증(AEAD) 권장

비대칭키 암호화

서로 다른 두 개의 키 — **공개키(Public Key)**와 개인키(Private Key) — 를 사용합니다.

대표 알고리즘:

알고리즘 키 길이 비고
RSA 2048~4096 bit 가장 널리 사용
ECC (ECDSA, ECDH) 256 bit RSA-3072와 동등 강도
Ed25519 256 bit 빠르고 안전, 최신 권장

특징:

  • 키 분배 문제 해결 — 공개키는 자유롭게 배포 가능
  • 느리다 — 대용량 데이터 직접 암호화에 부적합
  • 디지털 서명 가능

핵심 비교

항목 대칭키(AES) 비대칭키(RSA/ECC)
키 개수 1개 (공유) 2개 (공개/개인)
속도 매우 빠름 상대적으로 느림
키 분배 어려움 쉬움
데이터 크기 제한 없음 키 크기 이내
주 용도 대용량 데이터 암호화 키 교환, 서명
대표 알고리즘 AES, ChaCha20 RSA, ECC, Ed25519

하이브리드 암호화: 둘을 결합

실무에서는 두 방식을 결합해 단점을 상쇄합니다. TLS 핸드셰이크가 가장 대표적인 예입니다.

이렇게 하면:

  • 속도: 대량의 데이터는 빠른 대칭키로 암호화
  • 키 분배: 비대칭키로 안전하게 세션 키를 교환
  • 신원 검증: 인증서 서명으로 MITM 방지

디지털 서명

비대칭키의 또 다른 핵심 용도는 서명입니다. 메시지에서 해시를 만들고, 개인키로 서명합니다. 검증자는 공개키로 검증하여 무결성과 출처를 확인합니다.

JWT의 RS256, 코드 서명, 패키지 서명, Git 커밋 서명 등이 모두 이 메커니즘입니다.

키 관리(KMS)의 중요성

암호화 알고리즘이 아무리 안전해도, 키가 평문 파일로 코드 옆에 있으면 의미가 없습니다. 실무에서는 다음을 활용하세요.

  • AWS KMS, GCP KMS, Azure Key Vault
  • HashiCorp Vault
  • HSM(Hardware Security Module)

키는 정기적으로 **회전(Rotation)**해야 하며, 권한은 최소화해야 합니다.

Best Practices

  • 대칭키는 AES-256-GCM 또는 ChaCha20-Poly1305(AEAD)를 기본으로
  • 비대칭키는 Ed25519 또는 ECC P-256 권장, RSA는 3072bit 이상
  • ECB 모드, MD5/SHA-1 서명 금지
  • IV/Nonce는 재사용 금지, 매번 새로 생성
  • 키 저장은 KMS/HSM 사용
  • 자체 구현(Roll-your-own crypto)을 피하고 검증된 라이브러리만 사용

마무리

대칭키는 빠르지만 키를 어떻게 나눌지 고민이 필요하고, 비대칭키는 그 고민을 해결하지만 느립니다. 그래서 현실 세계의 거의 모든 보안 프로토콜은 두 방식을 하이브리드로 결합합니다. 알고리즘 이름을 외우는 것보다, “왜 이 둘을 함께 써야 하는가”를 이해하는 것이 보안의 출발점입니다.
