Vue 3 Composition API 입문

작성일 2026-05-19 Edited on 2026-06-15 In Frontend , JavaScript
Vue 3의 Composition API 핵심 개념과 setup, ref, reactive, computed, watch 사용법을 정리합니다.

Composition API란?

Composition API는 Vue 3에서 새롭게 도입된 컴포넌트 작성 방식입니다. 기존 Options API가 data, methods, computed 등으로 옵션을 나누어 작성했다면, Composition API는 로직 단위로 코드를 묶을 수 있도록 설계되어 재사용성과 가독성을 향상시킵니다.

특히 컴포넌트가 커질수록 관련된 로직이 여러 옵션에 흩어지는 문제를 해결하며, TypeScript와의 통합도 한층 자연스럽습니다.

setup 함수와 <script setup>

Composition API는 setup 함수에서 시작됩니다. Vue 3.2 이후로는 더 간결한 <script setup> 문법을 권장합니다.

<script setup> 안에서 선언한 변수와 함수는 자동으로 템플릿에 노출되어 별도의 return이 필요 없습니다.

ref와 reactive

반응형 상태를 만드는 두 가지 핵심 API입니다.

  • ref() - 원시값을 감싸 반응형으로 만듭니다. 값 접근 시 .value가 필요합니다.
  • reactive() - 객체나 배열을 반응형 프록시로 만듭니다.
API 대상 접근 방식 비고
ref 원시값/객체 .value 템플릿에서는 자동 언래핑
reactive 객체/배열 직접 접근 구조 분해 시 반응성 손실

computed와 watch

파생 상태를 만들 때는 computed, 부수 효과를 처리할 때는 watch를 사용합니다.

watchEffect는 의존성을 자동으로 추적합니다.

라이프사이클 훅과 컴포저블

라이프사이클 훅은 on 접두어가 붙은 함수로 사용합니다.

컴포저블(Composable) 은 재사용 가능한 로직을 함수로 추출하는 패턴입니다.

사용 측에서는 다음과 같이 호출합니다.

Options API와의 비교

항목 Options API Composition API
코드 구성 옵션별로 분리 기능별로 묶음
재사용성 mixins 의존 컴포저블로 깔끔하게
TypeScript 제한적 강력한 지원
학습 곡선 낮음 중간

기존 프로젝트는 Options API를 유지해도 무방하며, 새 프로젝트에서는 Composition API + <script setup>을 권장합니다.

모범 사례

  • 단순 컴포넌트라도 <script setup> 사용을 기본으로 합니다.
  • 반복되는 로직은 useXxx 컴포저블로 추출합니다.
  • reactive로 만든 객체를 구조 분해할 때는 toRefs를 사용해 반응성을 유지합니다.
  • ref와 reactive 중 하나만 일관되게 쓰기보다 상황에 맞춰 사용합니다.

마무리

Composition API는 Vue 3의 핵심 변화이자 강력한 기능입니다. 컴포넌트 로직을 기능 단위로 묶어 재사용성과 유지보수성을 크게 높여주며, TypeScript와의 통합도 자연스럽습니다. 작은 컴포저블부터 만들어보며 점진적으로 적용해보세요.
