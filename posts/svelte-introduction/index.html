Svelte 입문 - React와 다른 접근 방식

작성일 2026-05-20 Edited on 2026-06-15 In Frontend , JavaScript
컴파일러 기반 프론트엔드 프레임워크 Svelte의 핵심 개념과 React와의 차이점을 소개합니다.

Svelte란?

Svelte는 2016년 Rich Harris가 만든 컴파일러 기반 프론트엔드 프레임워크입니다. React, Vue가 런타임 라이브러리로 동작하는 것과 달리, Svelte는 빌드 시점에 컴포넌트를 순수한 JavaScript로 컴파일합니다. 결과적으로 런타임 의존성이 거의 없고 번들 크기가 작으며 성능이 뛰어납니다.

Svelte 5에서는 Runes 라는 새로운 반응성 모델이 도입되어 더욱 명시적이고 강력해졌습니다.

첫 컴포넌트

Svelte 컴포넌트는 .svelte 확장자를 사용하며, HTML/CSS/JS를 한 파일에 자연스럽게 작성합니다.

스타일은 자동으로 해당 컴포넌트에 스코프되므로 다른 컴포넌트에 영향을 주지 않습니다.

Runes로 본 반응성

Svelte 5의 Runes는 $ 접두어가 붙은 특수 함수입니다.

Rune 역할 React 대응
$state 반응형 상태 useState
$derived 파생값 useMemo
$effect 부수 효과 useEffect
$props 컴포넌트 props props 인자

컴포넌트 통신

부모는 props를 내려보내고, 자식은 콜백 함수로 이벤트를 전달합니다.

제어 흐름

Svelte는 템플릿 안에서 별도 문법으로 분기/반복을 표현합니다.

JSX와 비교하면 가독성이 좋고, {#await}처럼 비동기 상태도 템플릿에서 직접 다룹니다.

React와의 비교

항목 React Svelte
동작 방식 가상 DOM, 런타임 컴파일, 직접 DOM 업데이트
번들 크기 큼 매우 작음
학습 곡선 높음 낮음
문법 JSX HTML 확장
스타일 스코프 별도 도구 기본 제공
생태계 매우 큼 작음
풀스택 Next.js SvelteKit

React는 거대한 생태계와 일자리가 강점이고, Svelte는 개발 경험과 번들 크기가 강점입니다.

SvelteKit으로 풀스택

Next.js에 대응하는 SvelteKit은 라우팅, SSR, SSG, 폼 처리, 어댑터 기반 배포를 제공합니다.

모범 사례

  • 컴포넌트를 작게 유지하고 한 가지 책임만 갖게 합니다.
  • 전역 상태는 store 또는 $state.raw를 활용해 명시적으로 분리합니다.
  • 스타일은 컴포넌트 내부에 작성해 스코프의 이점을 누립니다.
  • 풀스택 앱은 SvelteKit을 사용해 라우팅/데이터 패턴 표준을 따릅니다.

마무리

Svelte는 “프레임워크 코드를 사라지게 만들자”는 발상에서 출발한 매력적인 도구입니다. 작은 번들, 빠른 성능, 친숙한 HTML 문법은 새 프로젝트에 시도해볼 충분한 이유입니다. React 사용자에게도 시각을 넓혀주므로 한 번쯤 학습해볼 가치가 있습니다.
