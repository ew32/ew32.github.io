dotfiles 관리법 - 내 환경을 코드로

작성일 2026-07-17 Edited on 2026-05-15 In 개발환경
설정 파일을 Git으로 관리하고 새 기기에서 즉시 복원하는 dotfiles 워크플로를 소개합니다.

왜 dotfiles를 관리해야 할까

.zshrc, .gitconfig, .vimrc, .tmux.conf처럼 홈 디렉터리에 숨겨진 점(.)으로 시작하는 설정 파일들을 통칭 dotfiles라고 부릅니다. 이를 Git으로 관리하면 다음과 같은 이점이 있습니다.

  • 새 컴퓨터에서 한 번의 명령으로 환경 복원
  • 변경 이력 추적과 롤백
  • 팀원·커뮤니티와 설정 공유
  • 백업 부담 제거

접근 방식

dotfiles 관리에는 크게 세 가지 패턴이 있습니다.

1. 심볼릭 링크 + GNU Stow

가장 가볍고 표준적인 방식입니다. 저장소 안에 패키지 단위로 디렉터리를 두고, stow가 자동으로 심볼릭 링크를 만들어 줍니다.

2. 베어 저장소(bare repo)

홈 디렉터리 자체를 Git 워킹 트리로 만드는 트릭입니다. 별도 폴더 없이도 동작합니다.

새 기기에서는 다음과 같이 복원합니다.

3. 전용 도구

chezmoi, yadm, dotbot 같은 도구는 템플릿(기기별 차이 처리), 비밀값 암호화 같은 고급 기능을 제공합니다. 여러 OS와 워크 PC/개인 PC를 동시에 관리한다면 유용합니다.

디렉터리 구조 예시

부트스트랩 스크립트

새 기기에서 한 줄로 모든 것을 셋업하는 스크립트가 핵심입니다.

비밀값 처리

API 키나 토큰은 절대 dotfiles 저장소에 커밋하면 안 됩니다. 두 가지 패턴이 흔합니다.

  • ~/.zshrc.local 같은 비공개 파일을 만들고 메인 .zshrc에서 [[ -f ~/.zshrc.local ]] && source ~/.zshrc.local
  • chezmoi나 git-crypt로 암호화해 커밋

기기별 차이 처리

회사/개인 컴퓨터, macOS/Linux 차이는 조건 분기로 처리합니다.

마무리

dotfiles 관리는 “내 개발 환경을 코드로 다루는” 첫걸음입니다. 작게 시작해 매번 새 도구를 도입할 때마다 한 줄씩 커밋하면, 어느 순간 새 기기 셋업이 5분 만에 끝나는 마법을 경험할 수 있습니다.
