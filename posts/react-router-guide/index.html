React Router v6 사용법

작성일 2026-05-20 Edited on 2026-05-18 In Frontend , JavaScript
React Router v6의 핵심 API인 BrowserRouter, Routes, useNavigate, 중첩 라우팅, 데이터 라우터 사용법을 정리합니다.

React Router 소개

React Router는 React 생태계에서 사실상 표준으로 사용되는 라우팅 라이브러리입니다. v6에서는 API가 크게 정리되어 더 직관적이고 작은 번들 크기를 갖게 되었으며, v6.4 이후로는 데이터 로딩까지 지원하는 데이터 라우터(Data Router) 가 추가되었습니다.

이 글에서는 v6 기준으로 핵심 사용법을 정리합니다.

기본 설정

먼저 라이브러리를 설치합니다.

앱의 최상위에 BrowserRouter를 두고 Routes 안에 Route를 선언합니다.

v5와 달리 component 대신 element prop을 사용하며, Switch는 Routes로 대체되었습니다.

링크와 프로그램 방식 이동

페이지 전환은 <Link> 또는 useNavigate 훅으로 수행합니다.

활성 상태 스타일링이 필요하면 <NavLink>를 사용합니다.

동적 경로와 쿼리

:파라미터 표기로 동적 세그먼트를 정의하고 useParams로 읽습니다.

중첩 라우팅과 Outlet

레이아웃을 공유하는 페이지들은 중첩 라우트로 깔끔하게 구성할 수 있습니다.

부모 컴포넌트에서는 자식이 렌더링될 위치에 <Outlet>을 둡니다.

데이터 라우터 (createBrowserRouter)

v6.4부터는 loader/action을 통해 데이터 페칭과 폼 처리를 라우트 정의에 포함할 수 있습니다.

이 패턴은 페이지 전환 시 데이터를 라우팅과 병렬로 가져와 폭포수 요청을 줄여줍니다.

보호 라우트

인증이 필요한 라우트는 래퍼 컴포넌트로 처리합니다.

v5와의 주요 차이

항목 v5 v6
라우트 매칭 Switch Routes
컴포넌트 지정 component element
중첩 표현 직접 처리 자동 + Outlet
useHistory 있음 useNavigate
데이터 로딩 별도 loader/action

모범 사례

  • 페이지 컴포넌트는 pages/ 또는 routes/로 디렉토리 구조를 분리합니다.
  • 공통 레이아웃은 중첩 라우트 + Outlet 패턴을 적극 사용합니다.
  • 코드 스플리팅을 위해 React.lazy와 함께 사용합니다.
  • 데이터 라우터를 도입하면 Suspense, 에러 경계, 폼 처리가 일관됩니다.

마무리

React Router v6는 더 적은 코드로 더 많은 일을 할 수 있도록 잘 정리된 API를 제공합니다. 작은 앱에서는 BrowserRouter로 시작하고, 데이터 페칭과 폼이 많아지면 데이터 라우터로 전환하는 것이 자연스러운 학습 경로입니다.
