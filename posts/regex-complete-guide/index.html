정규표현식 완벽 가이드

작성일 2026-05-29 Edited on 2026-05-12 In Tutorial
정규표현식의 메타문자, 그룹, 룩어라운드까지 핵심 문법을 예제와 함께 정리합니다.

정규표현식이란?

**정규표현식(Regular Expression, RegEx)**은 문자열의 패턴을 표현하기 위한 형식 언어입니다. 텍스트 검색, 입력값 검증, 추출, 치환 등 거의 모든 문자열 처리 작업에 사용됩니다.

정규표현식은 1950년대 수학자 Stephen Kleene이 형식 언어 이론에서 정립한 개념이 발전한 것으로, 오늘날에는 Perl, PCRE, POSIX 등 다양한 방언이 존재합니다. 이 글에서는 PCRE에 가까운 현대 정규식을 기준으로 설명합니다.

기본 매칭

정규식의 가장 단순한 형태는 리터럴 매칭입니다.

특수한 의미를 가진 메타문자는 별도로 처리됩니다.

메타문자 의미
. 임의의 한 문자 (개행 제외)
^ 문자열/줄의 시작
$ 문자열/줄의 끝
\ 이스케이프
| OR (또는)
() 그룹화
[] 문자 클래스
{} 반복 횟수

문자 클래스

대괄호 안의 문자 중 하나와 매치됩니다.

축약형도 자주 사용됩니다.

축약 의미
\d 숫자 [0-9]
\D 숫자 아님
\w 단어 문자 [a-zA-Z0-9_]
\W 단어 문자 아님
\s 공백 (스페이스, 탭, 개행)
\S 공백 아님
\b 단어 경계

수량자(Quantifier)

반복 횟수를 지정합니다.

수량자 의미
* 0회 이상
+ 1회 이상
? 0회 또는 1회
{n} 정확히 n회
{n,} n회 이상
{n,m} n회 이상 m회 이하

예제입니다.

탐욕(greedy)과 게으름(lazy)

기본적으로 수량자는 탐욕적이라 가능한 한 길게 매치합니다. ?를 붙이면 게으른 매칭이 됩니다.

HTML 태그 추출 같은 작업에서는 거의 항상 lazy 수량자가 필요합니다.

그룹과 캡처

소괄호로 묶으면 그룹이 되고, 매치된 부분을 캡처할 수 있습니다.

캡처를 원하지 않으면 비캡처 그룹 (?:...)을 사용합니다. 성능과 가독성에 유리합니다.

명명된 그룹도 가능합니다.

룩어라운드(Lookaround)

매치 위치 주변의 조건을 검사하되 결과에는 포함하지 않습니다.

구문 의미
(?=...) 긍정형 lookahead
(?!...) 부정형 lookahead
(?<=...) 긍정형 lookbehind
(?<!...) 부정형 lookbehind

예제입니다.

자주 쓰는 패턴

용도 패턴
이메일 (간단) ^[\w.-]+@[\w.-]+\.\w+$
URL https?://[^\s]+
IPv4 \b(?:\d{1,3}\.){3}\d{1,3}\b
전화번호 (한국) ^01[016-9]-\d{3,4}-\d{4}$
16진수 색상 ^#(?:[0-9a-fA-F]{3}){1,2}$
HTML 태그 <[^>]+>

이메일을 RFC에 맞춰 완벽히 검증하려면 매우 복잡해지므로, 간단한 형식 검사 + 실제 발송 확인이 현실적입니다.

디버깅 팁

복잡한 정규식은 작성과 디버깅이 어렵습니다.

  • regex101.com 같은 시각화 도구로 한 단계씩 확인
  • re.VERBOSE 플래그로 주석 달린 멀티라인 정규식 작성
  • 너무 복잡해지면 여러 단계로 분리해서 처리
  • 백트래킹 폭주(catastrophic backtracking) 주의

마무리

정규표현식은 처음에는 암호처럼 보이지만, 메타문자 -> 문자 클래스 -> 수량자 -> 그룹 -> 룩어라운드 순으로 단계적으로 익히면 충분합니다. 단, 너무 복잡한 정규식은 가독성과 성능 모두에 해가 되므로 적절한 시점에서 파서나 전용 라이브러리로 전환하는 판단도 중요합니다.
