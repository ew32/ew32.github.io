CSS 변수(Custom Properties) 활용법

작성일 2026-05-21 Edited on 2026-05-19 In Frontend , CSS
CSS Custom Properties의 선언과 사용, 동적 변경, 테마 구현 그리고 Sass 변수와의 차이를 정리합니다.

CSS 변수란?

CSS 변수(Custom Properties) 는 CSS 안에서 값을 재사용할 수 있도록 해주는 기능입니다. --이름으로 선언하고 var(--이름)으로 사용하며, 모든 모던 브라우저에서 지원됩니다.

Sass나 Less의 변수와 비슷해 보이지만, 결정적인 차이가 있습니다.

  • 런타임에 살아있다 - JavaScript로 변경 가능, DevTools에서 수정 가능
  • 상속된다 - DOM 트리를 따라 자식에게 전달됨
  • 계산식에 사용 가능 - calc()와 결합 가능

기본 선언과 사용

전역에서 사용할 변수는 보통 :root에 선언합니다.

특정 컴포넌트 범위에만 적용하려면 해당 선택자에서 선언합니다.

폴백값과 계산

var()는 두 번째 인자로 폴백값을 받을 수 있습니다.

calc()와 결합해 동적 계산이 가능합니다.

JavaScript와의 연동

DOM API로 CSS 변수를 읽고 쓸 수 있습니다.

이 특성을 활용하면 테마 전환, 사용자 설정, 애니메이션 등을 매우 깔끔하게 구현할 수 있습니다.

다크 모드 구현

CSS 변수와 prefers-color-scheme 미디어 쿼리, 그리고 data-theme 속성을 조합하는 패턴이 일반적입니다.

JavaScript로 사용자 선택을 반영합니다.

Sass 변수와의 비교

항목 CSS 변수 Sass 변수
실행 시점 런타임 컴파일 타임
JS 접근 가능 불가
상속 가능 불가
미디어 쿼리 내 변경 가능 불가
브라우저 지원 모던 브라우저 무관 (컴파일 후 사라짐)

Sass 변수는 빌드 시 고정값이 필요할 때, CSS 변수는 런타임에 변하는 값이 필요할 때 적합합니다. 둘은 보완 관계입니다.

디자인 토큰 시스템

큰 프로젝트에서는 디자인 토큰을 CSS 변수로 정의해 일관성을 유지합니다.

이렇게 정의된 토큰을 컴포넌트가 참조하면, 디자인 시스템을 코드로 표현하기 쉬워집니다.

모범 사례

  • 전역 토큰은 :root에, 컴포넌트 단위 변수는 해당 선택자에 둡니다.
  • 의미 있는 이름을 사용합니다 (--blue-500보다 --color-primary).
  • 폴백값을 함께 제공해 회복 탄력성을 높입니다.
  • DevTools에서 값을 바꿔보며 디자인을 빠르게 검증합니다.

마무리

CSS 변수는 단순한 변수 기능을 넘어 런타임에 동작하는 디자인 시스템의 기반입니다. 다크 모드, 테마, 동적 스타일링을 자바스크립트와 자연스럽게 결합할 수 있게 해주죠. 기존에 Sass 변수만 사용했다면, 이제 두 가지를 적절히 조합하는 방향을 추천합니다.
