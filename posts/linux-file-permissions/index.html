리눅스 파일 권한 완벽 이해

작성일 2026-05-25 Edited on 2026-07-20 In System , OS
리눅스 파일 권한 체계와 chmod, chown 명령어 사용법을 실무 예제와 함께 정리합니다.

들어가며

리눅스는 멀티 유저 운영체제이기 때문에 파일 권한 관리가 핵심입니다. 권한을 잘못 설정하면 보안 사고로 이어지거나, 반대로 너무 엄격하게 설정하면 정상 동작이 막힙니다. 이 글에서는 리눅스 권한 체계와 chmod, chown, umask 등 관련 명령어를 정리합니다.

권한 확인하기

ls -l 명령어로 파일과 디렉토리의 권한을 확인할 수 있습니다.

출력의 각 부분은 다음을 의미합니다.

  • - 또는 d: 파일 종류 (-: 일반 파일, d: 디렉토리, l: 심볼릭 링크)
  • rw-r--r--: 권한 정보 (소유자, 그룹, 그 외)
  • 1: 하드 링크 수
  • alice: 소유자(Owner)
  • developers: 그룹(Group)
  • 1024: 파일 크기

권한의 세 가지 영역

각 파일은 세 종류의 사용자에 대해 권한을 가집니다.

영역 의미
User(u) 파일 소유자
Group(g) 파일이 속한 그룹
Others(o) 나머지 모두

각 영역은 세 가지 권한을 가질 수 있습니다.

기호 의미 (파일) 의미 (디렉토리) 숫자 값
r (read) 읽기 목록 조회 4
w (write) 쓰기/수정 파일 생성/삭제 2
x (execute) 실행 디렉토리 진입 1

chmod로 권한 변경

chmod는 권한을 변경하는 명령어입니다. 두 가지 방식이 있습니다.

숫자(8진수) 모드

각 영역의 권한을 합산한 숫자를 사용합니다.

8진수 권한 의미
777 rwxrwxrwx 모든 권한 (위험)
755 rwxr-xr-x 실행 파일, 디렉토리
644 rw-r–r– 일반 문서
600 rw——- 비밀 파일
400 r——– 읽기 전용 비밀

기호 모드

chown으로 소유자 변경

chown은 파일의 소유자와 그룹을 변경합니다. root 권한이 필요합니다.

umask: 기본 권한 설정

새로 생성되는 파일과 디렉토리의 기본 권한은 umask 값으로 결정됩니다.

  • 파일 기본 권한: 666 - umask
  • 디렉토리 기본 권한: 777 - umask

특수 권한: SUID, SGID, Sticky Bit

권한 숫자 동작
SUID 4000 실행 시 소유자 권한으로 실행 (/usr/bin/passwd)
SGID 2000 디렉토리에 새 파일 생성 시 그룹 자동 상속
Sticky Bit 1000 디렉토리 내 파일은 소유자만 삭제 가능 (/tmp)

SSH 키 권한

ACL: 더 세밀한 권한 제어

전통적인 권한으로는 “특정 사용자에게만 권한 부여”가 어렵습니다. ACL(Access Control List) 을 사용하면 가능합니다.

마무리

리눅스 권한 체계는 단순해 보이지만, 실제 운영 환경에서는 SUID, SGID, ACL, umask 등 다양한 요소가 복합적으로 작용합니다. chmod로 권한을, chown으로 소유자를 변경할 수 있으며, 보안의 기본은 필요한 최소 권한만 부여하는 것입니다. 권한 관리를 잘 이해해 두면 서버 운영, 컨테이너 배포, 협업 환경 모두에서 큰 도움이 됩니다.
