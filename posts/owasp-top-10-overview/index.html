OWASP Top 10 웹 취약점 개요

작성일 2026-05-14 Edited on 2026-07-20 In Security
OWASP Top 10이 정의하는 대표적인 웹 애플리케이션 취약점 10가지와 각각의 방어 원칙을 정리합니다.

OWASP Top 10이란?

**OWASP(Open Worldwide Application Security Project)**는 전 세계 보안 커뮤니티가 모여 만든 비영리 단체로, 웹 보안 분야의 표준에 가장 큰 영향을 미치는 조직입니다. Top 10은 그중에서도 가장 자주 인용되는 문서로, 실제 사고 데이터와 설문을 기반으로 가장 빈번하고 심각한 웹 취약점 10가지를 추려 발표합니다.

본 글은 가장 최신판(2021)을 기준으로 각 항목의 핵심과 방어 원칙을 압축적으로 정리합니다.

A01: Broken Access Control

가장 빈번한 취약점 1위. 사용자가 자기 권한을 넘어선 리소스에 접근할 수 있는 모든 결함입니다. IDOR, 권한 상승, 강제 브라우징 등이 모두 포함됩니다.

방어:

  • 서버 측 권한 검증을 모든 엔드포인트에서 수행
  • 거부가 기본(Deny by default)
  • 객체 단위로 소유자 검증

A02: Cryptographic Failures

암호화 미적용 또는 잘못된 사용. 평문 전송, 약한 알고리즘(MD5, SHA-1, DES), 키 재사용, IV 재사용 등이 해당됩니다.

방어:

  • 전송 구간 TLS 1.2+ 강제
  • 저장 시 AEAD 모드(AES-GCM) 사용
  • KMS/HSM으로 키 관리
  • 자체 암호화 구현 금지

A03: Injection

SQL Injection, NoSQL Injection, Command Injection, LDAP Injection 등 외부 입력이 인터프리터의 구문으로 해석될 때 발생합니다.

방어:

  • 매개변수 바인딩(Prepared Statement)
  • ORM의 raw query 자제
  • 화이트리스트 입력 검증
  • 운영 시 최소 권한 원칙

A04: Insecure Design

코드가 아닌 설계 단계의 결함. 비밀번호 재설정 흐름이 토큰 검증 없이 동작하거나, 결제 검증이 클라이언트에서만 이뤄지는 경우 등이 해당됩니다.

방어:

  • 위협 모델링(Threat Modeling) 도입
  • 보안 요구사항을 기능 명세에 포함
  • 시큐어 디자인 패턴 라이브러리 활용

A05: Security Misconfiguration

기본 비밀번호, 디버그 모드 노출, 디렉토리 리스팅, 불필요한 기능 활성화, 미설정 HTTP 보안 헤더 등 설정상의 결함입니다.

방어:

  • Infrastructure as Code로 일관된 설정
  • 운영/개발 환경 분리, 기본 계정 제거
  • X-Frame-Options, Content-Security-Policy, Strict-Transport-Security 등 보안 헤더 적용

A06: Vulnerable and Outdated Components

사용 중인 라이브러리, 프레임워크, 컨테이너 이미지에 알려진 취약점(CVE)이 존재하는 경우입니다. Log4Shell처럼 단 하나의 의존성이 전체 인프라를 뚫는 사례가 늘고 있습니다.

방어:

  • SCA(Software Composition Analysis) 도구 도입(Snyk, Dependabot, Trivy)
  • SBOM(Software Bill of Materials) 생성
  • 정기적 업그레이드 및 패치 정책

A07: Identification and Authentication Failures

세션 고정, 무차별 대입 허용, 약한 비밀번호 정책, MFA 부재 등 인증 관련 결함입니다.

방어:

  • 비밀번호: Argon2id/bcrypt + Pepper
  • Rate Limit 및 계정 잠금
  • MFA 도입
  • 세션 ID는 로그인 시 회전

A08: Software and Data Integrity Failures

서명되지 않은 업데이트, 신뢰할 수 없는 CDN 의존, 비검증된 역직렬화 등 무결성 검증 부재가 원인입니다. SolarWinds 사건 이후 주목도가 크게 높아진 항목입니다.

방어:

  • 패키지/이미지 서명 검증(Sigstore, cosign)
  • CI/CD 파이프라인 보호
  • 역직렬화는 신뢰된 소스만 허용

A09: Security Logging and Monitoring Failures

침해를 탐지하지 못하면 방어가 무력화됩니다. 평균 침해 탐지 시간(MTTD)은 여전히 수개월 단위입니다.

방어:

  • 인증 실패, 권한 거부, 관리자 동작 등 보안 이벤트 로깅
  • 중앙 집중식 로그 및 알림(SIEM)
  • 개인정보·자격증명은 로그에 남기지 않음

A10: Server-Side Request Forgery (SSRF)

서버가 클라이언트의 입력에 따라 내부 네트워크로 요청을 보내도록 유도되는 취약점입니다. 클라우드 메타데이터 엔드포인트(169.254.169.254)를 노린 공격이 대표적입니다.

방어:

  • 외부로 나가는 요청에도 출력 검증
  • VPC 메타데이터 IMDSv2 강제(AWS)
  • 내부망으로의 요청은 명시적 차단
  • DNS Rebinding 방어를 위한 IP 재확인

우선순위와 적용 전략

우선순위 권장 조치
즉시 A01, A03, A07 — 인증/인가/입력 검증
단기 A05, A06 — 설정/의존성 정비
중기 A02, A09, A10 — 암호화/모니터링/SSRF
장기 A04, A08 — 설계 보안, 공급망 보안

마무리

OWASP Top 10은 유행이 아니라 데이터에 기반한 위협 목록입니다. 한 번에 다 잡으려 하지 말고, 자주 발생하는 항목부터 코드 리뷰 체크리스트, 보안 헤더, SAST/DAST/SCA 도구로 단계적으로 흡수해 나가세요. 보안은 단발 프로젝트가 아니라 지속적인 운영 활동입니다.
