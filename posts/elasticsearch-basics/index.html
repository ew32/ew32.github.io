Elasticsearch 기초 - 역색인부터 쿼리 DSL까지

작성일 2026-04-15 Edited on 2026-05-15 In Backend , Database
Elasticsearch의 핵심 개념인 역색인, 매핑, 분석기, 쿼리 DSL을 정리하고 실제 인덱싱과 검색 예제를 통해 검색 엔진의 동작을 이해합니다.

Elasticsearch란?

Elasticsearch는 Apache Lucene 기반의 분산 검색·분석 엔진입니다. JSON 문서를 저장하고, 풀텍스트 검색과 실시간 집계를 매우 빠르게 수행할 수 있어 로그 분석, 검색 서비스, 모니터링 등 다양한 분야에서 사용됩니다. ELK 스택(Elasticsearch + Logstash + Kibana)의 핵심 구성요소이기도 합니다.

핵심 개념

클러스터, 노드, 샤드

  • Cluster: 여러 노드의 모음
  • Node: 단일 Elasticsearch 인스턴스
  • Index: 비슷한 특성을 가진 문서들의 모음 (RDB의 데이터베이스 또는 테이블에 비유)
  • Document: JSON 형태의 데이터 단위 (RDB의 row)
  • Shard: 인덱스를 물리적으로 나눈 단위. 분산과 병렬 처리를 가능하게 함
  • Replica: 샤드의 복제본. 가용성과 검색 성능 향상

역색인(Inverted Index)

Elasticsearch의 빠른 검색의 비밀은 역색인 입니다. 일반적인 데이터베이스는 행 단위로 데이터를 저장하지만, 역색인은 “어떤 단어가 어떤 문서에 나타나는지”를 미리 기록해 둡니다.

예시: 다음 두 문서가 있다고 가정합니다.

  • Doc1: “Elasticsearch is fast”
  • Doc2: “Elasticsearch is distributed”

역색인을 만들면 다음과 같이 구성됩니다.

Term Documents
elasticsearch Doc1, Doc2
is Doc1, Doc2
fast Doc1
distributed Doc2

이 구조 덕분에 “fast”로 검색하면 즉시 Doc1을 찾을 수 있습니다.

매핑(Mapping)

매핑은 문서의 구조와 필드 타입을 정의합니다. 자동 매핑도 가능하지만, 운영 환경에서는 명시적 매핑 을 강력히 권장합니다.

text vs keyword

  • text: 분석기를 거쳐 토큰화됨. 풀텍스트 검색용
  • keyword: 분석 없이 그대로 저장. 정렬, 집계, exact match용

같은 필드를 두 용도로 쓰려면 multi-field로 구성하는 것이 일반적입니다.

분석기(Analyzer)

text 타입 필드는 인덱싱 시 분석기를 거칩니다. 분석기는 세 단계로 구성됩니다.

  1. Character Filter: 원본 텍스트 전처리 (HTML 제거 등)
  2. Tokenizer: 토큰 단위로 분리
  3. Token Filter: 토큰 변환 (소문자화, 불용어 제거, 어간 추출 등)

한국어는 형태소 분석을 위해 Nori 또는 은전한닢 같은 분석기를 사용합니다.

Query DSL

Elasticsearch의 강력함은 Query DSL 에서 나옵니다. JSON 기반으로 매우 표현력이 풍부한 쿼리를 작성할 수 있습니다.

match query (풀텍스트)

term query (정확 일치)

range query

bool query (조합)

  • must: 반드시 일치, 스코어에 영향
  • filter: 반드시 일치, 스코어에 영향 없음 (캐시됨)
  • must_not: 일치하면 제외
  • should: 일치하면 가산점

집계(Aggregation)

집계는 그룹별 통계, 시계열, 히스토그램 등 다양한 분석을 지원하며 Kibana 대시보드의 기반이 됩니다.

운영 시 고려사항

  • 샤드 수 설계: 너무 많으면 오버헤드, 너무 적으면 확장성 저하 (일반적으로 샤드당 10~50GB 권장)
  • 인덱스 라이프사이클(ILM): 시계열 데이터는 hot-warm-cold-delete 단계 자동화
  • Refresh interval: 기본 1초인데, 대량 색인 시 늘리면 성능 향상
  • Bulk API: 단건 색인 대신 _bulk 사용

마무리

Elasticsearch는 단순한 데이터 저장소가 아니라 검색·분석을 위해 설계된 분산 엔진입니다. 역색인, 매핑, 분석기의 동작 원리를 이해하고 Query DSL을 자유롭게 다루면 강력한 검색 경험과 실시간 분석을 손쉽게 구현할 수 있습니다.
