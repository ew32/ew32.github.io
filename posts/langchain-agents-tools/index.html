LangChain Agents와 Tools 이해하기

작성일 2026-07-11 Edited on 2026-05-20 In AI도구
LangChain의 Agent와 Tool 추상화를 살펴보고, 외부 도구를 호출하는 LLM 에이전트를 구현하는 방법을 정리합니다.

Agent와 Tool의 개념

LangChain의 Tool은 LLM이 호출할 수 있는 외부 기능을 표준화한 객체입니다. 검색 API, 계산기, 데이터베이스 쿼리, 사내 API 모두 Tool로 감쌀 수 있습니다. Agent는 사용자의 자연어 입력을 받아 “어떤 Tool을 어떤 인자로 호출할지”를 LLM이 스스로 결정하고, 결과를 보고 다음 행동을 이어가는 루프입니다.

이 패턴을 ReAct(Reason + Act)라고 부릅니다.

간단한 Tool 정의

@tool 데코레이터를 사용하면 함수가 곧 Tool이 됩니다. 함수의 docstring이 LLM에게 보여지는 설명이므로 명확하게 작성해야 합니다.

Agent 구성

LangGraph의 create_react_agent가 가장 단순한 진입점입니다.

LLM은 먼저 get_weather("서울")을 호출하고, 결과에서 21을 파싱해 add_numbers(21, 5)를 호출하는 식으로 두 Tool을 연쇄 사용합니다.

Tool 설계 원칙

좋은 Tool은 좋은 함수 시그니처와 같습니다.

  • 단일 책임: 한 Tool은 한 가지 일만
  • 명확한 docstring: LLM이 언제 호출할지 판단하는 유일한 근거
  • 타입 힌트 필수: Pydantic 스키마로 변환되어 인자 검증에 사용
  • 에러는 의미 있는 문자열로: 예외를 던지지 말고 “도시를 찾을 수 없음” 같이 LLM이 이해할 메시지를 반환

메모리와 멀티턴

대화형 에이전트에는 메모리가 필요합니다. LangGraph는 checkpointer로 상태를 보존합니다.

안전장치

에이전트는 자유도가 큰 만큼 폭주 위험이 있습니다.

  • max_iterations: 무한 루프 방지
  • Human-in-the-loop: 위험한 Tool은 실행 전 사용자 승인 단계 삽입
  • Tool 화이트리스트: 사용자 권한별로 노출 Tool 제한
  • 로깅과 트레이싱: LangSmith로 호출 흐름 시각화

마무리

LangChain Agent는 “LLM이 도구를 쓰게 만드는” 패턴을 표준화합니다. 작은 Tool을 명확히 정의하고, ReAct 루프와 메모리, 안전장치를 결합하면 실무에서 쓸 만한 자동화 에이전트를 만들 수 있습니다.
