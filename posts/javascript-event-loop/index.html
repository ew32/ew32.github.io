JavaScript 이벤트 루프 동작 원리

작성일 2026-05-17 Edited on 2026-07-20 In Frontend , JavaScript
JavaScript 런타임의 이벤트 루프, 콜 스택, 태스크 큐, 마이크로태스크 큐의 관계를 정리합니다.

이벤트 루프란?

JavaScript 엔진은 단일 스레드로 동작하지만, 우리는 setTimeout, fetch, 이벤트 리스너 같은 비동기 작업을 자유롭게 사용합니다. 이를 가능하게 만드는 핵심 메커니즘이 바로 이벤트 루프(Event Loop) 입니다.

이벤트 루프는 콜 스택이 비었는지 확인하고, 큐에 대기 중인 작업을 스택에 올려서 실행시키는 무한 루프입니다.

JavaScript 런타임 구성 요소

브라우저 환경의 JavaScript 런타임은 다음과 같이 구성됩니다.

구성 요소 역할
Call Stack 현재 실행 중인 함수의 호출 컨텍스트를 저장
Heap 객체 등 동적으로 할당된 메모리 영역
Web APIs 브라우저가 제공하는 API (DOM, fetch, setTimeout 등)
Task Queue 매크로태스크 큐 (setTimeout, setInterval, I/O 등)
Microtask Queue 마이크로태스크 큐 (Promise.then, queueMicrotask 등)
Event Loop 큐와 콜 스택 사이의 조정자

동작 순서

이벤트 루프는 다음 규칙을 반복합니다.

  1. 콜 스택이 비어있는지 확인
  2. 마이크로태스크 큐의 모든 작업을 먼저 처리
  3. 렌더링(필요한 경우)
  4. 태스크 큐에서 가장 오래된 작업 하나를 꺼내 스택에 올림
  5. 다시 1번으로

핵심은 “마이크로태스크 큐가 비워질 때까지 매크로태스크는 실행되지 않는다”는 점입니다.

매크로태스크 vs 마이크로태스크

  • 1, 4는 동기 코드이므로 즉시 실행
  • 3은 마이크로태스크 큐로 들어감
  • 2는 매크로태스크(타이머) 큐로 들어감
  • 동기 코드가 끝나면 마이크로태스크 큐가 먼저 비워지고, 그 다음 매크로태스크가 실행됨
종류 예시
매크로태스크 setTimeout, setInterval, setImmediate(Node), I/O, UI 렌더링
마이크로태스크 Promise.then/catch/finally, queueMicrotask, MutationObserver

콜 스택과 함수 실행

실행 과정에서 스택은 다음과 같이 변화합니다.

  1. foo() 호출 → 스택에 foo push
  2. foo 내부에서 bar() 호출 → bar push
  3. console.log('bar') 실행 → console.log push, 실행 후 pop
  4. bar 종료 → pop
  5. foo 종료 → pop, 스택 비어있음

이벤트 루프와 비동기 코드

Promise.then이 연속으로 체이닝된 경우, 각 then이 별도의 마이크로태스크로 들어가지만, 이벤트 루프는 하나의 틱(tick) 안에서 마이크로태스크 큐를 전부 비웁니다.

렌더링과의 관계

브라우저는 보통 60fps를 목표로 약 16.7ms마다 화면을 렌더링합니다. 이벤트 루프 한 틱에서 콜 스택이 너무 오래 점유되면 렌더링이 지연되어 화면이 멈춰 보입니다.

이런 무거운 작업은 Web Worker로 분리하거나, requestIdleCallback 등을 이용해 분할 처리해야 합니다.

requestAnimationFrame

requestAnimationFrame은 매크로태스크와 마이크로태스크 사이의 특수한 타이밍에 실행되는 렌더링 전용 콜백입니다. 애니메이션을 부드럽게 만들기 위해 사용합니다.

Node.js의 이벤트 루프

Node.js의 이벤트 루프는 브라우저와 비슷하지만, 단계(phase) 라는 개념이 추가됩니다.

  1. timers (setTimeout, setInterval)
  2. pending callbacks
  3. idle, prepare
  4. poll (I/O)
  5. check (setImmediate)
  6. close callbacks

각 단계 사이에서 마이크로태스크 큐가 비워집니다.

마무리

이벤트 루프는 JavaScript의 비동기 실행 모델을 이해하는 핵심입니다. 콜 스택, 태스크 큐, 마이크로태스크 큐의 우선순위를 정확히 알면 setTimeout(fn, 0)이 즉시 실행되지 않는 이유나 Promise가 빠르게 처리되는 이유를 명쾌하게 설명할 수 있습니다. 무거운 작업이 메인 스레드를 막지 않도록 설계하는 것이 사용자 경험의 핵심입니다.
