Shell 스크립트 기초 가이드

작성일 2026-05-25 Edited on 2026-06-15 In System , OS
리눅스 자동화의 핵심인 Bash 셸 스크립트의 문법과 실무 패턴을 정리합니다.

들어가며

서버 운영, 빌드 자동화, 배포 스크립트 등 리눅스 환경에서 반복 작업을 처리할 때 가장 많이 사용하는 도구가 셸 스크립트(Shell Script) 입니다. 이 글에서는 Bash를 기준으로 변수, 조건문, 반복문, 함수, 그리고 실무에서 자주 쓰이는 패턴을 정리합니다.

첫 스크립트 작성하기

스크립트 파일은 보통 .sh 확장자를 사용하며, 첫 줄에 셸을 지정하는 shebang을 적습니다.

실행 권한을 부여하고 실행합니다.

변수와 출력

  • 변수 선언 시 = 양쪽에 공백이 없어야 함
  • 변수 사용은 $변수명 또는 ${변수명}
  • ${변수명}은 변수 경계가 모호할 때 안전함

숫자 비교 문자열 비교 파일 검사
-eq (같음) = (같음) -e (존재)
-ne (다름) != (다름) -f (파일)
-lt (<) -z (빈 문자열) -d (디렉토리)
-gt (>) -n (빈 문자열 아님) -r (읽기 가능)
-le (<=) -w (쓰기 가능)
-ge (>=) -x (실행 가능)

종료 코드와 에러 처리

명령어의 종료 코드는 $?로 확인할 수 있습니다. 0은 성공, 0이 아니면 실패입니다.

set -euo pipefail은 견고한 스크립트의 표준 시작입니다.

디버깅 팁

추가로 ShellCheck 같은 정적 분석 도구를 활용하면 흔한 실수를 빠르게 찾을 수 있습니다.

마무리

셸 스크립트는 간단해 보이지만 자동화의 강력한 도구입니다. 변수, 조건문, 반복문, 함수 같은 기본 문법에 더해 set -euo pipefail, 적절한 에러 처리, ShellCheck 같은 도구를 활용하면 안정적인 스크립트를 작성할 수 있습니다. 작은 자동화부터 시작해 점차 복잡한 워크플로를 스크립트로 옮겨가 보세요.
