CSS 애니메이션 기초 - Transition과 Keyframes

작성일 2026-05-21 Edited on 2026-05-12 In Frontend , CSS
CSS transition과 @keyframes의 차이와 사용법, 그리고 성능 좋은 애니메이션을 만드는 팁을 정리합니다.

CSS 애니메이션 개요

웹에서 움직임은 사용자에게 상태 변화를 알려주고 시선을 유도합니다. CSS는 JavaScript 없이도 강력한 애니메이션을 만들 수 있는 두 가지 도구를 제공합니다.

  • Transition: 두 상태(A -> B) 사이를 부드럽게 보간
  • Animation + @keyframes: 여러 단계의 복잡한 움직임을 정의

이 글에서는 두 가지의 사용법과 성능 고려사항을 살펴봅니다.

Transition 기본

transition은 속성이 변할 때 변화를 부드럽게 만듭니다.

transition의 단축 속성은 네 가지를 받습니다.

속성 설명 예시
transition-property 어떤 속성을 애니메이트할지 opacity, all
transition-duration 지속 시간 300ms, 0.5s
transition-timing-function 가속 곡선 ease, linear, cubic-bezier()
transition-delay 지연 시간 100ms

Timing Function

가속 곡선은 동작의 인상을 크게 좌우합니다.

자연스러운 UI에는 ease-out이 무난하고, 튕기는 느낌은 cubic-bezier로 직접 조정합니다.

@keyframes로 복잡한 애니메이션

여러 단계의 움직임은 @keyframes로 정의하고 animation 속성으로 적용합니다.

키프레임 중간 단계도 정의할 수 있습니다.

animation의 주요 옵션은 다음과 같습니다.

Transition vs Animation

항목 Transition Animation
트리거 상태 변화(hover, class 추가 등) 자동 시작 가능
키프레임 시작/끝 두 지점 임의의 단계 정의
반복 불가 가능
적합한 상황 단순한 상태 전환 복잡한 움직임, 무한 반복

성능 좋은 애니메이션

브라우저가 애니메이션을 처리할 때, GPU 가속을 받는 속성과 그렇지 않은 속성이 있습니다.

속성 비용 권장
transform 매우 낮음 (Composite) 권장
opacity 매우 낮음 (Composite) 권장
width, height, top, left 높음 (Layout) 피하기
background-color, box-shadow 중간 (Paint) 가능하면 피하기

will-change 속성으로 브라우저에 힌트를 줄 수 있지만, 남용하면 메모리 사용이 늘어나니 꼭 필요한 곳에만 사용합니다.

접근성 - prefers-reduced-motion

전정 장애 등으로 움직임에 민감한 사용자를 위해 OS 설정을 존중해야 합니다.

실용 예제 - 로딩 스피너

모범 사례

  • 가능하면 transform과 opacity만 사용해 60fps를 유지합니다.
  • 200~400ms 정도가 UI 전환에 자연스러운 범위입니다.
  • 너무 화려한 애니메이션은 오히려 인지 부하를 늘립니다.
  • prefers-reduced-motion을 반드시 처리해 접근성을 보장합니다.

마무리

CSS만으로도 충분히 자연스럽고 성능 좋은 애니메이션을 만들 수 있습니다. 단순한 인터랙션은 transition, 복잡하거나 반복적인 움직임은 keyframes를 사용하고, GPU 친화적인 속성을 활용하는 것이 핵심입니다. 작은 디테일이 사용자 경험의 큰 차이를 만든다는 점을 기억하세요.
