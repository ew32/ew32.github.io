Core Web Vitals 이해와 최적화

작성일 2026-05-22 Edited on 2026-05-20 In Frontend , JavaScript
Google이 정의한 핵심 웹 성능 지표 Core Web Vitals의 개념과 실전 최적화 방법을 다룹니다.

Core Web Vitals란?

Core Web Vitals는 Google이 정의한 웹 사용자 경험 측정 지표로, 로딩 성능, 상호작용성, 시각적 안정성을 평가합니다. 2021년부터는 검색 랭킹 요소로도 반영되어 SEO 관점에서도 중요한 지표가 되었습니다.

현재 Core Web Vitals는 세 가지 핵심 지표로 구성됩니다.

  • LCP (Largest Contentful Paint): 로딩 성능
  • INP (Interaction to Next Paint): 상호작용 응답성 (2024년 FID 대체)
  • CLS (Cumulative Layout Shift): 시각적 안정성

LCP - 최대 콘텐츠 표시 시간

LCP는 뷰포트 내에서 가장 큰 콘텐츠 요소가 렌더링되는 시점을 측정합니다. 사용자가 페이지를 “로드되었다”라고 느끼는 순간을 정량화한 값입니다.

등급 LCP 값
Good 2.5초 이하
Needs Improvement 2.5초 ~ 4.0초
Poor 4.0초 초과

LCP 측정을 위한 PerformanceObserver 예시입니다.

LCP 최적화 방법

  1. 이미지 우선순위 지정: fetchpriority="high" 사용
  2. 프리로드: <link rel="preload">로 핵심 리소스 미리 로드
  3. 서버 응답 시간 단축: TTFB 개선
  4. 렌더링 차단 리소스 제거

INP - 다음 페인트까지의 상호작용 시간

INP는 사용자가 페이지에서 수행한 모든 상호작용(클릭, 탭, 키 입력)의 응답성을 측정합니다. 가장 느린 상호작용을 기준으로 평가하며, 200ms 이하가 권장됩니다.

INP 최적화 전략

  • 긴 작업을 작은 단위로 분할 (scheduler.yield() 활용)
  • requestIdleCallback으로 비필수 작업 지연
  • React 18+의 useTransition으로 우선순위 분리
  • 디바운스/스로틀로 이벤트 빈도 제어

CLS - 누적 레이아웃 이동

CLS는 페이지 로드 중 예상치 못한 레이아웃 이동의 합을 측정합니다. 사용자가 버튼을 누르려는 순간 페이지가 움직여 다른 요소를 클릭하게 되는 상황을 방지하기 위한 지표입니다.

CLS 점수 = impact fraction × distance fraction

등급 CLS 값
Good 0.1 이하
Needs Improvement 0.1 ~ 0.25
Poor 0.25 초과

CLS 최적화 방법

  1. 이미지/비디오에 width, height 명시
  2. 광고 슬롯에 미리 공간 예약
  3. font-display: optional 사용
  4. 동적 콘텐츠는 기존 콘텐츠 위에 삽입 금지

측정 도구

Core Web Vitals는 다양한 방식으로 측정할 수 있습니다.

  • Lighthouse: Chrome DevTools 내장, 랩 데이터
  • PageSpeed Insights: 필드 + 랩 데이터 결합
  • Chrome UX Report (CrUX): 실제 사용자 데이터
  • web-vitals 라이브러리: 실시간 측정

베스트 프랙티스

  • 핵심 리소스는 항상 preload, 비핵심은 prefetch
  • 이미지는 WebP/AVIF 사용, 차원 명시
  • 폰트는 self-host + font-display: swap
  • 자바스크립트 번들은 코드 스플리팅으로 최소화
  • 서드파티 스크립트는 async 또는 defer로 로드

마무리

Core Web Vitals는 단순한 성능 지표를 넘어 사용자 경험과 비즈니스 성과에 직결됩니다. LCP, INP, CLS 세 지표를 지속적으로 모니터링하고 점진적으로 개선해 나가면, 사용자 만족도와 검색 노출 모두를 효과적으로 향상시킬 수 있습니다.
