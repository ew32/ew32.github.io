TCP와 UDP 프로토콜 비교

작성일 2026-05-23 Edited on 2026-07-20 In Network , Theory
전송 계층의 양대 프로토콜인 TCP와 UDP의 차이점과 사용 시나리오를 정리합니다.

전송 계층의 두 프로토콜

**TCP(Transmission Control Protocol)**와 **UDP(User Datagram Protocol)**는 OSI 7계층 모델에서 **전송 계층(Layer 4)**에 속하는 가장 대표적인 프로토콜입니다. 두 프로토콜은 IP 위에서 동작하며, 응용 계층 데이터를 어떻게 전송할지에 대한 서로 다른 접근 방식을 제공합니다.

  • TCP: 신뢰성, 순서 보장, 연결 지향
  • UDP: 빠른 속도, 단순함, 비연결성

TCP의 특징

TCP는 신뢰성 있는 통신을 위해 설계된 연결 지향 프로토콜입니다. 모든 패킷이 순서대로, 누락 없이 전달되도록 보장합니다.

3-Way Handshake

TCP 연결은 다음 3단계로 시작됩니다.

  1. SYN: 클라이언트가 연결 요청과 초기 시퀀스 번호 전송
  2. SYN-ACK: 서버가 응답과 자신의 시퀀스 번호 전송
  3. ACK: 클라이언트가 최종 확인

4-Way Handshake (연결 종료)

신뢰성 보장 메커니즘

  • 시퀀스 번호: 패킷 순서 보장
  • 확인 응답(ACK): 수신 확인
  • 재전송: 일정 시간 내 ACK이 없으면 재전송
  • 흐름 제어: 수신자의 윈도우 크기에 맞춰 전송량 조절
  • 혼잡 제어: 네트워크 상태에 따라 전송 속도 조절 (Slow Start, Congestion Avoidance)

TCP 헤더 구조

필드 크기 설명
Source Port 16 bit 출발지 포트
Dest. Port 16 bit 목적지 포트
Sequence Number 32 bit 시퀀스 번호
Ack Number 32 bit 확인 응답 번호
Flags 9 bit SYN, ACK, FIN 등
Window 16 bit 윈도우 크기
Checksum 16 bit 오류 검출

총 헤더 크기는 최소 20바이트입니다.

UDP의 특징

UDP는 단순함과 속도에 집중한 비연결 지향 프로토콜입니다. 핸드셰이크 없이 즉시 데이터를 전송합니다.

UDP 헤더 구조

필드 크기 설명
Source Port 16 bit 출발지 포트
Dest. Port 16 bit 목적지 포트
Length 16 bit 전체 길이
Checksum 16 bit 오류 검출

총 헤더 크기는 8바이트로, TCP에 비해 매우 작습니다.

UDP의 특성

  • 연결 설정 없음: 즉시 전송 시작
  • 순서 보장 없음: 패킷이 뒤바뀌어 도착 가능
  • 재전송 없음: 손실되어도 신경 쓰지 않음
  • 흐름 제어 없음: 수신자 상태와 무관하게 전송
  • 브로드캐스트/멀티캐스트 지원

TCP vs UDP 비교표

항목 TCP UDP
연결 방식 연결 지향 비연결 지향
신뢰성 보장 미보장
순서 보장 보장 미보장
속도 느림 빠름
헤더 크기 20바이트 이상 8바이트
흐름 제어 있음 없음
혼잡 제어 있음 없음
브로드캐스트 미지원 지원

사용 시나리오

TCP가 적합한 경우

  • HTTP/HTTPS: 웹 브라우징
  • 이메일 (SMTP, IMAP, POP3)
  • 파일 전송 (FTP, SFTP)
  • 데이터베이스 연결
  • SSH 원격 접속

데이터의 정확성이 속도보다 중요한 모든 경우에 TCP를 선택합니다.

UDP가 적합한 경우

  • 실시간 스트리밍: 비디오/오디오 (일부 손실 허용)
  • 온라인 게임: 낮은 지연 시간 우선
  • DNS 질의: 단발성 짧은 요청
  • VoIP: 음성 통화
  • DHCP: IP 주소 할당
  • QUIC/HTTP/3 기반: 신뢰성은 응용 계층에서 구현

QUIC와 미래

최근에는 UDP 위에서 신뢰성을 보장하는 QUIC 프로토콜이 등장했습니다. HTTP/3의 기반이며, TCP의 신뢰성과 UDP의 빠른 연결 설정을 결합한 형태입니다. 이는 “TCP는 느리니까 UDP를 쓰자”라는 단순 비교가 점차 무의미해지고 있음을 의미합니다.

마무리

TCP와 UDP는 단순히 빠르고 느린 것이 아니라, 목적에 따라 적합한 도구가 다릅니다. 데이터 정확성이 핵심이라면 TCP, 실시간성과 단순함이 핵심이라면 UDP를 선택하면 됩니다. 최근에는 QUIC처럼 두 장점을 모두 취하는 하이브리드 접근도 늘어나고 있습니다.
