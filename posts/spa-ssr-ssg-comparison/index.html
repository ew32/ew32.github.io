SPA vs SSR vs SSG 렌더링 방식 비교

작성일 2026-05-20 Edited on 2026-05-15 In Frontend , JavaScript
클라이언트 사이드 렌더링, 서버 사이드 렌더링, 정적 사이트 생성의 동작 원리와 장단점을 비교합니다.

웹 렌더링 방식 개요

웹 페이지를 사용자에게 보여주는 방식은 시간이 지나면서 다양하게 발전해왔습니다. 같은 React/Vue 코드라도 언제, 어디서 HTML이 생성되는가에 따라 성능, SEO, 사용자 경험이 크게 달라집니다.

주요 세 가지 방식은 다음과 같습니다.

  • SPA (Single Page Application): 클라이언트(브라우저)에서 렌더링
  • SSR (Server Side Rendering): 매 요청마다 서버에서 렌더링
  • SSG (Static Site Generation): 빌드 시점에 미리 HTML 생성

SPA - 클라이언트 사이드 렌더링

브라우저가 거의 빈 HTML과 JavaScript 번들을 받아온 뒤, JS가 실행되며 화면을 그리는 방식입니다. 페이지 전환도 JS가 처리하므로 새로고침 없이 빠르게 이동할 수 있습니다.

  • 장점: 한 번 로드한 뒤 페이지 전환이 빠름, 풍부한 상호작용, 서버 부담 적음
  • 단점: 초기 로딩이 느림(번들 다운로드 + 실행), 빈 HTML로 인한 SEO 불리, 저사양 기기에서 체감 성능 저하

대표 도구: Create React App, Vite + React/Vue (SSR 미사용 시)

SSR - 서버 사이드 렌더링

요청이 들어오면 서버에서 React/Vue를 실행하여 완성된 HTML을 응답합니다. 클라이언트는 즉시 콘텐츠를 보고, 이후 JS가 로드되며 상호작용 가능 상태(hydration)가 됩니다.

  • 장점: 첫 화면 표시 빠름(FCP), SEO 우수, 동적 데이터 즉시 반영
  • 단점: 매 요청마다 서버 연산 발생, 인프라 비용/복잡도 증가, TTFB가 느려질 수 있음

대표 도구: Next.js, Nuxt, Remix, SvelteKit

SSG - 정적 사이트 생성

빌드 시점에 모든 페이지의 HTML을 미리 생성해 CDN에 올려두는 방식입니다. 요청이 오면 정적 파일을 그대로 전송하므로 매우 빠릅니다.

  • 장점: 최고 수준의 응답 속도, 서버 비용 최소화, 보안성 우수, 캐싱 친화적
  • 단점: 빌드 시간 증가, 자주 바뀌는 데이터에 부적합, 사용자별 콘텐츠 한계

대표 도구: Next.js (정적 모드), Hexo, Astro, Gatsby, Hugo

ISR과 하이브리드

최근 프레임워크는 위 방식을 혼합해 사용합니다. 대표적인 패턴이 ISR(Incremental Static Regeneration) 으로, SSG로 만든 페이지를 일정 주기마다 백그라운드에서 재생성합니다.

또한 React Server Components는 페이지 단위가 아니라 컴포넌트 단위로 서버 렌더링을 결정할 수 있어, 한 페이지 안에서도 정적/동적 영역을 자유롭게 섞을 수 있습니다.

비교표

항목 SPA SSR SSG
HTML 생성 시점 브라우저 요청 시 서버 빌드 시
첫 화면 속도 느림 보통 매우 빠름
SEO 불리 유리 매우 유리
서버 비용 낮음 높음 낮음
동적 데이터 좋음 매우 좋음 제한적
빌드 시간 짧음 짧음 페이지 수에 비례

어떻게 선택할까?

  • 블로그, 문서, 마케팅 사이트: SSG (Hexo, Astro 등)
  • 뉴스, 커머스, 대시보드 일부: SSR + ISR
  • 로그인 후 관리자 페이지, SaaS 내부: SPA 또는 일부 SSR
  • 혼합된 요구사항: Next.js, Nuxt, SvelteKit으로 페이지별 최적 모드 선택

모범 사례

  • 페이지마다 SEO와 데이터 변경 빈도를 기준으로 모드를 결정합니다.
  • 정적으로 만들 수 있는 것은 정적으로 만듭니다.
  • 동적 콘텐츠는 캐시 헤더와 ISR로 최대한 캐싱합니다.
  • Core Web Vitals(LCP, INP, CLS)를 지속적으로 측정합니다.

마무리

SPA, SSR, SSG는 서로 대체재가 아니라 각자의 영역이 있는 도구입니다. 현대 프레임워크는 이들을 혼합하여 페이지/컴포넌트 단위로 최적의 렌더링을 선택할 수 있게 해줍니다. 본인 프로젝트의 콘텐츠 특성을 잘 분석해 적절한 방식을 조합해보세요.
