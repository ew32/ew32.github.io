메모리 관리 - 페이징과 세그멘테이션

작성일 2026-05-25 Edited on 2026-05-20 In System , OS
운영체제가 메모리를 효율적으로 관리하는 두 가지 핵심 기법인 페이징과 세그멘테이션을 비교 정리합니다.

들어가며

현대 운영체제에서 메모리는 물리적인 RAM 그대로 사용되지 않습니다. 운영체제는 가상 메모리(Virtual Memory) 라는 추상화를 통해 각 프로세스에게 독립된 메모리 공간을 제공합니다. 이 가상 메모리를 실제 물리 메모리에 매핑하는 대표적인 두 가지 기법이 페이징(Paging) 과 세그멘테이션(Segmentation) 입니다.

메모리 단편화 문제

연속 할당 방식(Contiguous Allocation)은 프로세스에게 메모리를 연속된 한 덩어리로 할당합니다. 시간이 지나면 다음 두 가지 문제가 발생합니다.

  • 외부 단편화(External Fragmentation): 사용 가능한 메모리는 충분하지만, 연속된 공간이 없어 할당하지 못함
  • 내부 단편화(Internal Fragmentation): 할당받은 공간 중 사용하지 않는 부분

이 문제를 해결하기 위해 비연속 할당 방식인 페이징과 세그멘테이션이 등장했습니다.

페이징(Paging)

페이징은 메모리를 고정 크기의 블록으로 나누어 관리합니다.

  • 페이지(Page): 가상 메모리의 고정 크기 블록 (보통 4KB)
  • 프레임(Frame): 물리 메모리의 고정 크기 블록 (페이지와 동일 크기)
  • 페이지 테이블(Page Table): 가상 페이지 → 물리 프레임 매핑 정보

가상 주소는 [페이지 번호 | 오프셋] 형태로 구성되며, MMU(Memory Management Unit)가 페이지 테이블을 조회해 물리 주소로 변환합니다.

페이징의 장단점

장점 단점
외부 단편화 없음 내부 단편화 발생 (마지막 페이지)
메모리 할당 단순 페이지 테이블 자체가 메모리 차지
가상 메모리 구현에 적합 주소 변환 오버헤드 (TLB로 완화)

세그멘테이션(Segmentation)

세그멘테이션은 메모리를 논리적인 단위(Code, Data, Stack, Heap)로 나누어 관리합니다. 각 세그먼트의 크기는 가변적입니다.

  • 세그먼트 테이블(Segment Table): 각 세그먼트의 시작 주소(Base)와 크기(Limit) 저장
  • 가상 주소는 [세그먼트 번호 | 오프셋] 형태

세그멘테이션의 장단점

장점 단점
프로그램 구조에 맞춤(코드, 데이터 등) 외부 단편화 발생
보호 및 공유 단위 명확 가변 크기로 관리 복잡
내부 단편화 없음 메모리 할당 비효율적일 수 있음

페이징 vs 세그멘테이션

항목 페이징 세그멘테이션
분할 단위 고정 크기 페이지 가변 크기 세그먼트
단편화 내부 단편화 외부 단편화
분할 기준 물리적 크기 논리적 의미
주소 형태 (페이지 번호, 오프셋) (세그먼트 번호, 오프셋)

Segmented Paging (혼합 방식)

실제 x86 아키텍처는 세그멘테이션 + 페이징을 모두 사용해 왔습니다. 세그먼트로 논리적 분리를 하고, 각 세그먼트 내부를 페이지로 관리합니다. 다만 현대 x86-64에서는 세그멘테이션이 거의 사용되지 않고, 페이징 중심으로 동작합니다.

TLB와 성능 최적화

페이징의 주소 변환은 페이지 테이블을 메모리에서 읽어와야 해 느립니다. 이를 해결하기 위해 TLB(Translation Lookaside Buffer) 라는 CPU 내부 캐시를 사용합니다.

  • TLB 히트(Hit): 빠르게 주소 변환 완료
  • TLB 미스(Miss): 메모리에서 페이지 테이블 읽어옴

다단계 페이지 테이블(Multi-level Page Table)은 페이지 테이블 자체의 메모리 사용을 줄여 줍니다.

페이지 폴트(Page Fault)

가상 메모리에서는 모든 페이지가 항상 메모리에 있는 것이 아닙니다. 접근하려는 페이지가 디스크(스왑 영역)에 있을 때 페이지 폴트가 발생합니다.

페이지 교체 알고리즘(FIFO, LRU, LFU 등)은 어떤 페이지를 디스크로 내보낼지 결정합니다.

마무리

페이징은 고정 크기로 메모리를 분할해 단순하고 효율적인 관리를 제공하며, 세그멘테이션은 프로그램의 논리 구조에 맞춰 메모리를 보호하고 공유하는 데 유리합니다. 현대 운영체제는 주로 페이징을 사용하며, TLB와 다단계 페이지 테이블, 페이지 교체 알고리즘 등으로 성능을 최적화합니다. 메모리 관리 기법을 이해하면 가상 메모리, 캐시, 시스템 성능 튜닝의 기반을 확실하게 다질 수 있습니다.
