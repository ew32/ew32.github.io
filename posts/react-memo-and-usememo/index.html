React.memo / useMemo / useCallback 제대로 쓰기

작성일 2026-06-06 Edited on 2026-07-20 In Frontend , JavaScript
React 렌더링 최적화 도구인 React.memo, useMemo, useCallback의 차이와 올바른 사용 시점을 정리합니다.

왜 메모이제이션이 필요한가

React는 부모가 리렌더링되면 자식도 함께 리렌더링되는 것이 기본입니다. 대부분의 경우 이 비용은 충분히 저렴하지만, 다음과 같은 상황에서는 최적화가 필요합니다.

  • 자식 컴포넌트가 무거운 계산을 수행할 때
  • 트리 깊이가 깊고 리스트 항목이 많을 때
  • 부모가 자주 리렌더되지만 자식에 전달되는 props가 실제로는 거의 변하지 않을 때

이때 사용하는 세 가지 도구가 React.memo, useMemo, useCallback입니다.

React.memo

React.memo는 컴포넌트 자체를 메모이제이션합니다. props가 얕은 비교(shallow compare)로 같으면 리렌더링을 건너뜁니다.

부모가 리렌더되어도 name, price가 동일하다면 Item은 다시 렌더되지 않습니다.

useMemo

useMemo는 값을 메모이제이션합니다. 의존성 배열의 값이 바뀔 때만 다시 계산합니다.

filter 비용이 큰 경우 효과적입니다. 단순히 객체 두 개 합치는 정도의 가벼운 연산에는 오히려 오버헤드가 됩니다.

useCallback

useCallback은 함수를 메모이제이션합니다. 의존성이 같으면 동일한 함수 참조를 반환합니다.

자식이 React.memo로 감싸져 있을 때, 함수 참조가 매 렌더마다 새로 만들어지면 메모이제이션이 깨집니다. useCallback은 이를 막아 줍니다.

자주 하는 실수

1. 모든 곳에 메모이제이션 적용

메모이제이션 자체에도 비용이 있습니다. 의존성 비교, 참조 저장에 메모리가 사용됩니다. 측정 없이 모든 함수와 값을 useCallback, useMemo로 감싸는 것은 오히려 손해입니다.

2. 의존성 누락

ESLint의 react-hooks/exhaustive-deps 규칙을 켜두는 것을 추천합니다.

3. React.memo + 객체/함수 prop

useMemo로 객체를 안정화하거나, primitive로 풀어서 전달해야 합니다.

React 컴파일러

React 19와 함께 도입된 React Compiler(이전 명칭 React Forget)는 빌드 타임에 자동으로 메모이제이션을 삽입합니다. 도입이 끝나면 수동 useMemo/useCallback 사용은 크게 줄어들지만, 원리는 그대로 이해해두는 것이 좋습니다.

마무리

React.memo, useMemo, useCallback은 만능 최적화 도구가 아니라 측정 후에 적용해야 효과를 보는 도구입니다. React DevTools Profiler로 실제 렌더 횟수를 확인한 뒤 병목 지점에만 선별적으로 적용하세요. 그리고 React 19의 컴파일러 자동화 흐름도 함께 따라가는 것이 좋습니다.
