Node.js Streams 가이드 - Readable, Writable, Transform

작성일 2026-06-02 Edited on 2026-05-26 In Backend
Node.js 스트림의 4가지 종류와 backpressure, pipeline 활용법을 실전 예제로 정리합니다.

왜 스트림인가

거대한 파일을 메모리에 한 번에 올리면 OOM이 발생합니다. 스트림은 데이터를 조각(chunk) 단위로 흘려보내며 처리하므로 메모리 사용량을 일정하게 유지할 수 있습니다. Node.js의 파일, 네트워크, 압축, 암호화 API들이 모두 스트림 기반입니다.

스트림의 4가지 종류

  • Readable: 데이터를 읽을 수 있는 소스 (fs.createReadStream, http.IncomingMessage)
  • Writable: 데이터를 쓸 수 있는 대상 (fs.createWriteStream, http.ServerResponse)
  • Duplex: 읽고 쓰기 모두 가능 (net.Socket)
  • Transform: 입력을 변환해 출력 (zlib, crypto)

Readable 사용

data 이벤트를 등록하면 flowing 모드, read()를 직접 호출하면 paused 모드로 동작합니다.

비동기 iteration

가장 깔끔한 소비 방법으로 권장됩니다.

Writable 사용

write()가 false를 반환하면 내부 버퍼가 가득 찼다는 신호입니다. drain 이벤트를 기다린 뒤 다시 써야 합니다. 이것이 backpressure 처리의 핵심입니다.

pipe와 pipeline

pipe는 readable의 출력을 writable의 입력으로 자동 연결하며 backpressure도 처리합니다.

다만 pipe는 에러 전파가 불완전하므로 실제 코드에서는 pipeline을 권장합니다.

중간 단계에서 에러가 발생해도 모든 스트림이 안전하게 정리됩니다.

Transform 만들기

CSV 파서, JSON Lines 파서, 한 줄씩 처리하는 로직 등 거의 모든 변환을 Transform으로 구현할 수 있습니다.

Object Mode

기본적으로 스트림은 Buffer/문자열을 다루지만 objectMode: true로 임의의 객체를 흘려보낼 수 있습니다.

Web Streams 호환

Node 18+는 표준 Web Streams API와 상호 변환을 지원합니다.

마무리

스트림은 처음엔 복잡해 보이지만 backpressure와 pipeline 두 개념만 잘 이해하면 곧 자연스럽게 다룰 수 있습니다. 대용량 데이터, 파일 변환, 프록시, 실시간 처리 등 Node.js의 강점을 살리는 영역에서 스트림은 거의 필수입니다.
