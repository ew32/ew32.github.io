VSCode Dev Containers로 일관된 개발환경 만들기

작성일 2026-07-18 Edited on 2026-05-19 In 개발환경
Docker 기반 Dev Containers로 팀원 모두 동일한 개발환경을 사용하도록 구성하는 방법을 정리합니다.

Dev Containers란

Dev Containers는 Docker 컨테이너를 통째로 개발환경으로 사용하는 VSCode 기능입니다. “내 컴퓨터에서는 됩니다” 문제를 근본적으로 해결하고, 새 팀원이 들어왔을 때 “VSCode에서 Reopen in Container”만 누르면 모든 의존성과 도구가 자동으로 준비됩니다.

내부적으로는 devcontainer.json 명세가 표준화되어 있어 GitHub Codespaces, JetBrains Gateway 등 다른 도구에서도 동일하게 동작합니다.

사전 준비

  • Docker Desktop 또는 Docker Engine
  • VSCode + Dev Containers 확장

기본 구조

프로젝트 루트에 .devcontainer/ 폴더를 만듭니다.

가장 단순한 설정

미리 만들어진 이미지를 그대로 사용하는 방식입니다.

VSCode에서 Dev Containers: Reopen in Container를 실행하면 이미지를 풀고, 확장을 자동 설치하고, npm install까지 완료한 환경이 열립니다.

Dockerfile로 커스텀

도구를 추가해야 한다면 자체 Dockerfile을 사용합니다.

Features로 모듈 추가

매번 Dockerfile에 작성하기 번거로운 도구는 Features로 한 줄 추가할 수 있습니다.

Docker Compose 통합

DB나 캐시 같은 보조 서비스가 필요하면 Compose와 결합합니다.

앱 컨테이너는 코드 작업용, db 컨테이너는 사이드카로 함께 뜹니다.

라이프사이클 명령

시점 명령 용도
onCreateCommand 컨테이너 최초 생성 글로벌 도구 설치
postCreateCommand 컨테이너 첫 시작 의존성 설치, 마이그레이션
postStartCommand 매 시작 서비스 기동
postAttachCommand VSCode 접속 시 환영 메시지, 워치 모드

성능 팁

  • macOS/Windows에서 볼륨 마운트가 느리다면 cached나 named volume 사용
  • 의존성 폴더(node_modules, .venv)는 컨테이너 내 named volume에 두면 IO 성능이 크게 향상
  • 이미지 빌드 캐시를 GitHub Container Registry에 푸시해 팀원이 재빌드 없이 사용

GitHub Codespaces 호환

같은 devcontainer.json이 Codespaces에서도 동작합니다. 로컬은 Docker로, 원격은 Codespaces로 자유롭게 전환할 수 있습니다.

마무리

Dev Containers는 “환경 셋업”이라는 비용을 한 번에 코드화합니다. 팀 합류 시간이 며칠에서 몇 분으로 줄고, 로컬 머신 오염 없이 여러 프로젝트를 자유롭게 운영할 수 있습니다. 작게 시작해 점진적으로 도구를 추가해 보세요.
