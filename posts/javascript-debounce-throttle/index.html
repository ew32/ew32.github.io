JavaScript 디바운스와 쓰로틀

작성일 2026-05-18 Edited on 2026-07-20 In Frontend , JavaScript
이벤트 핸들러 성능을 최적화하는 디바운스와 쓰로틀 기법의 차이와 구현을 정리합니다.

왜 디바운스와 쓰로틀이 필요한가

scroll, resize, mousemove, input 같은 이벤트는 매우 짧은 간격으로 빈번하게 발생합니다. 이 이벤트마다 무거운 콜백을 실행하면 브라우저가 멈추거나 서버에 과도한 요청이 가는 등 성능 문제가 발생합니다.

디바운스(debounce) 와 쓰로틀(throttle) 은 이러한 이벤트 호출 빈도를 제어하는 두 가지 핵심 기법입니다.

디바운스란?

디바운스는 연속된 호출을 묶어, 마지막 호출 이후 일정 시간이 지난 뒤에 한 번만 실행하는 방식입니다.

비유하자면 엘리베이터가 사람이 계속 타는 동안 문을 닫지 않다가, 한동안 새로운 사람이 없을 때 출발하는 것과 같습니다.

적합한 사례:

  • 검색창 자동완성 (타이핑이 끝났을 때 API 호출)
  • 윈도우 resize 종료 시점 감지
  • 폼 입력 유효성 검사

디바운스 구현

300ms 동안 타이핑이 멈추면 그 때 한 번만 콜백이 실행됩니다.

쓰로틀이란?

쓰로틀은 일정 시간 간격으로 최대 한 번씩만 실행되도록 제한하는 방식입니다. 호출이 아무리 자주 일어나도 정해진 주기를 보장합니다.

비유하자면 수도꼭지를 조절해 일정한 양만 흐르게 하는 것과 같습니다.

적합한 사례:

  • 무한 스크롤 (스크롤 위치 체크)
  • 게임 입력 처리
  • 마우스 드래그 좌표 추적
  • API 호출 제한

쓰로틀 구현

200ms마다 최대 한 번씩 실행됩니다.

타이머 기반 쓰로틀

setTimeout을 활용한 변형도 자주 사용됩니다.

이 방식은 마지막 호출도 보장된다는 차이가 있습니다.

디바운스 vs 쓰로틀 비교

항목 디바운스 쓰로틀
동작 방식 마지막 호출 이후 일정 시간 후 1회 실행 일정 간격마다 최대 1회 실행
호출 보장 마지막 호출만 보장 주기적 실행 보장
활용 예 검색 자동완성, resize 종료 스크롤, 마우스 이동
비유 엘리베이터 문 닫힘 수도꼭지 조절

즉시 실행 옵션 (leading/trailing)

실무에서는 첫 호출과 마지막 호출 처리 방식을 옵션화하는 경우가 많습니다.

leading: true이면 첫 호출 즉시 실행 후 후속 호출은 무시합니다.

라이브러리 활용

매번 직접 구현하기보다 Lodash의 _.debounce, _.throttle을 사용하는 것이 안전합니다. 엣지 케이스(cancel, flush, leading/trailing)를 잘 처리해 줍니다.

React에서의 활용

언마운트 시 cancel을 호출해 메모리 누수를 방지하는 것이 포인트입니다.

마무리

디바운스와 쓰로틀은 빈번한 이벤트로 인한 성능 저하를 막는 가장 기본적인 도구입니다. 검색은 디바운스, 스크롤은 쓰로틀이라는 식으로 패턴을 외워두면 빠르게 결정할 수 있습니다. 직접 구현해 보며 동작 원리를 이해한 뒤, 실무에서는 검증된 라이브러리를 활용하는 것이 가장 좋은 방법입니다.
