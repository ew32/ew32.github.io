TypeScript 유틸리티 타입 완벽 가이드

작성일 2026-05-26 Edited on 2026-05-20 In Frontend , JavaScript
Partial, Pick, Omit, Record, ReturnType 등 자주 쓰는 TypeScript 유틸리티 타입을 예제로 익힙니다.

유틸리티 타입이란

TypeScript는 자주 쓰이는 타입 변환을 내장 유틸리티 타입으로 제공합니다. 이를 잘 활용하면 반복적인 타입 정의를 줄이고 표현력을 높일 수 있습니다.

Partial - 모든 속성을 선택적으로

Required - 모든 속성을 필수로

Partial의 반대 동작입니다.

Readonly - 모든 속성을 읽기 전용으로

Pick<T, K> - 특정 속성만 선택

Omit<T, K> - 특정 속성 제외

Pick과 Omit은 DTO 정의에 매우 유용합니다.

Record<K, V> - 키-값 매핑 타입

Exclude / Extract - 유니온 필터링

NonNullable

null과 undefined를 제거합니다.

ReturnType / Parameters

함수의 반환 타입과 파라미터 타입을 추출합니다.

Awaited

Promise의 결과 타입을 추출합니다.

조합 활용 예제

여러 유틸리티 타입을 조합해 실전 타입을 만들 수 있습니다.

마무리

유틸리티 타입은 단순해 보이지만 잘 조합하면 코드베이스의 타입 일관성을 크게 향상시킵니다. 새로운 모델을 만들기 전에 기존 타입에서 유틸리티 타입으로 파생할 수 있는지 먼저 검토해보는 습관을 들이세요.
