ChatGPT 프롬프트 엔지니어링 기법

작성일 2026-05-27 Edited on 2026-06-15 In AI도구
ChatGPT에서 원하는 답변을 얻기 위한 프롬프트 엔지니어링 핵심 기법과 패턴을 정리합니다.

프롬프트 엔지니어링이란?

**프롬프트 엔지니어링(Prompt Engineering)**은 대규모 언어 모델(LLM)이 의도한 결과를 생성하도록 입력 텍스트를 설계하는 기술입니다. 동일한 모델이라도 프롬프트의 품질에 따라 결과의 정확도, 형식, 깊이가 크게 달라집니다.

프롬프트는 단순한 질문이 아니라 모델에게 역할, 문맥, 제약 조건, 출력 형식을 명확히 전달하는 일종의 명세서로 이해해야 합니다.

기본 원칙

좋은 프롬프트는 다음 요소를 포함합니다.

  1. 명확성(Clarity): 모호한 표현을 피하고 구체적으로 작성
  2. 문맥(Context): 배경 정보와 목적 제공
  3. 제약(Constraint): 길이, 톤, 형식 등의 조건 명시
  4. 예시(Example): 원하는 출력 형태를 보여주는 샘플 제공

다음 두 프롬프트를 비교해 봅시다.

Zero-shot, One-shot, Few-shot

LLM에 예시를 제공하는 방식에 따라 세 가지로 구분합니다.

방식 설명 사용 시점
Zero-shot 예시 없이 지시만 제공 모델이 잘 아는 일반 작업
One-shot 예시 1개 제공 출력 형식 통제 필요 시
Few-shot 예시 2개 이상 제공 복잡한 패턴 학습이 필요할 때

Few-shot 예시는 다음과 같이 작성합니다.

Chain-of-Thought (CoT)

복잡한 추론 문제에서 모델에게 단계별로 생각하도록 유도하면 정확도가 크게 향상됩니다. “단계별로 생각해 봅시다(Let’s think step by step)”라는 마법의 문장이 대표적입니다.

복잡한 코드 분석, 수학 문제, 논리 퍼즐에서 특히 효과적입니다.

역할 부여(Role Prompting)

모델에게 특정 페르소나를 부여하면 답변의 톤과 깊이가 달라집니다.

역할 부여는 다음과 같은 상황에서 유용합니다.

  • 특정 직무 관점의 조언이 필요할 때
  • 어조(공식적/캐주얼)를 조정하고 싶을 때
  • 도메인 전문성을 끌어올리고 싶을 때

출력 형식 지정

업무에 활용하려면 구조화된 출력이 필수입니다. JSON, 표, 마크다운 등을 명시적으로 요청하세요.

출력 형식을 강제하면 후속 자동화 파이프라인에 연결하기 쉽습니다.

실전 패턴

패턴 활용 예
요약 + 조건 “이 글을 5문장으로, 초등학생이 이해할 수 있게 요약”
단계 분할 “1단계: 문제 분석, 2단계: 해결책 제시, 3단계: 코드 작성”
반례 요청 “이 코드에서 발생 가능한 엣지 케이스를 5개 나열”
자체 검증 “답변 후 스스로 오류가 없는지 검토하고 수정”

잘 작동하지 않을 때

프롬프트가 원하는 결과를 내지 못한다면 다음을 점검하세요.

  • 지시가 한 번에 너무 많지 않은가? → 단계 분할
  • 예시가 부족하지 않은가? → Few-shot 추가
  • 출력 형식이 모호하지 않은가? → 템플릿 명시
  • 모델이 사실을 만들어내는가? → 출처 요구 또는 검색 기반 도구 사용

마무리

프롬프트 엔지니어링은 LLM 시대의 새로운 인터페이스 설계 기술입니다. 명확한 지시, 적절한 예시, 단계적 사고 유도, 구조화된 출력의 네 가지 원칙만 잘 적용해도 결과 품질이 비약적으로 향상됩니다. 반복적으로 실험하며 자신만의 프롬프트 템플릿 라이브러리를 구축해보세요.
