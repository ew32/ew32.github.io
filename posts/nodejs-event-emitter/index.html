Node.js EventEmitter 패턴 완벽 가이드

작성일 2026-06-01 Edited on 2026-05-18 In Backend
Node.js의 핵심 비동기 빌딩 블록인 EventEmitter의 동작 원리와 활용 패턴, 주의사항을 정리합니다.

EventEmitter란

EventEmitter는 Node.js의 events 모듈이 제공하는 클래스로, 객체가 이름이 있는 이벤트를 발생시키고 리스너를 등록할 수 있게 합니다. Node 코어의 많은 API(stream, http, process)가 이 클래스를 기반으로 합니다.

주요 메서드

  • on(event, listener): 리스너 등록
  • once(event, listener): 단 한 번만 호출되는 리스너
  • emit(event, ...args): 이벤트 발생
  • off(event, listener) / removeListener: 리스너 제거
  • removeAllListeners(event?): 모든 리스너 제거
  • listenerCount(event): 등록된 리스너 수

동기 호출 동작

emit은 등록된 리스너들을 등록 순서대로 동기적으로 호출합니다. 그래서 리스너에서 던진 에러는 즉시 전파됩니다.

비동기 후처리를 원한다면 setImmediate나 queueMicrotask로 래핑하세요.

error 이벤트

error 이벤트는 특별합니다. 리스너가 없으면 프로세스가 종료됩니다.

운영 환경에서는 EventEmitter 기반 객체에 반드시 error 핸들러를 등록해야 합니다.

클래스 상속

도메인 객체가 이벤트를 발생시키려면 EventEmitter를 상속합니다.

이렇게 하면 비즈니스 로직과 부가 동작(메일, 로깅, 분석)을 분리할 수 있습니다.

MaxListeners 경고

기본적으로 한 이벤트에 11개 이상 리스너가 등록되면 경고가 출력됩니다. 이는 메모리 누수 탐지를 위한 안전장치입니다.

루프 안에서 무심코 on을 호출하는 코드를 발견하기 좋은 신호입니다.

once를 Promise로

Node 11+는 events.once를 제공해 이벤트를 await할 수 있게 합니다.

on을 비동기 iterable로

이벤트 스트림을 for await...of로 소비할 수 있어 깔끔합니다.

활용 패턴

  • Pub/Sub: 모듈 간 느슨한 결합
  • Lifecycle hooks: 시작/종료 이벤트 발행
  • Progress reporting: 작업 진행률 전달
  • Domain Events: DDD에서 도메인 이벤트 전파

주의사항

  • 리스너 등록과 해제의 균형을 맞춰야 메모리 누수가 없습니다.
  • 무거운 동기 작업을 리스너에 두면 emit 호출자를 차단합니다.
  • 이벤트 이름은 문자열 상수나 enum으로 관리해 오타를 방지하세요.

마무리

EventEmitter는 단순해 보이지만 Node.js 생태계의 근간을 이루는 패턴입니다. 동기 emit, error 이벤트 규약, MaxListeners의 의미만 잘 이해해도 견고한 비동기 코드를 작성할 수 있습니다.
