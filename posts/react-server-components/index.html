React Server Components 이해하기

작성일 2026-06-04 Edited on 2026-05-19 In Frontend , JavaScript
React Server Components(RSC)의 동작 원리와 클라이언트 컴포넌트와의 차이, 도입 시 주의점을 정리합니다.

React Server Components란?

React Server Components(RSC)는 서버에서만 렌더링되며 클라이언트로 JavaScript 번들이 전송되지 않는 새로운 형태의 React 컴포넌트입니다. 기존의 Server-Side Rendering(SSR)이 HTML을 한 번 만들어 보낸 뒤 클라이언트에서 다시 하이드레이션하는 방식이라면, RSC는 컴포넌트 자체를 서버 전용으로 선언해 번들 크기를 근본적으로 줄입니다.

Next.js 13의 App Router 이후 RSC는 기본 동작이 되었으며, React 19에서는 안정 기능으로 자리 잡았습니다.

클라이언트 컴포넌트와의 차이

구분 Server Component Client Component
실행 위치 서버 브라우저
번들 포함 미포함 포함
State / Effect 불가 가능
데이터 접근 DB, 파일 시스템 직접 fetch / API 호출
이벤트 핸들러 사용 불가 사용 가능

클라이언트 컴포넌트는 파일 최상단에 "use client" 지시어를 명시해야 합니다.

데이터 패칭의 단순화

RSC에서는 서버에서 곧바로 async/await를 쓸 수 있어 데이터 패칭 코드가 단순해집니다. 더 이상 useEffect 안에서 fetch하지 않고, 컴포넌트 함수 자체를 비동기로 선언해 사용합니다.

직렬화 제약

서버 컴포넌트가 클라이언트 컴포넌트로 props를 전달할 때, 값은 반드시 직렬화 가능해야 합니다. 즉 함수, 클래스 인스턴스, Date 객체 등은 전달할 수 없습니다.

대신 children 패턴으로 서버 트리 안에 클라이언트 트리를 끼워 넣을 수 있습니다.

도입 시 주의점

  • 라이브러리가 RSC를 지원하지 않으면 "use client" 컴포넌트로 감싸야 합니다.
  • 환경 변수, DB 접근은 서버 컴포넌트에서만 사용해 비밀이 노출되지 않도록 합니다.
  • 클라이언트 상태(zustand, redux 등)는 클라이언트 트리 안에서만 동작합니다.
  • useState, useEffect를 쓰는 순간 해당 파일은 클라이언트 컴포넌트가 되어야 합니다.

마무리

React Server Components는 번들 크기를 줄이고 데이터 접근을 단순화하는 강력한 모델입니다. 단, 클라이언트와 서버의 경계를 명확히 이해하고 직렬화 규칙과 "use client" 지시어 사용법을 숙지해야 합니다. Next.js App Router로 작은 프로젝트부터 적용해 보면서 멘탈 모델을 익히는 것을 추천합니다.
