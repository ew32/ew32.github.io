Tailwind CSS 시작하기

작성일 2026-05-21 Edited on 2026-07-20 In Frontend , CSS
유틸리티 우선 CSS 프레임워크 Tailwind CSS의 핵심 개념과 설치, 반응형, 커스터마이징을 소개합니다.

Tailwind CSS란?

Tailwind CSS는 미리 정의된 유틸리티 클래스를 조합하여 디자인을 만드는 Utility-First CSS 프레임워크입니다. Bootstrap처럼 완성된 컴포넌트를 제공하는 대신, flex, p-4, text-center 같은 작은 단위의 클래스를 마크업에 직접 붙여 스타일을 구성합니다.

처음에는 클래스가 많아 보여 거부감이 있을 수 있지만, 익숙해지면 다음과 같은 장점을 누릴 수 있습니다.

  • CSS 파일을 거의 작성하지 않아도 됨
  • 클래스명 고민에서 해방
  • 디자인 토큰이 일관됨
  • 사용하지 않는 클래스는 빌드 시 제거되어 매우 작은 CSS 결과물

설치와 기본 설정

Tailwind는 PostCSS 또는 자체 CLI로 설치할 수 있습니다. Vite 기반 프로젝트 기준의 예시입니다.

tailwind.config.js에서 콘텐츠 경로를 지정합니다.

CSS 진입점에 디렉티브를 추가합니다.

유틸리티 클래스 기본

다음은 카드 컴포넌트를 Tailwind로 작성한 예시입니다.

각 클래스가 무엇을 하는지 명확해 별도 CSS 파일을 열어보지 않아도 이해됩니다.

카테고리 예시
여백 m-4, mt-2, px-6, py-3
색상 bg-blue-500, text-gray-700
폰트 text-lg, font-semibold
레이아웃 flex, grid, grid-cols-3
크기 w-full, h-12, max-w-md
효과 shadow-md, rounded-lg, opacity-50

반응형 디자인

sm:, md:, lg:, xl:, 2xl: 접두어로 모바일 우선 반응형을 구성합니다.

기본 브레이크포인트는 다음과 같습니다.

접두어 최소 너비
sm 640px
md 768px
lg 1024px
xl 1280px
2xl 1536px

상태 변형

hover, focus, dark 모드 등도 접두어로 표현합니다.

다크 모드는 tailwind.config.js에서 darkMode: 'class'로 두고 최상위에 <html class="dark">를 토글하는 방식이 일반적입니다.

커스터마이징

디자인 토큰은 theme.extend에서 확장합니다.

@apply와 컴포넌트 추출

반복되는 클래스 조합은 @apply로 묶을 수 있습니다.

다만 React/Vue 같은 컴포넌트 기반 프레임워크에서는 컴포넌트로 추출하는 편이 더 자연스럽습니다.

기존 방식과의 비교

항목 Tailwind 전통 CSS/SCSS
클래스명 고민 거의 없음 필요
CSS 파일 크기 작음 (purge) 점점 커짐
마크업 길이 길어짐 짧음
디자인 일관성 토큰 기반 자동 별도 관리
학습 비용 클래스 명세 학습 일반 CSS

모범 사례

  • 반복되는 패턴은 컴포넌트로 추출합니다.
  • 디자인 토큰을 tailwind.config.js에서 정의하여 일관성을 유지합니다.
  • 너무 긴 클래스 목록은 clsx, cn 같은 유틸로 정리합니다.
  • 임의 값은 w-[37px] 같은 임의 값 표기를 적절히 활용하되 남용은 피합니다.

마무리

Tailwind CSS는 처음에는 낯설지만 익숙해지면 개발 속도와 일관성을 모두 잡을 수 있는 강력한 도구입니다. 디자인 시스템이 명확하고 컴포넌트 기반 프로젝트라면 잘 어울리는 선택입니다. 작은 프로젝트로 한번 도입해본 뒤 본인 스타일에 맞는지 판단해보세요.
