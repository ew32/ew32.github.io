AWS S3 활용 가이드

작성일 2026-05-26 Edited on 2026-06-05 In Cloud , Infrastructure
AWS S3의 주요 개념과 보안 설정, 비용 최적화 전략을 실무 관점에서 정리합니다.

들어가며

Amazon S3(Simple Storage Service) 는 AWS의 객체 스토리지 서비스로, 거의 모든 클라우드 워크로드에서 활용됩니다. 정적 웹사이트 호스팅, 데이터 레이크, 백업, 빅데이터 분석 등 다양한 용도로 쓰이며, 11 nines(99.999999999%)의 내구성을 제공합니다. 이 글에서는 S3의 핵심 개념과 실무에서 자주 사용하는 기능을 정리합니다.

S3의 핵심 개념

  • Bucket: 객체를 담는 컨테이너. 글로벌하게 유일한 이름을 가져야 함
  • Object: 저장되는 파일 단위. 최대 5TB
  • Key: 객체의 고유 식별자 (folder/subfolder/file.jpg)
  • Region: 버킷이 물리적으로 위치한 리전

S3는 파일 시스템처럼 보이지만 실제로는 플랫한 키-값 저장소입니다. folder/file.txt의 folder/는 prefix일 뿐 진짜 디렉토리가 아닙니다.

스토리지 클래스

S3는 사용 패턴에 따라 비용을 최적화할 수 있는 여러 스토리지 클래스를 제공합니다.

클래스 용도 특징
Standard 자주 접근 기본값, 가장 비쌈
Standard-IA 가끔 접근 저장 비용 저렴, 검색 비용 발생
One Zone-IA 가끔 접근, 단일 AZ Standard-IA보다 20% 저렴
Intelligent-Tiering 패턴 모름 자동으로 계층 이동
Glacier Instant 아카이브, 즉시 접근 분기 1회 접근 정도
Glacier Flexible 아카이브 분~시간 단위 복원
Glacier Deep Archive 장기 보관 가장 저렴, 12시간 복원

기본 명령어 (AWS CLI)

보안: IAM, 버킷 정책, ACL

S3는 여러 보안 계층을 제공합니다.

IAM 정책

사용자/역할 단위로 권한을 부여합니다.

버킷 정책

버킷 단위로 권한을 정의합니다. 특정 IP만 허용하거나, 공개 읽기 권한을 줄 때 사용합니다.

퍼블릭 액세스 차단

실수로 데이터가 공개되는 사고를 막기 위해 Block Public Access 설정을 켜두는 것이 기본입니다.

암호화

전송 중 암호화(HTTPS)는 기본이고, 저장 시 암호화는 다음 옵션이 있습니다.

  • SSE-S3: S3 관리 키 (기본)
  • SSE-KMS: AWS KMS 관리 키 (감사 로그 가능)
  • SSE-C: 고객 제공 키
  • 클라이언트 측 암호화: 업로드 전 직접 암호화

버저닝과 라이프사이클

버저닝

같은 키의 객체를 덮어써도 이전 버전이 보존됩니다.

라이프사이클 규칙

객체를 자동으로 다른 클래스로 이동하거나 삭제합니다.

정적 웹사이트 호스팅

S3는 정적 웹사이트(HTML, CSS, JS)를 직접 호스팅할 수 있습니다.

운영 환경에서는 CloudFront(CDN)와 결합해 HTTPS와 글로벌 캐싱을 함께 사용합니다.

Presigned URL

비공개 객체에 임시 접근 권한을 부여하는 방법입니다.

비용 최적화 팁

  • 적절한 스토리지 클래스 선택: 접근 패턴에 따라 IA, Glacier 활용
  • 라이프사이클 규칙: 자동 전환과 삭제
  • 불완전한 멀티파트 업로드 정리: 라이프사이클로 자동 삭제
  • S3 Storage Lens: 사용 패턴 분석
  • CloudFront 캐싱: 데이터 전송 비용 절감
  • VPC Endpoint: NAT 게이트웨이 비용 절감

마무리

S3는 단순한 파일 저장소를 넘어 데이터 레이크, 백업, 정적 호스팅, 분석 등 다양한 워크로드의 기반이 됩니다. 보안(IAM, 버킷 정책, 암호화), 비용(스토리지 클래스, 라이프사이클), 가용성(버저닝, 복제) 측면을 모두 고려해 설계해야 합니다. 처음에는 기본 옵션으로 시작하고, 사용량이 늘어나면 모니터링과 최적화를 점진적으로 적용하는 전략을 추천합니다.
