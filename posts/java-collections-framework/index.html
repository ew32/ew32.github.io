Java 컬렉션 프레임워크 완벽 정리 - List/Set/Map/Queue

작성일 2026-04-05 Edited on 2026-07-20 In Backend , Java
Java 컬렉션 프레임워크의 구조, 주요 구현체별 특징과 시간복잡도, 동기화 컬렉션, 불변 컬렉션까지 실무 선택 기준과 함께 정리합니다.

컬렉션 프레임워크 개요

Java 컬렉션 프레임워크(JCF)는 java.util 패키지의 자료구조 모음입니다. 핵심 인터페이스는 다음 네 가지입니다.

  • List — 순서가 있고 중복을 허용하는 시퀀스
  • Set — 중복을 허용하지 않는 집합
  • Queue / Deque — FIFO/LIFO 처리에 적합한 대기열
  • Map — 키-값 쌍 (엄밀히는 Collection을 상속하지 않지만 프레임워크에 포함)

모든 컬렉션은 Iterable → Collection을 정점으로 하는 계층 안에 있습니다. 어떤 구현을 선택하느냐가 성능과 동작을 결정합니다.

List 구현체

ArrayList

내부적으로 동적 배열을 사용합니다.

  • 인덱스 접근: O(1)
  • 끝에 추가: 평균 O(1), 리사이즈 시 O(n)
  • 중간 삽입·삭제: O(n)

일반적인 용도에서 가장 많이 쓰입니다. 초기 용량을 예측할 수 있다면 new ArrayList<>(capacity)로 생성해 리사이즈 비용을 줄이세요.

LinkedList

이중 연결 리스트 구현입니다.

  • 인덱스 접근: O(n)
  • 양 끝 삽입·삭제: O(1)

이름은 List지만 인덱스 접근이 느려서 List 용도로는 권장되지 않습니다. Deque(양쪽 끝 큐) 용도가 더 적합합니다. 실제 사용 시 ArrayDeque가 더 나은 선택일 수 있습니다.

Set 구현체

HashSet

내부적으로 HashMap을 사용합니다.

  • 추가·조회·삭제: 평균 O(1)
  • 순서 보장 없음

해시 충돌이 심하면 성능이 떨어지므로 equals와 hashCode를 올바르게 구현하는 것이 중요합니다.

LinkedHashSet

HashSet에 삽입 순서를 유지하는 연결 리스트를 추가한 구현입니다. 순서가 필요하면서 빠른 조회도 필요할 때 선택합니다.

TreeSet

Red-Black Tree(균형 이진 탐색 트리) 기반입니다.

  • 추가·조회·삭제: O(log n)
  • 정렬된 순서로 순회 가능

범위 검색(subSet, headSet, tailSet)이나 정렬된 순회가 필요할 때 적합합니다.

Map 구현체

HashMap

키의 해시 기반으로 동작합니다.

  • 조회·삽입·삭제: 평균 O(1)
  • 순서 보장 없음, null 키 1개 허용

Java 8부터는 해시 충돌이 일정 임계치(기본 8)를 넘으면 버킷 내부가 트리(Tree) 구조로 전환되어 최악의 경우도 O(log n)이 됩니다.

LinkedHashMap

삽입 순서나 접근 순서를 유지할 수 있습니다. accessOrder=true 옵션과 함께 removeEldestEntry를 오버라이드하면 LRU 캐시를 구현할 수 있습니다.

TreeMap

키 정렬을 유지합니다. NavigableMap 인터페이스로 범위 연산 제공.

ConcurrentHashMap

멀티스레드 환경에서 사용하는 Map. 세그먼트/버킷 단위 락으로 동시성을 확보하며, 일반적으로 Collections.synchronizedMap()보다 성능이 우수합니다.

Queue / Deque 구현체

ArrayDeque

배열 기반의 양방향 큐. 스택 대용으로도 적합하며, 옛 Stack 클래스보다 빠릅니다.

PriorityQueue

힙(Heap) 기반의 우선순위 큐. 가장 작은 원소(또는 Comparator 기준 우선순위가 높은 원소)를 O(log n)에 꺼냅니다.

BlockingQueue

스레드 안전한 큐. LinkedBlockingQueue, ArrayBlockingQueue 등이 있으며 생산자-소비자 패턴에 사용됩니다.

시간복잡도 요약

컬렉션 get/contains add remove 순서
ArrayList O(1) / O(n) 끝 O(1) O(n) 삽입 순서
LinkedList O(n) 양 끝 O(1) 양 끝 O(1) 삽입 순서
HashSet O(1) O(1) O(1) 없음
LinkedHashSet O(1) O(1) O(1) 삽입/접근
TreeSet O(log n) O(log n) O(log n) 정렬
HashMap O(1) O(1) O(1) 없음
TreeMap O(log n) O(log n) O(log n) 키 정렬
ArrayDeque - O(1) O(1) FIFO/LIFO
PriorityQueue peek O(1) O(log n) O(log n) 우선순위

동기화 컬렉션

Collections.synchronizedList(list)처럼 래퍼로 만들 수도 있지만 성능이 떨어집니다. 동시 접근이 많다면 다음을 우선 고려하세요.

  • ConcurrentHashMap (Map)
  • CopyOnWriteArrayList — 읽기 위주, 쓰기 매우 적을 때
  • ConcurrentLinkedQueue, LinkedBlockingQueue (Queue)

불변 컬렉션 (Java 9+)

생성된 컬렉션은 수정 시 UnsupportedOperationException을 던집니다. null도 허용하지 않습니다. 방어 복사용으로 List.copyOf(other)도 유용합니다.

실무 선택 가이드

  • 순서 있는 일반 리스트 → ArrayList
  • 자주 양 끝에서 삽입/삭제 → ArrayDeque
  • 중복 제거 + 빠른 조회 → HashSet
  • 정렬된 집합/범위 검색 → TreeSet
  • 빠른 키-값 매핑 → HashMap
  • 삽입 순서 유지 매핑 → LinkedHashMap
  • 멀티스레드 키-값 → ConcurrentHashMap
  • 우선순위 큐잉 → PriorityQueue
  • 생산자-소비자 → BlockingQueue 계열

마무리

컬렉션은 자바 프로그램의 기본기입니다. 인터페이스(List, Set, Map)에 코드를 의존시키고, 상황에 맞는 구현체를 골라 끼우는 것이 좋은 설계입니다. 성능 문제가 의심된다면 가장 먼저 의심해야 할 곳이 컬렉션 선택과 초기 용량입니다.
