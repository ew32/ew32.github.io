Terraform 기초 - IaC로 인프라 관리하기

작성일 2026-05-26 Edited on 2026-05-18 In Cloud , Infrastructure
HashiCorp Terraform을 활용해 인프라를 코드로 관리하는 핵심 개념과 실전 사용법을 정리합니다.

들어가며

클라우드 인프라가 복잡해질수록 콘솔에서 수동으로 리소스를 관리하는 것은 비효율적이고 위험합니다. IaC(Infrastructure as Code) 는 인프라를 코드로 정의하고 관리하는 방식이며, Terraform은 그 대표적인 도구입니다. 멀티 클라우드를 지원하고, 선언적 문법으로 직관적이며, 거대한 생태계를 가지고 있습니다.

IaC가 필요한 이유

  • 재현성: 동일한 환경을 여러 번 만들 수 있음
  • 버전 관리: Git으로 인프라 변경 이력 추적
  • 자동화: CI/CD 파이프라인과 통합
  • 문서화: 코드 자체가 인프라 명세서
  • 협업: 코드 리뷰로 변경사항 검토

Terraform의 특징

  • 선언적 언어(HCL): 원하는 상태만 기술
  • 멀티 클라우드 지원: AWS, GCP, Azure, K8s 등
  • 상태 관리: 실제 인프라와 코드의 매핑 정보 보관
  • 플랜과 적용 분리: 변경사항을 미리 확인 가능

설치와 첫 실행

기본 워크플로

  1. terraform init: 프로바이더 설치
  2. terraform plan: 변경사항 미리보기
  3. terraform apply: 인프라에 적용
  4. terraform destroy: 리소스 삭제

상태 파일과 백엔드

Terraform은 실제 인프라와 코드를 매핑하기 위해 상태 파일(terraform.tfstate) 을 사용합니다. 로컬에 저장하면 협업이 어려우므로, 보통 원격 백엔드(S3 + DynamoDB)를 사용합니다.

DynamoDB는 state locking에 사용되어, 여러 명이 동시에 적용하지 못하도록 막아줍니다.

공개 모듈은 Terraform Registry에서 찾아 쓸 수도 있습니다.

Data Source: 기존 리소스 참조

워크스페이스로 환경 분리

실무에서는 워크스페이스보다 디렉토리 분리 방식을 더 선호하는 경우가 많습니다.

자주 쓰는 명령어 정리

모범 사례

  • 상태 파일은 반드시 원격 백엔드에 저장
  • 민감 정보는 환경 변수나 Secret Manager 사용 (코드에 직접 X)
  • 환경별 디렉토리 분리 (dev/, staging/, prod/)
  • 모듈로 재사용성 확보
  • terraform plan 결과를 PR에 첨부해 리뷰
  • tfsec, Checkov 같은 보안 스캐너 적용

마무리

Terraform은 클라우드 인프라를 코드로 안전하게 관리할 수 있게 해주는 강력한 도구입니다. 변수, 모듈, 원격 백엔드, 데이터 소스 같은 기능을 익혀두면 작은 프로젝트부터 대규모 멀티 클라우드 환경까지 일관된 방식으로 운영할 수 있습니다. 처음에는 단순한 EC2 인스턴스부터 시작해, 점차 모듈과 CI/CD 파이프라인을 추가하는 방향으로 성장시켜 가는 것을 추천합니다.
