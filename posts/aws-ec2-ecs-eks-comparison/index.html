AWS EC2, ECS, EKS 비교

작성일 2026-05-26 Edited on 2026-05-20 In Cloud , Infrastructure
AWS의 컴퓨팅 서비스인 EC2, ECS, EKS의 차이와 선택 기준을 정리합니다.

들어가며

AWS에는 애플리케이션을 실행할 수 있는 여러 컴퓨팅 서비스가 있습니다. 그중에서도 EC2, ECS, EKS는 가장 자주 비교되는 세 가지입니다. 비슷해 보이지만 추상화 수준과 운영 책임의 범위가 다릅니다. 이 글에서는 각 서비스의 특징과 선택 기준을 정리합니다.

EC2: 가상 서버

Amazon EC2(Elastic Compute Cloud) 는 AWS에서 가장 기본적이고 오래된 컴퓨팅 서비스입니다. 가상 머신을 직접 실행하고 운영체제, 미들웨어, 애플리케이션을 자유롭게 구성할 수 있습니다.

특징:

  • 인스턴스 타입 선택(t-시리즈, m-시리즈, c-시리즈 등)
  • AMI(Amazon Machine Image)로 부팅
  • EBS, S3, EFS 등 스토리지 연결
  • 보안 그룹과 VPC로 네트워크 구성

사용 시나리오

  • 레거시 애플리케이션 마이그레이션
  • 컨테이너화하기 어려운 워크로드
  • 운영체제 수준 제어가 필요한 경우
  • 특정 GPU/메모리 인스턴스가 필요한 경우

ECS: AWS 네이티브 컨테이너 오케스트레이션

Amazon ECS(Elastic Container Service) 는 AWS가 자체 개발한 컨테이너 오케스트레이션 플랫폼입니다. Kubernetes보다 단순하며 AWS 서비스와 깊게 통합되어 있습니다.

핵심 개념:

  • Task Definition: 컨테이너의 명세 (이미지, CPU, 메모리, 환경 변수)
  • Task: Task Definition의 실행 인스턴스
  • Service: 일정 수의 Task를 유지하는 컨트롤러
  • Cluster: Task가 실행되는 논리 그룹

실행 모드는 두 가지입니다.

모드 설명
EC2 Launch Type EC2 인스턴스 위에서 컨테이너 실행, 인프라 관리 필요
Fargate 서버리스, 컨테이너만 신경 쓰면 됨

사용 시나리오

  • AWS 중심의 컨테이너 운영
  • Kubernetes 학습 부담을 피하고 싶을 때
  • Fargate로 서버리스 운영 원할 때
  • IAM, ALB, CloudWatch와 빠르게 통합하려 할 때

EKS: 매니지드 Kubernetes

Amazon EKS(Elastic Kubernetes Service) 는 AWS가 운영하는 매니지드 Kubernetes 서비스입니다. 컨트롤 플레인은 AWS가 관리하고, 사용자는 워커 노드와 워크로드만 관리합니다.

특징:

  • 표준 Kubernetes API와 100% 호환
  • EC2, Fargate 모두 워커 노드로 사용 가능
  • AWS IAM과 RBAC 통합
  • 클라우드 간 이식성 우수

사용 시나리오

  • 멀티 클라우드/하이브리드 전략
  • Kubernetes 생태계 도구를 적극 활용할 때 (Helm, Istio, ArgoCD 등)
  • 대규모 마이크로서비스 운영
  • 표준 K8s 경험 보유

세 서비스 비교

항목 EC2 ECS EKS
추상화 수준 VM 컨테이너 컨테이너(K8s)
학습 곡선 낮음 중간 높음
AWS 종속성 중간 높음 낮음
운영 부담 높음 낮음(Fargate 시 최소) 중간
가격(컨트롤 플레인) 없음 없음 시간당 $0.10
이식성 낮음 낮음 높음
생태계 모든 OS/소프트웨어 AWS 중심 CNCF 표준

선택 기준

EC2를 선택해야 할 때

  • 컨테이너화가 어려운 레거시 애플리케이션
  • 운영체제와 커널 수준 제어가 필요할 때
  • 특정 하드웨어(GPU, 고메모리)가 필요한 경우

ECS를 선택해야 할 때

  • AWS 중심으로 운영하며 단순함을 원할 때
  • Fargate로 서버리스 컨테이너 운영
  • Kubernetes의 복잡도를 피하고 싶을 때

EKS를 선택해야 할 때

  • Kubernetes 표준 API 필요
  • 멀티 클라우드 또는 온프레미스 호환성 필요
  • 복잡한 마이크로서비스 아키텍처
  • CNCF 생태계 도구 적극 활용

Fargate: 공통 서버리스 옵션

ECS와 EKS 모두 Fargate를 통해 서버리스 컨테이너를 실행할 수 있습니다. 노드 관리, OS 패치, 스케일링을 AWS가 자동 처리합니다.

장점:

  • 인프라 관리 불필요
  • 컨테이너 단위 과금
  • 자동 격리와 보안 강화

단점:

  • 인스턴스 직접 사용 대비 비용 높음
  • 일부 고급 기능 제한

마무리

EC2는 가장 유연하고 익숙한 가상 서버, ECS는 AWS 네이티브의 단순한 컨테이너 플랫폼, EKS는 표준 Kubernetes 기반의 강력한 오케스트레이션을 제공합니다. 정답은 없습니다. 팀의 Kubernetes 역량, AWS 의존도, 이식성 요구사항, 운영 부담 등을 종합해 결정해야 합니다. 시작이 어렵다면 ECS + Fargate 조합으로 단순하게 출발해, 필요 시 EKS로 확장하는 전략도 좋은 선택입니다.
