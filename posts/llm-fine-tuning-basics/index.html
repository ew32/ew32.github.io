LLM 파인튜닝 기초

작성일 2026-07-12 Edited on 2026-07-20 In AI도구
LLM 파인튜닝의 개념과 종류, LoRA와 SFT의 차이, 실제 작업 흐름까지 입문자 관점에서 정리합니다.

파인튜닝이란

파인튜닝은 사전학습된 LLM의 가중치 일부 또는 전부를 도메인 데이터로 추가 학습시키는 기법입니다. 프롬프트 엔지니어링이나 RAG로 해결되지 않는 다음과 같은 상황에 유용합니다.

  • 모델이 특정 형식이나 어조를 일관되게 따라야 할 때
  • 사내 전문 용어를 자연스럽게 이해해야 할 때
  • 추론 비용/지연을 줄이기 위해 작은 모델에 지식을 압축할 때

반면 단순히 “최신 사실”을 알게 하고 싶다면 RAG가 더 적합합니다.

파인튜닝 종류

Full Fine-tuning

모든 파라미터를 갱신합니다. 효과는 크지만 GPU 메모리 요구가 매우 큽니다.

LoRA (Low-Rank Adaptation)

기존 가중치를 동결하고, 각 레이어에 작은 저랭크 행렬만 학습합니다. 7B 모델도 단일 GPU(24GB)에서 학습 가능할 정도로 효율적이며, 어댑터 파일이 수십 MB로 작아 배포가 쉽습니다.

QLoRA

모델을 4bit로 양자화한 뒤 LoRA를 적용해 메모리를 더 줄입니다.

SFT vs RLHF/DPO

  • SFT(Supervised Fine-tuning): 입력-출력 쌍으로 단순 지도학습
  • DPO/RLHF: 선호 데이터(좋은 답 vs 나쁜 답)로 정렬 학습

데이터셋 준비

파인튜닝 품질의 80%는 데이터에서 나옵니다. JSONL 형식의 instruction 데이터셋이 일반적입니다.

주의할 점은 양보다 질입니다. 일관된 1,000개가 잡음 섞인 10,000개보다 낫습니다.

LoRA 학습 예시

Hugging Face의 peft와 trl 라이브러리를 사용한 최소 예시입니다.

평가와 함정

학습 손실만 보지 말고 별도의 평가셋에서 정성/정량 평가를 합니다.

  • 과적합: 학습 데이터 그대로 외워버려 일반화 실패
  • 카타스트로픽 망각: 기존 일반 능력 손상. 일반 instruction 데이터를 일부 섞어 완화
  • 분포 편향: 데이터가 한쪽에 치우치면 응답도 치우침

언제 파인튜닝 대신 다른 방법을 쓸까

상황 권장
사실/지식 주입 RAG
응답 형식/스타일 고정 파인튜닝
빠른 프로토타입 프롬프트 엔지니어링
도메인 + 형식 모두 필요 RAG + 파인튜닝 결합

마무리

파인튜닝은 만능이 아니지만 적절한 상황에서는 강력한 도구입니다. LoRA로 시작해 작은 데이터셋부터 실험하고, 평가 파이프라인을 먼저 갖추는 것이 성공의 지름길입니다.
