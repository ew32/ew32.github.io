Vim 기초 - 모드, 이동, 편집

작성일 2026-05-28 Edited on 2026-05-19 In 개발환경
Vim의 모드 개념과 기본 이동, 편집, 시각 모드 사용법을 입문자 관점에서 정리합니다.

왜 Vim인가?

Vim은 1991년 등장한 모달(modal) 텍스트 에디터로, 거의 모든 유닉스 계열 시스템에 기본 탑재되어 있습니다. 마우스 없이 키보드만으로 효율적으로 텍스트를 편집할 수 있도록 설계되었으며, 한 번 익숙해지면 다른 에디터에서도 Vim 키바인딩을 찾게 될 만큼 중독성이 강합니다.

원격 서버 작업, Git 커밋 메시지, 설정 파일 수정 등 터미널 환경의 표준 도구이므로 개발자라면 기본기는 익혀두는 것이 좋습니다.

모드(Mode)의 이해

Vim의 가장 큰 특징은 모드입니다. 모드마다 키의 의미가 달라집니다.

모드 진입 키 용도
Normal Esc 이동, 명령 실행 (기본 모드)
Insert i, a, o 텍스트 입력
Visual v, V, Ctrl+v 영역 선택
Command : ex 명령어 입력
Replace R 덮어쓰기

다른 에디터와 가장 큰 차이는 Normal 모드가 기본이라는 점입니다. 입력하려면 명시적으로 Insert 모드로 들어가야 합니다.

기본 이동

Normal 모드에서 화살표 대신 hjkl을 사용합니다.

명령 동작
w 다음 단어 시작으로
b 이전 단어 시작으로
e 단어 끝으로
0 줄 처음으로
^ 줄의 첫 공백 아닌 문자로
$ 줄 끝으로
gg 파일 처음으로
G 파일 끝으로
42G 42번째 줄로
Ctrl+d 반 페이지 아래
Ctrl+u 반 페이지 위

Insert 모드 진입

명령 동작
i 커서 위치에 삽입
I 줄 처음에 삽입
a 커서 다음에 삽입
A 줄 끝에 삽입
o 아래 새 줄 만들고 삽입
O 위에 새 줄 만들고 삽입

삽입을 마치면 항상 Esc로 Normal 모드에 돌아옵니다.

편집 명령

Vim 편집의 강력함은 연산자(operator) + 대상(motion) 조합에서 나옵니다.

연산자 의미
d 삭제 (delete)
c 변경 (change, 삭제 후 insert)
y 복사 (yank)
p 붙여넣기 (paste)

이를 motion과 결합합니다.

**텍스트 객체(text object)**는 더 정교한 대상 선택을 가능하게 합니다.

객체 의미
iw inner word
aw a word (공백 포함)
i" 따옴표 안쪽
a" 따옴표 포함
ip 단락 안쪽
i{ 중괄호 블록 안쪽

예를 들어 ci"는 따옴표 안 내용을 모두 지우고 입력 모드로 진입합니다.

실행 취소와 재실행

명령 동작
u undo
Ctrl+r redo
. 마지막 명령 반복

. 명령은 Vim의 진가가 드러나는 기능으로, 동일한 편집을 여러 위치에 빠르게 적용할 수 있습니다.

Visual 모드

선택 후 편집하는 방식이 익숙하다면 Visual 모드를 사용합니다.

명령 동작
v 문자 단위 선택
V 라인 단위 선택
Ctrl+v 블록(사각형) 선택

선택 후 d, y, c 등 연산자를 적용할 수 있습니다. 블록 모드는 여러 줄에 같은 텍스트를 삽입할 때 유용합니다.

검색과 치환

파일과 명령 모드

기본 설정 (.vimrc)

Vim 동작은 홈 디렉터리의 .vimrc로 커스터마이즈합니다.

학습 팁

  • vimtutor 명령으로 30분짜리 공식 튜토리얼을 진행해보세요
  • 처음 일주일은 일부러 다른 에디터 사용을 줄이고 Vim만 사용
  • 화살표 키 사용을 금지하면 hjkl이 빠르게 손에 익음
  • 한 번에 모든 명령을 외우지 말고 매주 3~5개씩 추가

마무리

Vim의 학습 곡선은 가파르지만, 모드 + 연산자 + 모션이라는 일관된 문법을 이해하면 의외로 빠르게 익숙해집니다. 익숙한 IDE에 Vim 키바인딩 플러그인을 설치해 점진적으로 도입하는 것도 좋은 방법입니다. 한 번 손에 익으면 평생 무기가 됩니다.

# Linux # Vim # 에디터 # 단축키
VSCode 필수 단축키와 확장 프로그램
tmux 활용 - 터미널 멀티플렉서 입문
