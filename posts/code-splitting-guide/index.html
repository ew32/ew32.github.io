코드 스플리팅과 동적 임포트

작성일 2026-05-22 Edited on 2026-06-15 In Frontend , JavaScript
번들 크기를 줄이고 초기 로딩 성능을 개선하는 코드 스플리팅 전략을 정리합니다.

코드 스플리팅이란?

**코드 스플리팅(Code Splitting)**은 하나의 거대한 번들을 여러 개의 작은 청크(chunk)로 분리하여, 사용자가 실제 필요로 하는 코드만 다운로드하게 만드는 기법입니다. 초기 페이지 로딩 시 모든 코드를 받아오지 않으므로 TTI(Time To Interactive)와 LCP가 크게 개선됩니다.

대규모 SPA에서 전체 번들이 수 MB에 달하면, 사용자가 처음 진입하는 페이지와 무관한 코드까지 다운로드, 파싱, 컴파일됩니다. 코드 스플리팅은 이 비효율을 제거합니다.

동적 임포트 (Dynamic Import)

ES2020 표준인 import() 함수는 Promise를 반환하며, 런타임에 모듈을 비동기로 로드합니다.

Webpack, Rollup, Vite 등 모든 모던 번들러는 import()를 자동으로 별도 청크로 분리합니다.

React에서의 코드 스플리팅

React.lazy + Suspense

라우트나 컴포넌트 단위로 코드를 분리할 때 가장 일반적인 패턴입니다.

라우트 기반 vs 컴포넌트 기반

전략 장점 단점
라우트 기반 구현 단순, 효과적 페이지 내 무거운 컴포넌트 미해결
컴포넌트 기반 세밀한 제어 관리 복잡도 상승
라이브러리 분리 외부 의존성 격리 적절한 vendor 청크 설계 필요

Webpack 설정

Webpack의 splitChunks 옵션으로 공통 모듈을 자동 분리할 수 있습니다.

Magic Comments

Webpack에서는 주석으로 청크 이름과 로딩 방식을 지정할 수 있습니다.

  • webpackPrefetch: 브라우저가 한가할 때 미리 로드
  • webpackPreload: 부모 청크와 병렬로 로드

Prefetch와 Preload

키워드 우선순위 사용 시점
preload 높음 현재 페이지에 곧 필요한 리소스
prefetch 낮음 다음 페이지에서 필요할 가능성 있는 리소스

조건부 로드 패턴

기능이 실제로 필요한 시점에만 로드합니다.

Tree Shaking과의 조합

코드 스플리팅과 Tree Shaking(사용하지 않는 코드 제거)을 함께 활용해야 효과가 극대화됩니다.

package.json에 "sideEffects": false를 추가하면 번들러가 더 적극적으로 Tree Shaking을 수행합니다.

청크 분석

webpack-bundle-analyzer로 번들 구성을 시각화하면 어디서부터 분리해야 할지 명확해집니다.

Vite는 rollup-plugin-visualizer를 사용합니다.

베스트 프랙티스

  • 초기 번들은 150~200KB(gzipped) 이하 유지
  • 라우트 단위 분리부터 시작
  • 자주 사용되지 않는 무거운 라이브러리(차트, 에디터, 지도)는 별도 청크로
  • Vendor 청크는 캐시 효율을 위해 분리
  • Suspense fallback은 가벼운 스켈레톤 UI 권장

마무리

코드 스플리팅은 단순히 번들을 작게 만드는 것이 아니라, 사용자 여정과 우선순위에 맞춰 코드를 배치하는 작업입니다. 동적 임포트, React.lazy, Webpack splitChunks 등을 조합하면 대규모 SPA도 빠른 초기 로딩과 매끄러운 페이지 전환을 동시에 달성할 수 있습니다.
