CSS Flexbox와 Grid 완벽 비교

작성일 2026-05-05 Edited on 2026-05-19 In Frontend , CSS
현대 CSS 레이아웃의 두 축인 Flexbox와 Grid의 사용법과 차이점을 정리합니다.

Flexbox vs Grid

항목 Flexbox Grid
차원 1차원 (행 또는 열) 2차원 (행과 열)
적합한 경우 컴포넌트 내부 정렬, 메뉴, 카드 페이지 전체 레이아웃, 복잡한 격자
흐름 방향 콘텐츠 중심 레이아웃 중심

두 기술은 대체재가 아니라 상호 보완재입니다. 페이지 전체 구조는 Grid로, 컴포넌트 내부는 Flexbox로 잡는 것이 일반적입니다.

주축과 교차축

flex-direction이 row면 주축은 가로, 교차축은 세로입니다. column이면 반대입니다. justify-content는 주축을, align-items는 교차축을 제어한다는 점이 핵심입니다.

정렬 속성 비교

두 레이아웃 모두 justify-content, align-items, justify-items, align-self를 사용하지만 의미가 약간 다릅니다.

속성 Flexbox Grid
justify-content 주축 정렬 인라인 축 정렬
align-items 교차축 정렬 블록 축 정렬
place-items 미지원 align-items + justify-items

브라우저 지원 및 폴백

현대 브라우저는 두 기술 모두 완벽 지원합니다. IE 지원이 필요한 경우(드뭅니다) @supports를 활용해 폴백을 제공할 수 있습니다.

선택 기준 정리

  • 한 방향으로만 배치: Flexbox
  • 콘텐츠가 공간을 정함: Flexbox
  • 격자형 2차원 레이아웃: Grid
  • 레이아웃이 콘텐츠를 정함: Grid
  • 요소가 겹쳐야 함: Grid (grid-area 활용)

마무리
