웹 이미지 최적화 전략 - WebP, AVIF, Lazy Loading

작성일 2026-05-22 Edited on 2026-05-20 In Frontend , HTML
최신 이미지 포맷과 로딩 전략으로 웹 성능을 극대화하는 종합 가이드입니다.

왜 이미지 최적화가 중요한가?

평균적인 웹페이지 전송 용량의 약 50% 이상이 이미지입니다. HTTP Archive 통계에 따르면 페이지당 평균 이미지 용량은 1MB를 넘어서며, 이는 LCP에 가장 큰 영향을 주는 요소입니다. 이미지 최적화는 가장 적은 노력으로 가장 큰 성능 향상을 얻을 수 있는 영역입니다.

차세대 이미지 포맷

WebP

Google이 2010년 발표한 포맷으로, JPEG 대비 25~35% 작은 용량을 제공합니다.

  • 장점: 모든 모던 브라우저 지원 (96%+), 손실/무손실 모두 지원, 알파 채널 지원
  • 단점: 일부 구형 브라우저 미지원

AVIF

AV1 코덱 기반의 최신 포맷으로, WebP보다 약 20~30% 더 작은 용량을 제공합니다.

  • 장점: 최고 압축률, HDR 지원, 광범위한 색 영역
  • 단점: 인코딩 속도가 느림, 일부 브라우저 미지원
포맷 평균 용량 (vs JPEG) 브라우저 지원 알파 채널
JPEG 100% 100% 미지원
WebP 70% 96% 지원
AVIF 50% 90% 지원

picture 요소로 포맷 폴백

브라우저 호환성을 유지하면서 최신 포맷을 활용하려면 <picture> 요소를 사용합니다.

브라우저는 위에서부터 지원하는 포맷을 선택하며, 모두 미지원이면 마지막 <img> 태그를 사용합니다.

반응형 이미지

다양한 화면 크기에 맞는 이미지를 제공하면 모바일에서 불필요한 대용량 이미지 전송을 막을 수 있습니다.

  • srcset: 사용 가능한 이미지 후보와 너비 기술
  • sizes: 뷰포트별 이미지의 실제 표시 크기 안내

Lazy Loading

화면에 보이지 않는 이미지는 늦게 로드하여 초기 페이지 로딩을 가속화합니다.

네이티브 lazy loading

  • loading="lazy": 뷰포트 근처로 스크롤될 때 로드
  • loading="eager": 즉시 로드 (기본값)
  • 첫 화면(LCP) 이미지에는 절대 lazy 사용 금지

IntersectionObserver 활용

더 세밀한 제어가 필요한 경우 IntersectionObserver를 사용합니다.

fetchpriority와 preload

LCP 이미지처럼 중요한 리소스는 우선순위를 명시적으로 높여야 합니다.

이미지 CDN 활용

이미지 CDN은 URL 파라미터만으로 동적으로 포맷 변환, 리사이즈, 압축을 수행합니다.

  • Cloudinary: f_auto,q_auto,w_800
  • Imgix: ?auto=format&w=800
  • Cloudflare Images: 자동 최적화

SVG와 아이콘 최적화

아이콘은 가능한 한 SVG를 사용하고, SVGO로 최적화합니다.

인라인 SVG는 추가 요청을 줄이며, CSS로 색상 변경이 가능한 장점이 있습니다.

베스트 프랙티스 체크리스트

  • 모든 이미지에 width와 height 속성 명시 (CLS 방지)
  • WebP/AVIF 포맷 + JPEG 폴백 제공
  • srcset과 sizes로 반응형 처리
  • LCP 이미지: fetchpriority="high" + preload
  • 뷰포트 밖 이미지: loading="lazy"
  • 의미 있는 alt 속성 (접근성)
  • 이미지 CDN 도입 검토

마무리

이미지 최적화는 포맷 선택, 크기 조정, 로딩 전략의 세 축으로 접근해야 합니다. AVIF/WebP 도입과 반응형 이미지, lazy loading을 조합하면 대부분의 웹사이트에서 페이지 용량을 절반 이하로 줄일 수 있으며, LCP가 크게 개선됩니다.
