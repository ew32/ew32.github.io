CDN 동작 원리와 활용

작성일 2026-05-24 Edited on 2026-05-20 In Network , Theory
전 세계 사용자에게 빠르게 콘텐츠를 전달하는 CDN의 구조와 캐싱 전략을 정리합니다.

CDN이란?

**CDN(Content Delivery Network)**은 전 세계 여러 지역에 분산 배치된 서버 네트워크로, 사용자에게 지리적으로 가까운 위치에서 콘텐츠를 전달하는 시스템입니다. 정적 자산뿐 아니라 최근에는 동적 콘텐츠, API, 엣지 컴퓨팅까지 영역을 확장하고 있습니다.

CDN을 사용하면 다음과 같은 효과를 얻을 수 있습니다.

  • 지연 시간 감소: 사용자와 가까운 엣지에서 응답
  • 오리진 부하 감소: 캐시 히트로 원본 서버 보호
  • 대역폭 비용 절감: 엣지에서 압축, 캐싱
  • 가용성 향상: 오리진 장애 시에도 캐시된 콘텐츠 제공
  • 보안 강화: DDoS 방어, WAF, 봇 차단

CDN의 구조

CDN은 크게 두 가지 계층으로 구성됩니다.

1. 오리진 서버 (Origin Server)

콘텐츠의 원본을 보유한 서버. 사용자가 직접 접근하지 않으며, CDN을 통해서만 전달됩니다.

2. 엣지 서버 (Edge Server / PoP)

전 세계 주요 도시에 배치된 캐시 서버. **PoP(Point of Presence)**라고도 부릅니다. 사용자 요청은 가장 가까운 PoP로 라우팅됩니다.

사용자 요청 라우팅

사용자가 가장 가까운 엣지로 라우팅되는 방법은 크게 두 가지입니다.

DNS 기반 라우팅

GeoDNS로 지역별로 다른 엣지 IP를 반환합니다.

Anycast 라우팅

같은 IP 주소가 전 세계 여러 위치에서 BGP로 광고됩니다. 라우팅 프로토콜이 가장 가까운 위치로 자동 전달합니다. Cloudflare, AWS CloudFront 일부 등이 이 방식을 사용합니다.

캐싱 동작

CDN의 핵심은 캐시입니다. 캐시 동작은 HTTP 헤더로 제어됩니다.

주요 캐시 헤더

헤더 역할
Cache-Control 캐시 정책 (max-age, public, private 등)
Expires 만료 시각 (구식)
ETag 콘텐츠 식별자 (조건부 요청용)
Last-Modified 마지막 수정 시각
Vary 캐시 키에 포함할 헤더

캐시 정책 예시

s-maxage는 CDN 같은 공유 캐시 전용 TTL입니다.

캐시 키와 Vary

CDN은 URL을 기본 캐시 키로 사용하지만, 같은 URL이라도 헤더에 따라 다른 응답을 줘야 하면 Vary 헤더로 명시합니다.

위 응답은 압축 방식과 언어별로 별도 캐시 엔트리를 가지게 됩니다.

캐시 히트율과 무효화

캐시 히트율이 90% 이상이면 오리진 트래픽이 1/10 이하로 줄어듭니다.

캐시 무효화 방법

  1. TTL 만료 대기: 가장 단순, 즉시 반영은 어려움
  2. Purge API: CDN에 명시적 무효화 요청
  3. 버전 URL: app.js?v=2 또는 app.abc123.js (권장)
  4. Soft Purge: 만료로 표시하되 갱신 전까지 stale 사용

동적 콘텐츠와 엣지 컴퓨팅

전통적 CDN은 정적 자산에 집중했지만, 현대 CDN은 동적 콘텐츠도 가속합니다.

동적 콘텐츠 가속 기법

  • 연결 재사용: 엣지-오리진 간 keep-alive
  • TCP 최적화: 엣지에서 BBR 등 혼잡 제어
  • TLS 종료: 엣지에서 TLS 처리, 오리진은 평문 또는 mTLS
  • HTTP/3 활성화: 사용자-엣지 구간만 적용해도 큰 효과

엣지 컴퓨팅

CDN 엣지에서 직접 코드를 실행하는 서비스가 일반화되었습니다.

서비스 런타임
Cloudflare Workers V8 isolate
AWS Lambda@Edge Node.js, Python
Vercel Edge Functions V8 isolate
Fastly Compute@Edge WebAssembly

CDN 도입 시 고려사항

캐시 가능 콘텐츠 식별

콘텐츠 캐시 적합성 TTL 권장
이미지, 비디오 매우 적합 1년
CSS, JS (해시) 매우 적합 1년 immutable
HTML 보통 1~5분
API GET (공개) 가능 1~60초
API POST/PUT 부적합 no-store
개인화 페이지 부적합 no-store

보안 고려사항

  • 개인정보 응답에 캐시 헤더 누락 금지: Cache-Control: private 필수
  • 인증된 응답 캐시 주의: Vary: Authorization 또는 캐시 제외
  • TLS 인증서 관리: CDN이 발급/갱신 자동화

주요 CDN 서비스

서비스 강점 특징
Cloudflare 가격, DDoS 방어 무료 플랜, Workers
AWS CloudFront AWS 통합 Lambda@Edge
Fastly 실시간 무효화 VCL, Compute@Edge
Akamai 글로벌 커버리지 엔터프라이즈
KT, LG U+ CDN 한국 최적화 국내 PoP

성능 측정

CDN 효과는 다음 지표로 측정합니다.

  • TTFB (Time To First Byte): 엣지 도입으로 큰 감소
  • 캐시 히트율: 90% 이상 목표
  • 오리진 대역폭: 캐시로 절감되는 양
  • 에러율: 5xx 응답 비율

마무리

CDN은 단순히 정적 파일을 빠르게 전달하는 도구를 넘어, 글로벌 사용자에게 일관된 성능과 가용성을 제공하는 인프라로 진화하고 있습니다. 적절한 캐시 헤더 설계, 콘텐츠별 TTL 전략, 그리고 엣지 컴퓨팅 활용까지 고려하면 비용을 절감하면서 사용자 경험을 크게 향상시킬 수 있습니다.
