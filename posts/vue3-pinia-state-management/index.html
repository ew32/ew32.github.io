Vue 3에서 Pinia로 상태 관리하기

작성일 2026-06-08 Edited on 2026-07-20 In Frontend , JavaScript
Vuex의 공식 후속인 Pinia의 핵심 개념과 Composition API 기반 사용법, Vuex와의 차이를 정리합니다.

Pinia란?

Pinia는 Vue 3의 공식 상태 관리 라이브러리로, Vuex 5 제안을 기반으로 만들어진 사실상의 Vuex 후속입니다. Vue 2/3 모두 지원하며, TypeScript 지원, Devtools 통합, 모듈 분리가 자연스러운 점이 특징입니다.

Vuex와 비교한 주요 변화:

  • Mutation 제거: state를 action에서 직접 변경
  • 네임스페이스 없음: 각 store가 곧 모듈
  • 타입 추론 자동: TS 환경에서 그대로 좋음
  • Composition API와 자연스럽게 결합

설치 및 등록

Option Store

기존 Vuex와 유사한 형태입니다.

Setup Store

Composition API 스타일로 작성할 수도 있습니다. ref/computed/함수가 곧 state/getter/action에 대응됩니다.

컴포넌트에서 사용

storeToRefs를 사용하지 않고 일반 구조분해를 하면 반응성을 잃습니다.

state 직접 변경 vs $patch

$patch는 변경을 묶어 한 번의 트랜잭션처럼 적용하므로 Devtools에서 보기 좋습니다.

스토어 간 조합

스토어는 함수 안에서 다른 스토어를 자유롭게 호출할 수 있습니다.

영속화 플러그인

pinia-plugin-persistedstate로 localStorage에 자동 저장할 수 있습니다.

마무리

Pinia는 Vuex보다 간결한 API와 우수한 타입 추론, 자연스러운 Composition API 통합을 제공합니다. 작은 프로젝트에서는 Setup Store, 큰 프로젝트에서는 Option Store로 시작하면 무리 없이 도입할 수 있습니다. Vue 3 신규 프로젝트라면 Vuex 대신 Pinia를 선택하는 것이 표준입니다.
