MongoDB 기초 가이드

작성일 2026-05-10 Edited on 2026-05-20 In Backend , Database
대표적인 문서지향 NoSQL 데이터베이스 MongoDB의 구조, 기본 CRUD, 인덱스, 데이터 모델링 패턴을 정리합니다.

MongoDB란

MongoDB는 JSON 유사 형식의 BSON 문서를 저장하는 문서지향(document-oriented) NoSQL 데이터베이스입니다. 스키마가 유연해 빠른 개발이 가능하고, 수평 확장(샤딩)이 기본 설계에 포함되어 있습니다.

RDBMS 용어 MongoDB 용어
Database Database
Table Collection
Row Document
Column Field
Join $lookup / 임베딩

문서 구조

문서는 BSON(Binary JSON)으로 저장되며, 중첩 객체와 배열을 자유롭게 가질 수 있습니다.

_id는 자동 생성되는 기본키이며, ObjectId는 12바이트로 타임스탬프를 포함합니다.

인덱스

MongoDB도 B-Tree 인덱스를 사용합니다. 인덱스가 없으면 컬렉션 풀스캔이 발생합니다.

집계 파이프라인

aggregate는 SQL의 GROUP BY를 일반화한 강력한 도구입니다.

자주 쓰는 스테이지: $match, $group, $project, $lookup, $unwind, $sort, $limit.

데이터 모델링 패턴

임베딩(Embedding)

연관된 데이터를 한 문서에 함께 저장합니다. 읽기 성능이 좋고 원자성이 보장됩니다.

참조(Referencing)

다른 컬렉션의 _id를 저장하고 $lookup으로 조인합니다.

선택 기준

상황 추천 패턴
1:소수, 함께 자주 조회 임베딩
1:다수, 무제한 성장 참조
자주 갱신되는 공유 데이터 참조
16MB 문서 크기 초과 가능성 참조

트랜잭션

MongoDB 4.0부터 멀티 도큐먼트 ACID 트랜잭션을 지원합니다.

다만 가능하면 단일 문서로 모델링해 원자성을 자연스럽게 확보하는 편이 좋습니다.

복제와 샤딩

  • Replica Set: 동일한 데이터를 여러 노드에 복제, 자동 페일오버
  • Sharding: 데이터를 샤드 키로 분산해 수평 확장

마무리

MongoDB는 유연한 스키마와 수평 확장성을 제공해 빠르게 변하는 도메인에 적합합니다. 그러나 “스키마 없음”은 곧 “스키마를 애플리케이션이 관리해야 함”을 의미합니다. 임베딩과 참조의 트레이드오프, 인덱스 전략, 집계 파이프라인을 이해하면 MongoDB의 강점을 충분히 끌어낼 수 있습니다.
