JavaScript 프로토타입 체인 이해하기

작성일 2026-05-18 Edited on 2026-05-26 In Frontend , JavaScript
JavaScript의 프로토타입 기반 상속과 프로토타입 체인 동작 원리를 정리합니다.

프로토타입이란

JavaScript는 프로토타입 기반(prototype-based) 언어입니다. 클래스 기반 언어와 달리, 객체가 다른 객체를 직접 상속받는 형태입니다. 모든 객체는 [[Prototype]] 이라는 내부 슬롯을 통해 자신의 부모 객체(프로토타입)를 참조합니다.

이 부모 객체에 정의된 프로퍼티와 메서드는 자식 객체에서도 사용 가능하며, 이를 가능하게 하는 메커니즘이 프로토타입 체인입니다.

__proto__와 prototype

두 용어는 자주 혼동되지만 의미가 다릅니다.

용어 의미
__proto__ 모든 객체가 가진, 자신의 프로토타입을 가리키는 내부 슬롯 접근자
prototype 함수만 가진 속성. new로 생성된 인스턴스의 __proto__가 될 객체

프로토타입 체인 동작

객체의 프로퍼티에 접근할 때 JavaScript는 다음 순서로 탐색합니다.

  1. 객체 자신에게 프로퍼티가 있는지 확인
  2. 없으면 __proto__(프로토타입)에서 찾음
  3. 거기에도 없으면 그 프로토타입의 __proto__로 이동
  4. null에 도달할 때까지 반복

프로토타입 상속 예시

이 패턴은 ES6 class 문법의 내부 동작 그 자체입니다.

ES6 class와의 관계

Object.create

직접 프로토타입을 지정해 객체를 만드는 가장 깔끔한 방법입니다.

hasOwnProperty와 in

프로토타입 체인을 거치지 않고 객체 자신의 속성만 확인하려면 hasOwnProperty(또는 ES2022의 Object.hasOwn)를 사용합니다.

메서드 오버라이딩

탐색은 가장 가까운 프로토타입부터 시작하므로 자식의 메서드가 우선됩니다.

주의할 점

주의 사항 설명
__proto__ 직접 사용 비표준에 가깝고 성능 저하. Object.getPrototypeOf/setPrototypeOf 사용 권장
프로토타입 변경 비용 런타임에 프로토타입을 바꾸면 V8 엔진의 최적화가 깨짐
for...in은 상속 속성도 순회 Object.keys 또는 hasOwn 체크 필요
함수의 prototype은 객체 화살표 함수는 prototype 속성이 없어 생성자로 사용 불가

마무리

JavaScript의 객체 지향 모델은 클래스가 아닌 프로토타입에 기반합니다. class 문법은 편리한 외피일 뿐, 내부에서는 모든 것이 프로토타입 체인을 통해 동작합니다. 이 메커니즘을 이해하면 라이브러리 코드를 읽거나 디버깅할 때 큰 도움이 되며, 메모리 효율적인 객체 설계도 가능해집니다.
