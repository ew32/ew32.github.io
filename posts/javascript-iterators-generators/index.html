JavaScript Iterator와 Generator 패턴

작성일 2026-05-28 Edited on 2026-05-20 In Frontend , JavaScript
JavaScript의 반복 프로토콜과 제너레이터 함수를 활용해 지연 평가, 무한 시퀀스, 비동기 흐름을 구현하는 방법을 다룹니다.

Iterator 프로토콜

JavaScript의 반복 가능한 객체(iterable)는 [Symbol.iterator]() 메서드를 가지고 있으며, 이 메서드는 next()를 가진 iterator를 반환합니다. next()는 { value, done } 형태의 객체를 반환합니다.

for...of, 스프레드 연산자, 배열 구조 분해는 모두 이 프로토콜에 기반해 동작합니다.

Generator 함수

function* 구문으로 정의되며, 호출 시 즉시 실행되지 않고 generator 객체를 반환합니다. yield로 값을 하나씩 내보냅니다.

제너레이터는 그 자체로 iterable이자 iterator이므로 별도의 Symbol.iterator 구현 없이 바로 사용할 수 있습니다.

무한 시퀀스와 지연 평가

필요할 때만 값을 생성하기 때문에 메모리 효율적입니다.

yield* - 위임

다른 iterable에 반복을 위임할 수 있습니다.

양방향 통신

제너레이터는 외부에서 값을 주입할 수도 있습니다.

실전 예제 - 페이지네이션

async function*은 비동기 제너레이터로, for await...of로 소비할 수 있습니다.

마무리

Iterator와 Generator는 명령형 스타일을 유지하면서도 함수형 컬렉션 처리, 지연 평가, 비동기 스트림 처리를 가능하게 합니다. 데이터 양이 크거나 처리 흐름이 단계적인 경우 매우 강력한 도구가 됩니다.
